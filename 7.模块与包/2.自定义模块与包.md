# 自定义模块与包

## 1. 知识点定义

**自定义模块**就是你自己的 `.py` 文件，写好函数、类、变量保存即可。**包（Package）**是一个包含 `__init__.py` 文件的目录，用于将多个相关模块组织在一起。

> 小白理解：
> 模块是单个工具箱（一个 `.py` 文件），包是一组工具箱放进一个文件夹（目录 + `__init__.py`）。当你的工具箱太多，就需要用包来分类管理。

它主要解决的问题是：
- 将代码按功能拆分成多个文件，方便维护
- 将相关模块组织成层级结构，像文件夹一样管理
- 发布和共享你自己的代码库

| 概念 | 是什么 | 本质 |
|------|--------|------|
| 模块（Module） | 单个 `.py` 文件 | 文件 |
| 包（Package） | 含有 `__init__.py` 的目录 | 目录 |
| 库（Library） | 多个模块和包的集合 | 逻辑概念 |

## 2. 语法格式

### 2.1 创建和导入自定义模块

```python
# 文件: my_math.py —— 一个自定义数学工具模块
"""自定义数学工具模块"""

PI = 3.14159

def add(a, b):
    """加法"""
    return a + b

def multiply(a, b):
    """乘法"""
    return a * b

# 文件: main.py —— 导入自定义模块
import my_math                          # 方式一：import 模块名
print(my_math.add(3, 5))               # 8

from my_math import add, Circle         # 方式二：from ... import
print(add(10, 20))                      # 30

import my_math as mm                    # 方式三：as 起别名
print(mm.multiply(6, 7))               # 42
```

### 2.2 包的目录结构

```
my_package/               # 这是一个包
    __init__.py           # 包的初始化文件（标识这是一个包）
    module_a.py           # 模块 A
    module_b.py           # 模块 B
    sub_package/          # 子包
        __init__.py
        module_c.py
```

### 2.3 `__init__.py` 的三大作用

```python
# 作用一：标识这是一个包（没有它，Python 可能不识别为包）

# 作用二：定义 __all__ 控制 from package import * 的导出
# 文件: my_package/__init__.py
__all__ = ['add', 'Circle']      # 只导出这两个名称

from .module_a import add
from .module_b import Circle

# 作用三：初始化代码（包被首次导入时自动执行）
print(f"包 {__name__} 被加载了")
VERSION = "1.0.0"

# 在包级别导入常用模块，方便用户使用
from .module_a import calculate   # 用户可直接 my_package.calculate()
```

### 2.4 包的层级导入

```python
# 假设包结构:
# data_package/
#     __init__.py
#     reader.py       (有 read_csv 函数)
#     utils/
#         __init__.py
#         validator.py (有 validate 函数)

# 导入顶层模块
import data_package.reader
data_package.reader.read_csv("data.csv")

# from ... import ... 方式（更简洁）
from data_package.reader import read_csv
from data_package.utils.validator import validate
```

### 2.5 包内相对导入

```python
# 文件: myapp/services/user_service.py
# 需要导入同包的 helper 和父包的 config

from . import helper              # . 表示当前包（同目录）
from .. import config             # .. 表示父包（上一级目录）
from ..models.user import User    # 从兄弟包导入

# 符号含义:
# .  = 当前包（同一目录）
# .. = 父包（上一级目录）
# ... = 祖父包（上两级目录）
```

## 3. 重点说明

### 3.1 `__init__.py` 是包的"身份证"

Python 3.3+ 引入了"命名空间包"，没有 `__init__.py` 也可能被识别为包。但**显式创建 `__init__.py` 仍然是推荐做法**，因为它提供更明确的包语义和初始化逻辑。

```python
# 即使 __init__.py 是空文件，也要创建它
# 否则可能导致导入行为不一致
```

### 3.2 绝对导入 vs 相对导入的选择

| 对比维度 | 绝对导入 | 相对导入 |
|---------|---------|---------|
| 写法示例 | `from myapp.config import URL` | `from ..config import URL` |
| 可读性 | 路径一目了然 | 需要推算 `..` 回到哪里 |
| 重构友好 | 移动文件要改导入路径 | 包内移动影响较小 |
| 适用场景 | **日常开发优先推荐** | 大型包内部模块间 |
| 顶层脚本 | ✅ 可用 | ❌ 报错 |

> **建议**：日常开发优先使用绝对导入。包里模块很多且经常移动时，考虑相对导入。

### 3.3 相对导入直接运行报错的原因

```bash
$ python my_package/sub.py
# ImportError: attempted relative import with no known parent package
```

**原因**：直接运行时 `__name__` 是 `'__main__'`，Python 不认为它属于任何包，相对导入失效。

**解决方案**：
- 用 `python -m my_package.sub` 方式运行（推荐）
- 包内模块改用绝对导入

### 3.4 `__all__` 控制导出

如果没有定义 `__all__`，`from package import *` 会导入所有不以下划线开头的名称，造成命名污染。

```python
# __init__.py 中定义 __all__
__all__ = ['add', 'Circle']  # 只有这两个会被 from package import * 导入
```

## 5. 典型例题（附带答案）

### 例题 1

**题目：**

创建一个包 `text_tools/`，包含：
- `__init__.py`：定义 `__all__ = ['count_chars', 'count_words']`
- `char_utils.py`：定义 `count_chars(s)` 统计字符数
- `word_utils.py`：定义 `count_words(s)` 统计单词数

然后用 `from text_tools import *` 测试，验证只有 `__all__` 中的名字被导入。

**答案：**

```python
# ======== text_tools/__init__.py ========
__all__ = ['count_chars', 'count_words']

from .char_utils import count_chars
from .word_utils import count_words

# ======== text_tools/char_utils.py ========
def count_chars(s):
    """统计字符数"""
    return len(s)

def count_vowels(s):
    """统计元音字母数（不在 __all__ 中，不会被 * 导入）"""
    return sum(1 for c in s.lower() if c in 'aeiou')

# ======== text_tools/word_utils.py ========
def count_words(s):
    """统计单词数"""
    return len(s.split())

# ======== test.py ========
from text_tools import *

print(count_chars("hello"))       # 5 —— ✅ 在 __all__ 中
print(count_words("a b c"))       # 3 —— ✅ 在 __all__ 中
# print(count_vowels("hello"))    # NameError —— ❌ 不在 __all__ 中
```

**解析：** `__all__` 就像一个"白名单"，只有列表中的名称会被 `from package import *` 导出。

### 例题 2

**题目：**

创建三层包结构，在 `ship.py` 中用绝对导入从另外两个模块导入内容：

```
shop/
├── __init__.py
├── inventory.py      (定义 get_stock 函数)
├── billing/
│   ├── __init__.py
│   └── invoice.py   (定义 generate_invoice 函数)
└── delivery/
    ├── __init__.py
    └── ship.py       (从 inventory 和 invoice 导入)
```

**答案：**

```python
# ======== shop/inventory.py ========
def get_stock(product):
    stock = {"apple": 100, "banana": 50}
    return stock.get(product, 0)

# ======== shop/billing/invoice.py ========
def generate_invoice(items):
    total = sum(price for _, price in items)
    return f"总价: {total}元"

# ======== shop/delivery/ship.py ========
from shop.inventory import get_stock            # 绝对导入
from shop.billing.invoice import generate_invoice

def process_order(product, quantity, price):
    stock = get_stock(product)
    if stock >= quantity:
        invoice = generate_invoice([(product, price * quantity)])
        return f"订单已处理: {invoice}, 剩余库存: {stock - quantity}"
    return f"库存不足: 需要 {quantity}, 当前 {stock}"

if __name__ == '__main__':
    print(process_order("apple", 10, 5))
```

**解析：** 绝对导入从项目根目录写完整路径，在任何文件中写法一致，可读性最好。

## 6. 易错点

### 易错点 1：忘了创建 `__init__.py`

错误写法：
```python
# 创建了 my_package/ 目录，放了 module.py，但没有 __init__.py
import my_package  # 可能失败！
```

错误原因：没有 `__init__.py`，Python 可能不识别该目录为包。

正确写法：
```python
# 务必在包目录下创建 __init__.py（即使是空文件）
# my_package/
#     __init__.py   ← 必须有！
#     module.py
```

### 易错点 2：相对导入在顶层脚本中使用

错误写法：
```python
# main.py（顶层脚本）
from . import utils  # SyntaxError 或 ImportError！
```

错误原因：顶层脚本的 `__name__` 是 `'__main__'`，没有包上下文，无法使用相对导入。

正确写法：
```python
# main.py（顶层脚本）—— 用绝对导入
import utils
# 或
from utils import some_function
```

### 易错点 3：`from package import *` 导入不该导出的内容

错误写法：
```python
# __init__.py 没定义 __all__
from my_package import *   # 导入了所有内部模块名，造成命名污染
```

错误原因：没有 `__all__` 时，`*` 会导入所有不以下划线开头的名称。

正确写法：
```python
# __init__.py 中定义 __all__ 控制导出
__all__ = ['public_func1', 'public_func2']
```

### 易错点 4：修改模块代码后导入不更新

错误写法：
```python
import my_module
# 修改了 my_module.py，保存了
import my_module   # 用的还是旧的缓存！
```

错误原因：`sys.modules` 缓存了已导入的模块。

正确写法：
```python
import importlib
importlib.reload(my_module)   # 强制重新加载
# 或者重启 Python 解释器
```

## 7. 课后练习（附带答案）

### 练习 1

**题目：**

创建包 `calculator/`，包含：
```
calculator/
├── __init__.py
├── basic.py       (定义 add、subtract 函数)
└── advanced.py    (用相对导入从 basic 导入 add，实现 add_three)
```

在 `advanced.py` 中用相对导入引用 `basic.py` 中的函数。分别用 `python advanced.py`（直接运行）和 `python -m calculator.advanced`（模块方式运行），观察区别。

**答案：**

```python
# ======== calculator/__init__.py ========
# 空文件即可

# ======== calculator/basic.py ========
def add(a, b):
    return a + b

def subtract(a, b):
    return a - b

# ======== calculator/advanced.py ========
from .basic import add      # 相对导入

def add_three(a, b, c):
    return add(add(a, b), c)

if __name__ == '__main__':
    print(add_three(1, 2, 3))

# 直接运行: python calculator/advanced.py → ImportError!
# 正确运行: python -m calculator.advanced → 输出 6
```

**说明：** 相对导入只能通过 `python -m` 方式运行，因为需要包上下文。

### 练习 2

**题目：**

创建包 `my_utils/`，在 `__init__.py` 中定义 `__all__`，从子模块中导入函数。同时在 `__init__.py` 中定义一个包级别的常量 `VERSION`。测试 `from my_utils import *` 的效果。

**答案：**

```python
# ======== my_utils/__init__.py ========
__all__ = ['double', 'greet']
VERSION = "2.0.0"

from .math_utils import double
from .str_utils import greet

# ======== my_utils/math_utils.py ========
def double(x):
    return x * 2

def triple(x):        # 不在 __all__ 中
    return x * 3

# ======== my_utils/str_utils.py ========
def greet(name):
    return f"Hello, {name}!"

# ======== test.py ========
from my_utils import *

print(double(5))      # 10       —— ✅ 在 __all__ 中
print(greet("World")) # Hello!   —— ✅ 在 __all__ 中
# print(VERSION)      # NameError —— ❌ 不在 __all__ 中
# print(triple(5))    # NameError —— ❌ 不在 __all__ 中
```

**说明：** 即使 `VERSION` 定义在 `__init__.py` 中，只要不在 `__all__` 列表中，`import *` 就不会导入它。

### 练习 3

**题目：**

创建一个模拟插件系统：创建两个不同目录，都放在 `sys.path` 中，它们共享同一个顶级包名 `plugins`。验证 Python 可以把两个目录中的模块当作同一个包来访问。

**答案：**

```python
import sys

# 假设两个目录结构:
# /path/a/plugins/module_a.py   (有函数 func_a: return "A")
# /path/b/plugins/module_b.py   (有函数 func_b: return "B")

sys.path.append("/path/a")
sys.path.append("/path/b")

# 两个目录中的模块被当作同一个 plugins 包访问
from plugins import module_a
from plugins import module_b

print(module_a.func_a())  # "A"
print(module_b.func_b())  # "B"

# 注意：目录中不需要 __init__.py（Python 3.3+ 隐式命名空间包）
```

**说明：** 这是"隐式命名空间包"的用法，允许一个包的模块分散在不同目录中。

## 8. 答题总结

- 自定义模块就是你写的 `.py` 文件，导入方式和标准库完全一样。
- 包是一个包含 `__init__.py` 的目录，用于组织多个相关模块。
- `__init__.py` 有三大作用：标识包、定义 `__all__` 控制导出、执行初始化代码。
- 绝对导入从项目根目录写完整路径（推荐），相对导入用 `.` 和 `..`（仅包内使用）。
- 包内使用相对导入的模块不能用 `python 模块.py` 直接运行，要用 `python -m 包.模块`。
- 定义 `__all__` 可以控制 `from package import *` 的导出内容。
- 修改被导入的模块后，用 `importlib.reload()` 强制刷新，或重启解释器。
