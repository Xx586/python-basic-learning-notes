# 模块导入

## 1. 知识点定义

**模块（Module）** 就是单个 `.py` 文件。每一个以 `.py` 结尾的 Python 文件都是一个模块。模块里可以定义函数、类、变量，也可以包含可执行的代码。

> 小白理解：
> 模块就像一个"工具箱"。你写好各种工具函数放在工具箱里，以后哪个项目需要，直接把工具箱搬过来用就行。不用每次都重新打造工具。

它主要解决的问题是：
- 代码太多，一个文件写不下 → 按功能拆分到不同模块
- 多个程序要用同一段逻辑 → 写成模块，到处 `import`
- 避免变量名冲突 → 每个模块有独立的命名空间
- 方便测试和维护 → 模块级别的单元测试

模块分为三类：
1. **标准库模块**：Python 自带的，如 `os`、`sys`、`math`、`random`
2. **第三方模块**：通过 pip 安装的，如 `numpy`、`requests`
3. **自定义模块**：你自己写的 `.py` 文件

## 2. 语法格式

### 2.1 import 模块名

```python
# 导入整个模块，使用时需要加"模块名."前缀
import math

print(math.sqrt(16))   # 4.0 —— 必须写 math.
print(math.pi)         # 3.141592653589793
```

**优点**：命名清晰，不会和当前文件的变量冲突。
**缺点**：每次使用都要写模块名前缀。

### 2.2 from 模块名 import 具体内容

```python
# 只导入需要的部分，使用时不需要模块名前缀
from math import sqrt, pi

print(sqrt(25))   # 5.0 —— 直接使用
print(pi)         # 3.141592653589793
```

> **注意**：这种方式可能和当前文件的同名变量冲突！

```python
pi = 100              # 自己定义的 pi
from math import pi   # 覆盖了上面的 100！
print(pi)             # 3.141592653589793，你定义的 100 丢了
```

### 2.3 from 模块名 import *（不推荐！）

```python
from math import *    # 导入模块中的所有公开名称

print(sin(0))   # 0.0  —— 可以直接用所有名称
print(pi)       # 3.141592653589793
```

**为什么不推荐？**
1. **命名污染**：你不知道导入了什么，可能覆盖你的变量
2. **可读性差**：看代码时不知道某个函数来自哪个模块
3. **调试困难**：出问题时很难追踪来源

### 2.4 as 给模块起别名

```python
import numpy as np                  # 给模块起简短别名
import pandas as pd
import matplotlib.pyplot as plt

arr = np.array([1, 2, 3])          # 使用时更简洁
df = pd.DataFrame({'a': [1, 2, 3]})
```

**常用约定别名：**

| 模块 | 别名 | 用途 |
|------|------|------|
| `numpy` | `np` | 数值计算 |
| `pandas` | `pd` | 数据分析 |
| `matplotlib.pyplot` | `plt` | 绘图 |

### 2.5 `__name__ == '__main__'`

```python
# 文件: my_module.py

def add(a, b):
    return a + b

def subtract(a, b):
    return a - b

# 下面的代码只在直接运行本文件时执行
# 被其他文件 import 时不会执行
if __name__ == '__main__':
    print("正在直接运行 my_module.py")
    print(f"1 + 2 = {add(1, 2)}")
    print(f"5 - 3 = {subtract(5, 3)}")
```

**原理**：
- 直接运行时：`__name__` 的值是 `'__main__'`
- 被 `import` 时：`__name__` 的值是模块名（文件名不含 `.py`）

## 3. 重点说明

### 3.1 三种导入方式对比

| 导入方式 | 使用时写法 | 命名冲突风险 | 推荐度 |
|---------|-----------|:---:|:---:|
| `import math` | `math.sqrt(16)` | 低 | ⭐⭐⭐ |
| `from math import sqrt` | `sqrt(16)` | 中 | ⭐⭐ |
| `from math import *` | `sqrt(16)` | 高 | ❌ |

> **建议**：日常优先使用 `import 模块名`，命名清晰不冲突。个别用得非常频繁的函数可以考虑 `from ... import ...`。

### 3.2 模块搜索路径 sys.path

当你执行 `import xxx` 时，Python 按照 `sys.path` 列表中的路径顺序依次搜索：

```python
import sys
for path in sys.path:
    print(path)

# 典型输出（按优先级从高到低）:
# ''                            ← ① 当前目录（最高优先级）
# '/usr/lib/python3.12'         ← ② 标准库目录
# '/usr/lib/python3.12/site-packages'  ← ③ 第三方库目录
```

如果需要导入不在默认路径中的模块：

```python
import sys
sys.path.append('/Users/me/my_python_libs')  # 添加自定义路径
import my_module  # 现在可以找到了
```

### 3.3 模块缓存机制

Python 有一个 `sys.modules` 字典记录所有已导入的模块。重复 `import` 同一个模块时，不会再次执行它，直接从缓存取出。

```python
import my_module  # 第一次导入，模块代码被执行
import my_module  # 第二次导入，直接从缓存取，代码不执行

# 如果修改了模块代码，需要强制重新加载：
import importlib
importlib.reload(my_module)  # 强制重新加载
```

### 3.4 绝对导入 vs 相对导入

```python
# 绝对导入：从项目根目录出发，写完整路径
from utils.helper import format_data
from models.user import User

# 相对导入：用 . 和 .. 表示相对位置（只能在包内部使用）
from .helper import format_data     # . 表示当前包（同目录）
from ..utils import format_data     # .. 表示父包（上一级）
```

| 对比维度 | 绝对导入 | 相对导入 |
|---------|---------|---------|
| 可读性 | 一目了然 | 需要推算路径 |
| 适用场景 | 日常开发、顶层脚本 | 大型包的内部模块间 |
| 推荐度 | **优先推荐** | 包内部模块间使用 |

## 4. Demo 样例

```python
import math           # 导入整个 math 模块
import sys            # 导入 sys 模块查看搜索路径

# ========== 1. import 模块名（推荐方式） ==========
print("=" * 40)
print("1. import 模块名")

print(f"sqrt(16) = {math.sqrt(16)}")    # 必须加模块名前缀 math.
print(f"pi = {math.pi}")                # 圆周率常量
print(f"sin(90°) = {math.sin(math.pi / 2)}")  # sin(π/2) = 1.0

# ========== 2. from ... import ... ==========
print("\n" + "=" * 40)
print("2. from ... import ...")

from math import sqrt, pow              # 只导入需要的函数

print(f"sqrt(25) = {sqrt(25)}")         # 直接使用，不需要 math. 前缀
print(f"pow(2, 10) = {pow(2, 10)}")     # 2的10次方 = 1024.0

# ========== 3. as 起别名 ==========
print("\n" + "=" * 40)
print("3. as 起别名")

import math as m                        # 给模块起一个短别名
print(f"ceil(4.3) = {m.ceil(4.3)}")     # 向上取整 = 5
print(f"floor(4.7) = {m.floor(4.7)}")   # 向下取整 = 4

# ========== 4. __name__ 演示 ==========
print("\n" + "=" * 40)
print("4. __name__ 属性")

print(f"当前文件的 __name__ 是: {__name__}")  # 直接运行时输出 '__main__'
# 如果这个文件被 import，__name__ 会变成模块名

# ========== 5. sys.path 查看搜索路径 ==========
print("\n" + "=" * 40)
print("5. 模块搜索路径（前3条）")
for i, path in enumerate(sys.path[:3]):
    print(f"  [{i}] {path}")

print("\n模块导入演示完成！")
```

**运行结果：**

```
========================================
1. import 模块名
sqrt(16) = 4.0
pi = 3.141592653589793
sin(90°) = 1.0

========================================
2. from ... import ...
sqrt(25) = 5.0
pow(2, 10) = 1024.0

========================================
3. as 起别名
ceil(4.3) = 5
floor(4.7) = 4

========================================
4. __name__ 属性
当前文件的 __name__ 是: __main__

========================================
5. 模块搜索路径（前3条）
  [0] 
  [1] /usr/lib/python3.12
  [2] /usr/lib/python3.12/site-packages

模块导入演示完成！
```

## 5. 典型例题（附带答案）

### 例题 1

**题目：**

创建一个 `tools.py` 文件，定义常量 `VERSION = "1.0"`、函数 `double(x)` 返回 `x * 2`、函数 `greet(name)` 返回 `f"Hello, {name}"`。然后在另一个文件中分别用 `import tools` 和 `from tools import double, greet` 两种方式导入并调用。

**答案：**

```python
# ======== tools.py ========
VERSION = "1.0"

def double(x):
    """返回 x 的两倍"""
    return x * 2

def greet(name):
    """返回问候语"""
    return f"Hello, {name}"

# ======== main.py（方式一：import 模块名）========
import tools

print(tools.VERSION)           # 1.0
print(tools.double(5))         # 10
print(tools.greet("World"))    # Hello, World

# ======== main.py（方式二：from ... import）========
from tools import double, greet

print(double(5))               # 10 —— 直接使用
print(greet("World"))          # Hello, World
# print(VERSION)               # 报错！没有导入 VERSION
```

**解析：** `import tools` 导入整个模块，所有内容都可用但需要前缀；`from tools import double, greet` 只导入指定函数，直接使用但可能和当前文件变量冲突。

### 例题 2

**题目：**

写一个文件 `test_name.py`，打印 `__name__` 的值。分别直接运行它和在另一个文件中 `import` 它，观察输出的区别并解释原因。

**答案：**

```python
# ======== test_name.py ========
print(f"我的 __name__ 是: {__name__}")

if __name__ == '__main__':
    print("我是直接运行的")
else:
    print("我是被导入的")

# 直接运行 python test_name.py 输出：
# 我的 __name__ 是: __main__
# 我是直接运行的

# 在另一个文件中 import test_name 输出：
# 我的 __name__ 是: test_name
# 我是被导入的
```

**解析：** 直接运行时 `__name__` 为 `'__main__'`，被导入时为模块名（不含 `.py`）。这就是 `if __name__ == '__main__':` 能区分两种场景的原理。

## 6. 易错点

### 易错点 1：循环导入（Circular Import）

错误写法：
```python
# a.py
from b import func_b    # 导入 b → b 又导入 a → a 还没执行完，func_a 未定义！

# b.py
from a import func_a    # 此时 a.py 只执行了一半，func_a 还未定义
```

错误原因：两个模块互相导入，形成循环依赖，导致 `ImportError`。

正确写法：
```python
# 方案1：延迟导入（把 import 放到函数内部）
def func_a():
    from b import func_b   # 函数被调用时才导入，此时所有模块已加载完成
    func_b()

# 方案2：提取公共模块，让 a 和 b 都只依赖公共模块，不互相依赖
```

### 易错点 2：文件名与标准库重名

错误写法：
```python
# 创建了一个文件叫 random.py
import random    # 导入了你自己的 random.py，而不是标准库！
```

错误原因：当前目录优先级最高，Python 优先找到了你的同名文件。

正确写法：
```python
# 避免使用以下标准库名作为文件名：
# math, os, sys, json, random, re, time, datetime, collections, itertools 等
```

### 易错点 3：相对导入在脚本直接运行时报错

错误写法：
```bash
$ python my_package/sub.py
```

而 `sub.py` 中使用了：
```python
from . import helper   # 相对导入
```

错误原因：直接用 `python sub.py` 运行时，`__name__` 是 `'__main__'`，Python 不知道它属于哪个包，相对导入失效。

正确写法：
```bash
# 用 -m 方式以模块身份运行
python -m my_package.sub
```

### 易错点 4：from module import * 的意外覆盖

错误写法：
```python
from math import *
from cmath import *   # cmath 也有 sin，但处理复数

# 现在 sin 是谁的？后导入的覆盖了先导入的！
# 而且你不知道到底导入了哪些名字
```

错误原因：`*` 导入所有公开名称，多个模块间可能冲突。

正确写法：
```python
import math
import cmath
# 使用时明确写出 math.sin() 和 cmath.sin()，永远不会搞混
```

## 7. 课后练习（附带答案）

### 练习 1

**题目：**

创建一个 `my_utils.py` 文件，定义函数 `is_palindrome(s)` 判断回文字符串，以及函数 `count_words(s)` 统计单词数。在另一个文件中导入并测试。

**答案：**

```python
# ======== my_utils.py ========
def is_palindrome(s):
    """判断字符串是否是回文（正着读反着读一样）"""
    s = s.lower().replace(" ", "")    # 统一小写，去掉空格
    return s == s[::-1]               # 正序和反序比较

def count_words(s):
    """统计字符串中的单词数"""
    return len(s.split())

# ======== test.py ========
import my_utils

print(my_utils.is_palindrome("A man a plan a canal Panama"))  # True
print(my_utils.count_words("Hello world Python"))             # 3
```

**说明：** `[::-1]` 是字符串反转的切片写法。`split()` 默认按空白字符分割。

### 练习 2

**题目：**

创建两个文件 `a.py` 和 `b.py`，让它们互相 import 对方的内容，观察循环导入报错信息。然后用"延迟导入"的方式修复。

**答案：**

```python
# ======== a.py（修复后）========
def func_a():
    from b import func_b   # 延迟导入：函数调用时才 import
    func_b()

# ======== b.py（修复后）========
def func_b():
    from a import func_a   # 延迟导入
    func_a()
```

**说明：** 延迟导入把 `import` 从模块顶层移到函数内部，此时所有模块的顶层代码都已执行完毕，不再有循环依赖问题。

### 练习 3

**题目：**

在桌面创建一个 `my_lib` 文件夹，里面放一个 `hello.py`（包含 `say_hi()` 函数）。在另一个 Python 脚本中，通过修改 `sys.path` 来成功导入 `hello` 模块。

**答案：**

```python
import sys
sys.path.append('/Users/你的用户名/Desktop/my_lib')  # Mac 路径
# sys.path.append('C:\\Users\\你的用户名\\Desktop\\my_lib')  # Windows 路径

import hello
hello.say_hi()

# 验证路径已加入
print(f"当前搜索路径数: {len(sys.path)}")
print(f"最后一条: {sys.path[-1]}")
```

**说明：** `sys.path.append()` 在列表末尾添加路径，也可以用 `sys.path.insert(0, path)` 插到最前面（优先搜索）。

## 8. 答题总结

- 模块就是 `.py` 文件，`import` 就是"借用"别人写好的代码。
- `import 模块名` 最安全清晰；`from 模块 import 函数` 更简洁但有命名冲突风险；`from 模块 import *` 不推荐。
- `as` 可以给模块起别名，如 `import numpy as np`。
- `__name__ == '__main__'` 用于区分"直接运行"和"被导入"两种场景。
- Python 按 `sys.path` 中的顺序搜索模块，当前目录优先级最高。
- 避免循环导入（可延迟导入解决）、避免文件名与标准库重名。
- 相对导入（`.` 和 `..`）只能在包内部使用，顶层脚本用绝对导入。
