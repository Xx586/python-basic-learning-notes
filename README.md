## 4. 每章主要学习内容

### 1. 环境与基础
Python 安装、IDE 选择（PyCharm/VS Code）、pip 包管理、虚拟环境、代码注释与编码规范、print 输出、input 输入、变量初识、关键字、命名规则、常见报错初识。

### 2. 数据类型与运算符
基本数据类型（int/float/bool/str/NoneType）、字符串操作（切片/格式化/f-string/常用方法）、容器类型（列表/元组/字典/集合）、类型转换与类型判断、运算符（算术/比较/逻辑/赋值/成员/身份/位/三元）、表达式。

### 3. 流程控制
顺序结构、条件判断（if/elif/else/嵌套/三元表达式/match-case）、循环（for/while/range/break/continue/pass/循环嵌套）、推导式（列表/字典/集合推导式）、简单菜单程序。

### 4. 函数
函数定义与调用、参数类型（位置/默认/关键字/*args/**kwargs）、返回值与多值返回、作用域（局部/全局/global/nonlocal/LEGB）、lambda 匿名函数、递归函数、闭包、装饰器、类型注解、常用内置函数。

### 5. 面向对象编程
类和对象基础、方法类型（实例/类/静态/@property）、继承与多态（单继承/多继承/super/方法重写）、封装（单下划线/双下划线）、魔术方法、dataclass、抽象类、反射、面向对象综合案例。

### 6. 文件操作与异常处理
文件操作（open/read/write/文件模式/with 语句/编码）、路径操作（os.path/pathlib）、JSON 读写、CSV 读写、异常处理（try-except-else-finally/raise/自定义异常/常见异常类型）。

### 7. 模块与包
模块导入（import/from import/as）、自定义模块、包（__init__.py）、`__name__ == "__main__"`、常用标准库（os/sys/datetime/time/math/random/collections/itertools/functools）。

### 8. 正则表达式
正则表达式基础语法（元字符/量词/字符集/分组/锚点）、re 模块函数详解（match/search/findall/sub/split/compile）、进阶技巧（贪婪vs非贪婪/零宽断言/re.VERBOSE/常见验证案例）。

### 9. 迭代器与生成器
可迭代对象与迭代器（iter/next/for循环原理）、生成器（yield/生成器表达式/惰性计算）。

### 10. 进阶语法
上下文管理器（with 原理/__enter__/__exit__/contextlib）、深拷贝与浅拷贝（copy.copy/copy.deepcopy）、解包操作（星号解包/字典解包/函数参数解包）、多线程与多进程、协程基础（async/await/asyncio）、类型注解（typing）、简单项目组织结构。

## 6. Demo 样例的学习方法

每个知识点的 Demo 代码都可以直接运行。学习方法：

1. **先看理论**：阅读知识点文档的"知识点定义"和"语法格式"部分。
2. **再看 Demo**：理解 Demo 代码的每一行注释。
3. **手敲代码**：不要只复制粘贴，自己手敲一遍加深记忆。
4. **运行观察**：运行代码，观察输出结果是否和文档中一致。
5. **修改变量**：尝试修改变量值、条件、数据内容，观察结果变化。
6. **故意出错**：尝试制造文档中"易错点"提到的错误，观察报错信息。
7. **完成练习**：做完课后练习和典型例题后再进入下一个知识点。

每章的 `demo/` 文件夹中有独立的 `.py` Demo 文件，可以复制出来运行。

## 7. 如何运行 Python 代码

### 方法一：命令行运行

```bash
# 运行单个 Python 文件
python 文件名.py

# 如果电脑同时安装了 Python 2 和 Python 3
python3 文件名.py
```

### 方法二：在 IDE 中运行

- **PyCharm**：右键 `.py` 文件 → Run
- **VS Code**：打开 `.py` 文件 → 点击右上角 ▶ 按钮

### 方法三：交互式运行（适合测试小段代码）

```bash
# 在终端输入 python 或 python3 进入交互模式
python3
>>> print("Hello World")
Hello World
>>> exit()  # 退出
```

## 8. 零基础学习建议

1. **按顺序学，不要跳**：10 个章节由浅入深，跳章节会导致后面的内容看不懂。
2. **每天至少写 50 行代码**：编程是"动手"学科，光看不练等于没学。
3. **遇到报错不要慌**：看最后一行报错信息，找到行号，检查拼写和符号。
4. **手敲比复制更重要**：肌肉记忆比视觉记忆更持久。
5. **学完一章再学下一章**：确保每个章节的练习题都做完再前进。
6. **做好笔记**：每个知识点的"答题总结"部分就是最好的笔记素材。
7. **学习时保持英文输入法**：只在写字符串内容时切换到中文，避免中文标点报错。

## 9. 常见报错处理方式

初学者最容易遇到的 6 种报错：

| 错误类型 | 中文意思 | 常见原因 | 解决思路 |
|----------|----------|----------|----------|
| `SyntaxError` | 语法错误 | 少写括号/冒号、中文标点 | 检查符号是否英文半角 |
| `NameError` | 名称错误 | 变量名拼错、变量未定义 | 检查变量名是否正确拼写 |
| `TypeError` | 类型错误 | 不同类型做不兼容操作 | 检查数据类型，必要时转换 |
| `IndentationError` | 缩进错误 | Tab 和空格混用、忘缩进 | 统一用 4 个空格缩进 |
| `IndexError` | 索引错误 | 列表/字符串访问超出范围 | 检查索引是否在范围内 |
| `FileNotFoundError` | 文件未找到 | 文件路径错误 | 检查路径是否正确，文件是否存在 |

排查步骤：**看最后一行 → 看行号 → 检查拼写 → 检查符号 → 检查缩进**
