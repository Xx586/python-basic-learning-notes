# 常见报错初识

## 1. 知识点定义

**报错**（Error）是 Python 解释器在发现代码有问题时给出的**错误提示信息**。报错不是坏事——它是 Python 在帮你找问题，告诉你"这里写的不对，需要改"。

它主要解决的问题是：
- 告诉程序员代码哪里出了问题
- 提示问题可能的原因
- 帮助定位 bug 的位置

在实际写 Python 代码时，报错是无法避免的。即使是工作多年的程序员也会遇到报错。关键在于：**能看懂报错信息，并根据提示快速定位和修复问题**。

> 小白理解：
> 把 Python 解释器想象成一个非常较真的语文老师。你交了一篇作文（代码），老师在有问题的地方打了个红叉并写了批注（报错信息）。刚开始看不懂批注很正常，慢慢学会看"红叉在哪儿"、"批注说了啥"，就能自己改错了。

---

## 2. 语法格式

Python 报错信息的基本结构：

```text
Traceback (most recent call last):
  File "文件名.py", line 行号, in <module>
    出错的代码行
错误类型: 错误描述
```

说明：
- **Traceback**：追溯信息，告诉我们错误发生在哪里。
- **File**：出错的文件名。
- **line**：出错的行号，这是最重要的信息——直接去那一行找问题。
- **错误类型**：错误的类别名称。
- **错误描述**：对错误的文字说明。

## 3. 重点说明

### 3.1 初学者必须认识的 6 种常见错误

| 错误类型 | 中文意思 | 常见原因 | 举例 |
|----------|----------|----------|------|
| `SyntaxError` | 语法错误 | 少写括号、少写冒号、中文标点 | `print "hello"`（缺括号） |
| `NameError` | 名称错误 | 变量名拼错、变量未定义就使用 | `print(nam)` 写错了变量名 |
| `TypeError` | 类型错误 | 不同类型数据做不兼容的操作 | `"age" + 18`（字符串+整数） |
| `IndentationError` | 缩进错误 | Tab 和空格混用、缩进不统一 | if 下面忘了缩进 |
| `IndexError` | 索引错误 | 访问列表/字符串时超出范围 | `lst[10]` 但列表只有 5 个元素 |
| `ValueError` | 值错误 | 类型对但值不合适 | `int("abc")`（字母不能转整数） |

### 3.2 排查错误的步骤

遇到报错不要慌，按以下步骤来：

1. **看最后一行**：错误类型和描述是最直接的提示。
2. **看行号**：找到 `line 行号`，去对应行以及它的上一行检查。
3. **检查拼写**：变量名有没有写错？函数名有没有写错？
4. **检查符号**：括号、冒号、引号、逗号是不是英文半角？
5. **检查缩进**：该缩进的地方缩进了吗？Tab 和空格混用了吗？

### 3.3 了解即可：更多错误类型

| 错误类型 | 中文意思 | 触发场景 |
|----------|----------|----------|
| `KeyError` | 键错误 | 字典中访问不存在的键 |
| `AttributeError` | 属性错误 | 对象没有这个属性或方法 |
| `ZeroDivisionError` | 除零错误 | 除以 0 |
| `FileNotFoundError` | 文件未找到 | 打开不存在的文件 |
| `ImportError` | 导入错误 | 导入不存在的模块 |

## 4. Demo 样例

### Demo 1：认识 6 种常见报错

```python
# 每种报错都是独立的，注释/取消注释某段来看效果

# ===== 1. SyntaxError：语法错误 =====
# 错误：print 在 Python 3 中必须加括号
# print "hello"  # 取消注释会报 SyntaxError


# ===== 2. NameError：名称错误 =====
# 错误：变量 nam 拼错了，应该是 name
name = "张三"
# print(nam)  # NameError: name 'nam' is not defined


# ===== 3. TypeError：类型错误 =====
# 错误：字符串和整数不能直接拼接
# result = "年龄：" + 18  # TypeError: can only concatenate str (not "int") to str


# ===== 4. IndentationError：缩进错误 =====
# 错误：if 语句下面的代码没有缩进
score = 85
# if score >= 60:
# print("及格")  # IndentationError: expected an indented block


# ===== 5. IndexError：索引错误 =====
# 错误：列表只有 3 个元素，索引 0,1,2，没有索引 5
fruits = ["苹果", "香蕉", "橘子"]
# print(fruits[5])  # IndexError: list index out of range


# ===== 6. ValueError：值错误 =====
# 错误：字符串 "abc" 不能转换成整数
# num = int("abc")  # ValueError: invalid literal for int() with base 10: 'abc'


# 正常运行的部分
print("程序运行完毕！")
```

运行结果：

```text
程序运行完毕！
```

> 上面的代码不会报错，因为错误语句被注释掉了。你可以逐一取消注释来观察每种报错信息。

### Demo 2：真实报错信息解析

```python
# 这段代码有bug，运行后会报错
total = 0
numbers = [10, 20, 30]
for i in range(4):       # 这里 range(4) 会产生 0,1,2,3，但列表只有3个元素
    total += numbers[i]   # 当 i=3 时，numbers[3] 不存在，报 IndexError
print("总和：", total)
```

运行结果：

```text
Traceback (most recent call last):
  File "demo.py", line 4, in <module>
    total += numbers[i]
IndexError: list index out of range
```

错误解析：
- **最后一行**：`IndexError` — 索引错误；`list index out of range` — 列表索引超出范围。
- **行号**：line 4 — 去第 4 行检查。
- **原因**：`range(4)` 产生 0,1,2,3，但列表 `numbers` 只有 3 个元素（索引 0,1,2），访问 `numbers[3]` 时超出范围。
- **修复**：把 `range(4)` 改成 `range(len(numbers))` 或 `range(3)`。

## 5. 典型例题（附带答案）

### 例题 1

题目：

下面的代码运行时会报什么错误？为什么？如何修复？

```python
age = input("请输入年龄：")
next_year = age + 1
print("明年你", next_year, "岁")
```

答案：

报错类型：`TypeError: can only concatenate str (not "int") to str`

原因：`input` 返回的是字符串，不能直接和整数 `1` 做加法。

修复：

```python
age = int(input("请输入年龄："))  # 输入后立刻转成整数
next_year = age + 1
print("明年你", next_year, "岁")
```

### 例题 2

题目：

运行以下代码，报错信息如下，请找出问题并修复。

```text
Traceback (most recent call last):
  File "test.py", line 2, in <module>
    print(studnet_name)
NameError: name 'studnet_name' is not defined
```

代码：

```python
student_name = "小明"
print(studnet_name)
```

答案：

错误在第 2 行：`studnet_name` 拼写错误，少写了 `e`，`student` 写成了 `studnet`。

修复：

```python
student_name = "小明"
print(student_name)  # 改成正确的拼写
```

## 6. 易错点

### 易错点 1：看到报错就慌，不看报错信息

初学者常犯的心态错误：一看到满屏红字就关掉，不敢看。

正确做法：**专门看最后一行和行号**，这两个信息能直接告诉你哪里错了。

### 易错点 2：中文标点导致的 SyntaxError

错误写法：

```python
print（"你好"）     # 括号是中文全角括号
print("你好"，"世界") # 逗号是中文逗号
```

错误原因：Python 只认识英文半角标点。使用中文输入法时常不小心带出中文符号。

正确写法：

```python
print("你好")        # 英文括号
print("你好", "世界")  # 英文逗号
```

> 小技巧：写代码时保持输入法是英文状态。只在写字符串内容（引号内的文字）时切换到中文。

### 易错点 3：少写冒号

错误写法：

```python
if score >= 60        # SyntaxError: 少了冒号
    print("及格")
```

错误原因：`if`、`elif`、`else`、`for`、`while`、`def`、`class` 后面都必须跟冒号 `:`。

正确写法：

```python
if score >= 60:       # 加冒号
    print("及格")
```

### 易错点 4：缩进错误

错误写法：

```python
if score >= 60:
print("及格")         # IndentationError: if 下面的代码没有缩进
```

错误原因：Python 用缩进表示代码块。`if` 后面的代码必须缩进（通常 4 个空格）。

正确写法：

```python
if score >= 60:
    print("及格")     # 前面有 4 个空格的缩进
```

## 7. 课后练习（附带答案）

### 练习 1

题目：

指出以下每段代码的错误类型和修复方法：

```python
# 代码 A
print("hello"

# 代码 B
x = 10
print(X)

# 代码 C
name = "小明"
print("你好，" + name + "，你今年" + 18 + "岁")
```

答案：

**代码 A**：`SyntaxError`——少写了右括号。修复：`print("hello")`。

**代码 B**：`NameError`——变量名大小写不对，定义时是小写 `x`，使用时是大写 `X`。修复：改成 `print(x)`。

**代码 C**：`TypeError`——字符串不能和整数 `18` 直接拼接。修复：`print("你好，" + name + "，你今年" + str(18) + "岁")` 或使用逗号分隔 `print("你好，", name, "，你今年", 18, "岁")`。

### 练习 2

题目：

以下代码运行时报错如下，请找出问题。

```text
Traceback (most recent call last):
  File "calc.py", line 3, in <module>
    result = a + b
NameError: name 'a' is not defined
```

代码：

```python
num1 = 10
num2 = 20
result = a + b
print(result)
```

答案：

第 3 行使用了变量 `a` 和 `b`，但前两行定义的变量名是 `num1` 和 `num2`。变量名不一致。

修复：

```python
num1 = 10
num2 = 20
result = num1 + num2  # 使用正确的变量名
print(result)
```

## 8. 答题总结

- 报错是 Python 在帮你找问题，不是惩罚你。要学会看懂报错信息。
- 报错看三个关键信息：**错误类型**（最后一行）、**文件名**、**行号**。
- 初学者最常见的错误：`SyntaxError`（语法错误）、`NameError`（名称错误）、`TypeError`（类型错误）、`IndentationError`（缩进错误）。
- 排查顺序：看错误类型 → 找对应行号 → 检查拼写/符号/缩进。
- 中文标点（括号、逗号、引号）是 SyntaxError 的头号元凶，写代码时保持英文输入法。
