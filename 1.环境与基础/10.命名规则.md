# 命名规则

## 1. 知识点定义

**命名规则**是 Python 中为变量、函数、类等起名字时需要遵守的规范和约定。规则分为两类：**硬性规则**（违反会报错）和**推荐规范**（违反不报错但会被认为写得不好）。

它主要解决的问题是：
- 确保名字能被 Python 解释器正确识别（硬性规则）
- 让代码清晰易读，自己和别人都能看懂（推荐规范）
- 统一团队协作时的命名风格

在实际写 Python 代码时，从第一个变量名开始就要养成良好的命名习惯。好的命名让代码像说话一样自然，坏的命名让代码像天书。

> 小白理解：
> 给你的孩子起名字，不能叫"1号小孩"（数字不能开头），也不能叫"如果"（关键字）。你应该起个好听、有意义的名字，比如"小明"。写代码也是一样，变量要起一个"见名知意"的好名字。

---

## 2. 语法格式

### 硬性规则（必须遵守，违反会报错）

```python
# ✅ 正确的命名
name = "张三"           # 纯小写字母
age = 18               # 纯小写字母
my_score = 95          # 字母 + 下划线
_user = "hidden"       # 以下划线开头（通常表示私有）
student2 = "李四"       # 字母 + 数字
MAX_SIZE = 100         # 全大写（通常表示常量）

# ❌ 错误的命名
2name = "张三"          # SyntaxError: 不能以数字开头
my-name = "张三"        # SyntaxError: 不能包含短横线 -
my name = "张三"        # SyntaxError: 不能包含空格
class = "一班"          # SyntaxError: class 是关键字
if = 10                # SyntaxError: if 是关键字
```

### 硬性规则总结

| 规则 | 示例（正确） | 示例（错误） |
|------|-------------|-------------|
| 只能使用字母、数字、下划线 | `my_var` | `my-var`（有短横线） |
| 不能以数字开头 | `var1` | `1var` |
| 不能是 Python 关键字 | `my_if` | `if` |
| 区分大小写 | `name` ≠ `Name` ≠ `NAME` | — |
| 不能包含空格 | `user_name` | `user name` |

## 3. 重点说明

### 3.1 推荐的命名风格（PEP 8 规范）

PEP 8 是 Python 官方的代码风格指南，以下是常用的命名规范：

| 命名对象 | 推荐风格 | 示例 |
|----------|----------|------|
| **变量名** | 小写字母 + 下划线（snake_case） | `student_name`, `total_score`, `user_id` |
| **函数名** | 小写字母 + 下划线（snake_case） | `get_name()`, `calculate_total()` |
| **类名** | 每个单词首字母大写（PascalCase） | `Student`, `MyClass`, `DataProcessor` |
| **常量** | 全大写 + 下划线 | `PI`, `MAX_SIZE`, `DEFAULT_TIMEOUT` |
| **私有变量/方法** | 单下划线开头 | `_internal_data`, `_private_method()` |
| **特殊方法** | 双下划线开头和结尾 | `__init__`, `__str__`, `__len__` |

### 3.2 命名的最高原则：见名知意

变量名应该告诉读代码的人"这个变量是干什么用的"：

```python
# ❌ 不好的命名：看不出含义
a = 18
b = "张三"
c = [85, 90, 78]
x = a * 2

# ✅ 好的命名：一看就知道是什么
age = 18
name = "张三"
scores = [85, 90, 78]
double_age = age * 2
```

### 3.3 常用命名缩写约定

在代码中有些缩写是大家约定俗成的，看到就能理解：

| 缩写 | 全称 | 常见用途 |
|------|------|----------|
| `i`, `j`, `k` | index | 循环计数器 |
| `n` | number | 数量 |
| `s` | string | 字符串 |
| `f` | file | 文件对象 |
| `fn` / `func` | function | 函数 |
| `obj` | object | 对象 |
| `lst` / `arr` | list / array | 列表 |
| `idx` | index | 索引/下标 |

### 3.4 中文命名的注意事项

Python 3 支持中文变量名，但**强烈不建议使用**：

```python
年龄 = 18         # 能运行，但不推荐
print(年龄)       # 英文环境中可能乱码或不便编辑
```

理由：不符合国际编码习惯、某些环境可能不兼容、团队协作时外国人看不懂。

## 4. Demo 样例

### Demo 1：好命名 vs 坏命名

```python
# ========== 坏命名示例 ==========
# 以下代码能运行，但完全不知道在做什么
a = 100
b = 20
c = 7
d = a * b * (1 - c / 100)
print(d)  # 输出什么？很难猜

# ========== 好命名示例 ==========
# 以下代码意思一目了然：计算打折后的价格
original_price = 100        # 商品原价
discount_rate = 20          # 折扣率（20 表示打 8 折）
tax_rate = 7                # 税率 7%
# 最终价格 = 原价 × 折扣 × 含税系数
final_price = original_price * (1 - discount_rate / 100) * (1 + tax_rate / 100)
print("最终价格：", final_price)
```

运行结果：

```text
7440.0
最终价格： 85.6
```

> 对比：同样的计算逻辑，好命名的代码不需要注释就能读懂，坏命名的代码连作者自己一星期后回来都看不懂。

### Demo 2：不同命名风格示例

```python
# 变量名：snake_case（全小写 + 下划线）
user_name = "张三"
student_count = 30
max_retry_times = 3

# 常量：UPPER_CASE（全大写 + 下划线）
PI = 3.14159
MAX_CONNECTIONS = 100
DEFAULT_LANGUAGE = "zh_CN"

# 函数名：snake_case
def calculate_average(numbers):
    """计算平均值"""
    return sum(numbers) / len(numbers)

# 类名：PascalCase（每个单词首字母大写）
class Student:
    def __init__(self, name, age):
        self.name = name
        self.age = age

# 测试代码
scores = [85, 92, 78, 90]
avg = calculate_average(scores)
print("平均分：", avg)

stu = Student("小明", 20)
print(stu.name, "同学，", stu.age, "岁")
```

运行结果：

```text
平均分： 86.25
小明 同学， 20 岁
```

## 5. 典型例题（附带答案）

### 例题 1

题目：

以下变量名哪些符合 Python 命名规则？哪些不符合？说明原因。

```text
user_name    _count    3stars    my-var    total$    class
```

答案：

| 变量名 | 是否合法 | 原因 |
|--------|----------|------|
| `user_name` | ✅ | 字母+下划线，符合规则 |
| `_count` | ✅ | 下划线开头是允许的 |
| `3stars` | ❌ | 数字不能作为开头 |
| `my-var` | ❌ | 短横线 `-` 不是合法字符 |
| `total$` | ❌ | `$` 不是合法字符 |
| `class` | ❌ | `class` 是 Python 关键字 |

### 例题 2

题目：

请将以下不符合规范的变量名改写成符合 PEP 8 风格的名字：

```text
StudentName (用于保存学生姓名)
MYAGE (用于保存我的年龄)
x (用于保存购物车商品总价)
```

答案：

```python
# StudentName → student_name （变量名用小写+下划线，PascalCase留给类名）
student_name = "张三"

# MYAGE → my_age （全大写通常只用于常量，普通变量用小写+下划线）
my_age = 20

# x → total_price （一个字母的名字看不出含义，改成描述性的名字）
total_price = 258.50
```

## 6. 易错点

### 易错点 1：变量名包含特殊字符

错误写法：

```python
my-name = "张三"    # SyntaxError: 短横线不能用于变量名
user@name = "张三"  # SyntaxError: @ 不能用于变量名
total$ = 100       # SyntaxError: $ 不能用于变量名
```

错误原因：Python 变量名只能包含字母、数字和下划线三种字符。

正确写法：

```python
my_name = "张三"    # 用下划线代替短横线
user_name = "张三"  # 用下划线代替特殊字符
total_money = 100   # 用有意义的英文单词
```

### 易错点 2：用中文拼音随意命名

错误写法：

```python
shuliang = 100      # 拼音命名，别人很难猜到是"数量"
jiage = 50.5        # 拼音命名，应该是"价格"
```

错误原因：拼音在同音字多的情况下很难理解，团队协作时尤其不便。

正确写法：

```python
quantity = 100      # 英文命名，国际通用
price = 50.5        # 一看就懂
```

### 易错点 3：变量名太短没有意义

错误写法：

```python
a1 = 80
a2 = 90
a3 = 75
a4 = (a1 + a2 + a3) / 3
```

错误原因：`a1`、`a2`、`a3` 看不出是成绩还是年龄还是数量。

正确写法：

```python
math_score = 80
english_score = 90
chinese_score = 75
average_score = (math_score + english_score + chinese_score) / 3
```

## 8. 答题总结

- Python 命名硬性规则：只能包含字母、数字、下划线，不能以数字开头，不能是关键字。
- PEP 8 推荐规范：变量和函数用 `snake_case`（小写+下划线），类名用 `PascalCase`（首字母大写），常量用 `UPPER_CASE`（全大写+下划线）。
- 命名最高原则：**见名知意**，别人看到名字就能猜到用途。
- 不要用拼音、不要用单个字母（除了循环计数 i/j）、不要用中文命名。
- 好的命名比注释更重要——代码本身就是最好的文档。
