# Python 安装与环境配置

## 1. 知识点定义

Python 是一种解释型、面向对象的高级编程语言，由 Guido van Rossum 于 1991 年首次发布。Python 以简洁易读的语法著称，广泛应用于 Web 开发、数据科学、人工智能、自动化运维等领域。

目前主流的 Python 版本是 **Python 3.x**（截至 2026 年，推荐使用 Python 3.11+）。**Python 2.x 已于 2020 年 1 月 1 日正式停止维护，请不要学习或使用 Python 2。**

正确安装和配置 Python 环境是学习的第一步：

- **运行代码的基础**：没有 Python 解释器，你的 `.py` 文件只是一堆文本。
- **PATH 环境变量**：配置 PATH 后，你才能在终端中直接输入 `python` 运行解释器。
- **系统级 vs 用户级**：不要把系统自带的 Python 和你开发用的 Python 混在一起。

> 小白理解：
> 安装 Python 就像给电脑装了一个"翻译官"。你写的 Python 代码是"人话"，Python 解释器负责把它翻译成电脑能懂的"机器话"。没有这个翻译官，你写的代码就只是一篇英文作文，电脑什么也做不了。

## 2. 语法格式

### 2.1 Windows 安装

1. 访问 Python 官网：https://www.python.org/downloads/
2. 点击黄色按钮 "Download Python 3.x.x"，下载安装包。
3. **关键步骤**：双击运行安装程序，**务必勾选页面底部的 "Add Python to PATH"**（这个复选框默认是未勾选的！）。
4. 选择 "Install Now"（默认安装）或 "Customize installation"（自定义安装路径）。

验证安装：

```powershell
python --version
# 输出示例：Python 3.11.5
```

### 2.2 macOS 安装

**方式一：官方安装包**
访问 https://www.python.org/downloads/ 下载 macOS 安装包（`.pkg` 文件），按向导完成安装。

**方式二：Homebrew（推荐）**

```bash
# 首先安装 Homebrew（如果尚未安装）
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

# 用 Homebrew 安装 Python
brew install python@3.12
```

验证安装：

```bash
python3 --version
# 输出示例：Python 3.12.3
```

> **注意**：macOS 上，系统自带的 `python` 命令指向 Python 2。你安装的 Python 3 使用 `python3` 命令。

### 2.3 Linux 安装

大多数 Linux 发行版已预装 Python 3：

```bash
# Ubuntu / Debian
sudo apt update
sudo apt install python3 python3-pip python3-venv

# CentOS / Fedora
sudo dnf install python3 python3-pip

# Arch Linux
sudo pacman -S python python-pip

# 验证安装
python3 --version
```

### 2.4 环境变量 PATH 配置

**PATH** 是操作系统用来查找可执行文件的目录列表。

**Windows：**
1. 右键"此电脑" → "属性" → "高级系统设置" → "环境变量"。
2. 在"系统变量"中找到 `Path`，双击编辑。
3. 添加 Python 安装目录（如 `C:\Python311\`）和 Scripts 目录（如 `C:\Python311\Scripts\`）。

**macOS / Linux：**
编辑 Shell 配置文件（`~/.zshrc` 或 `~/.bashrc`）：

```bash
export PATH="/usr/local/opt/python@3.12/libexec/bin:$PATH"
source ~/.zshrc
```

### 2.5 pip 是什么？

pip 是 Python 的包管理工具，安装 Python 3.4+ 时会自动附带：

```bash
pip --version
# 或
pip3 --version
```

## 3. 重点说明

### 3.1 版本选择

| 版本 | 状态 | 建议 |
|------|------|------|
| Python 2.x | 2020年已停止维护 | ❌ 不要学 |
| Python 3.8-3.10 | 稳定版本 | ✅ 可用 |
| Python 3.11+ | 最新稳定版 | ✅ 推荐（性能提升明显） |

### 3.2 安装位置建议

| 系统 | 推荐安装路径 | 原因 |
|------|-------------|------|
| Windows | `C:\Python311\` | 避免中文路径和空格 |
| macOS | `/usr/local/bin/python3` | Homebrew 默认位置 |
| Linux | `/usr/local/bin/python3` | 不覆盖系统 Python |

### 3.3 pip 自动附带

Python 3.4+ 会自动附带 pip，可以验证：

```bash
pip3 --version
```

如果提示找不到 pip，使用：

```bash
python3 -m pip --version
```

## 4. Demo 样例

```python
# -*- coding: utf-8 -*-
"""
验证 Python 环境是否安装成功
运行方式：在终端中执行 python 验证环境.py
"""

import sys
import platform


def check_python_version():
    """检查 Python 版本信息"""
    print("=" * 50)
    print(f"Python 版本: {sys.version}")
    print(f"Python 版本号: {sys.version_info.major}.{sys.version_info.minor}.{sys.version_info.micro}")
    print(f"操作系统: {platform.system()} {platform.release()}")
    print(f"架构: {platform.architecture()[0]}")
    print("=" * 50)

    # 检查 Python 版本是否满足最低要求
    if sys.version_info < (3, 8):
        print("[警告] 建议使用 Python 3.8 或更高版本!")
    else:
        print("[通过] Python 版本符合要求 (>= 3.8)")


def check_pip():
    """尝试检查 pip 是否可用"""
    try:
        import pip
        print(f"[通过] pip 已安装，版本: {pip.__version__}")
    except ImportError:
        print("[警告] 无法导入 pip 模块，请检查 pip 是否安装")


def check_path():
    """显示 Python 的安装路径"""
    print(f"\nPython 可执行文件路径: {sys.executable}")
    print(f"Python 搜索路径: ")
    for i, p in enumerate(sys.path, 1):
        print(f"  {i}. {p}")


if __name__ == "__main__":
    check_python_version()
    check_pip()
    check_path()
    print("\n环境验证完成！")
```

## 5. 典型例题（附带答案）

### 例题 1

题目：

在终端中分别运行 `python --version` 和 `python3 --version`，观察两个命令的输出是否一致。如果不一致，说明原因是什么？

答案：

```bash
# Windows（只安装了Python 3.11）：
$ python --version
Python 3.11.5

# macOS（系统自带 Python 2，Homebrew 安装了 Python 3）：
$ python --version
Python 2.7.18    # 系统自带的旧版本
$ python3 --version
Python 3.12.3    # 新安装的版本
```

解析：
macOS 上系统自带的 `python` 指向 Python 2，需要明确使用 `python3`。可以通过创建别名（`alias python=python3`）来统一。

答案：

```python
# my_first.py
print("你好，我是[你的名字]！这是我在 Python 世界的第一行代码。")
```


## 6. 易错点

### 易错点 1：Windows 忘记勾选 "Add Python to PATH"

错误现象：安装完成后，命令行输入 `python` 显示"不是内部或外部命令"。

错误原因：安装时未将 Python 添加到 PATH 环境变量。

正确做法：
1. 重新运行安装程序，选择 "Modify"（修改），确保勾选 "Add Python to PATH"。
2. 或者手动添加 PATH（见"环境变量 PATH 配置"）。

### 易错点 2：macOS 上 `python` 和 `python3` 混淆

错误现象：输入 `python` 打开的是 Python 2.7，而不是 Python 3。

错误原因：macOS 系统自带 Python 2.7，`python` 命令默认指向它。

正确做法：始终使用 `python3` 和 `pip3` 命令。或在 `.zshrc` 中添加别名：

```bash
alias python=python3
alias pip=pip3
```

### 易错点 3：多个 Python 版本共存导致混乱

错误现象：系统里既有 Python 3.9、Python 3.10，又有 Anaconda 的 Python，搞不清楚 `python` 指向哪个。

错误原因：多次安装不同版本，PATH 中出现了多个 Python 路径。

正确做法：
- 使用 `which python3` 查看当前使用的 Python 路径。
- 使用虚拟环境隔离项目。
- 使用 `pyenv` 管理多个 Python 版本。

### 易错点 4：Linux 卸载系统 Python 导致系统崩溃

错误现象：卸载系统 Python 后，桌面环境或系统工具无法运行。

错误原因：某些 Linux 发行版的系统工具依赖 Python。卸载系统 Python 导致依赖断裂。

正确做法：**永远不要卸载系统自带的 Python！**安装新版本到 `/usr/local/` 或使用 `pyenv`。

### 易错点 5：安装路径包含中文或空格

错误现象：Windows 用户名是中文（如 `C:\Users\张三\`），导致某些库运行时找不到路径或乱码报错。

错误原因：部分 Python 库对非 ASCII 路径支持不完善。

正确做法：将 Python 安装在无中文、无空格的路径（如 `C:\Python311\`）。

## 7. 课后练习（附带答案）

### 练习 1：查看 Python 安装路径

题目：

在终端中使用 `which`（macOS/Linux）或 `where`（Windows）命令，找出当前 Python 解释器的完整路径。

答案：

```bash
# macOS / Linux
$ which python3
/usr/local/bin/python3

# 可以进一步查看这个文件是不是符号链接
$ ls -l /usr/local/bin/python3
lrwxr-xr-x  1 user  admin  40 Sep 11 08:19 /usr/local/bin/python3 -> ../Cellar/python@3.11/3.11.5/bin/python3

# Windows
> where python
C:\Python311\python.exe
```

### 练习 2：PATH 环境变量练习

题目：

查看你当前系统的 PATH 环境变量，确认 Python 的安装目录是否在其中。

答案：

```bash
# macOS / Linux
$ echo $PATH
/usr/local/bin:/usr/bin:/bin:...

# Windows (命令提示符)
> echo %PATH%
C:\Python311\;C:\Python311\Scripts\;...

# Windows (PowerShell)
> $env:PATH -split ';'
```

## 8. 答题总结

- Python 是解释型语言，需要先安装解释器（推荐 3.11+），不要使用已停止维护的 Python 2。
- Windows 安装时**务必勾选 "Add Python to PATH"**，这是新手最常见的坑。
- macOS 自带 Python 2，要用 `python3` 命令；可以通过别名统一。
- **永远不要卸载系统自带的 Python**（尤其是 Linux），安装新版本到独立目录。
- 使用 `python --version` 和 `pip --version` 验证安装是否成功。
- 安装路径建议避免中文和空格，减少潜在兼容问题。
