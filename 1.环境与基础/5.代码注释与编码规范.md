# 代码注释与编码规范

## 1. 知识点定义

注释是写给**人**看的文字，Python 解释器会忽略它们。好的注释回答的是"**为什么这样做**"，而不是"做了什么"（代码本身已经说明了做了什么）。

PEP 8（Python Enhancement Proposal 8）是 Python 官方代码风格指南。遵循它能让你的代码和社区保持一致，提高可读性。

注释的作用：
- **帮助理解**：解释复杂的业务逻辑或算法思路。
- **便于维护**：半年后回来看自己的代码，注释能帮你快速回忆。
- **团队协作**：让同事能理解你的代码意图。
- **生成文档**：文档字符串（docstring）可以被工具自动提取生成 API 文档。

> 小白理解：
> 注释就像给未来的自己（或同事）留的便签条。代码告诉电脑"做什么"，注释告诉人"为什么要这么做"。半年后你回来看自己写的代码，如果没有注释，就像看天书。

## 2. 语法格式

### 2.1 单行注释

用 `#` 开头，`#` 后面通常加一个空格。

```python
# 这是单行注释，解释下面这行代码的作用
name = "张三"

age = 25  # 行内注释：放在代码右侧，解释这行代码
```

常用标记：

```python
# TODO: 这里之后需要加上缓存机制
# FIXME: 这个方法在输入为空时会崩溃
# HACK: 临时方案，等后端接口修复后替换
```

### 2.2 多行注释（文档字符串）

使用**三个引号**（`"""` 或 `'''`）的字符串作为多行注释：

```python
"""
这是多行注释（其实是多行字符串）
可以写很多行内容
Python 解释器不会执行它
"""
```

#### 文档字符串（docstring）—— Google 风格

```python
def calculate_area(width, height):
    """
    计算矩形的面积。

    Args:
        width (float): 矩形的宽度
        height (float): 矩形的高度

    Returns:
        float: 矩形面积（宽度 x 高度）

    Raises:
        ValueError: 当宽度或高度为负数时抛出
    """
    if width < 0 or height < 0:
        raise ValueError("宽度和高度不能为负数")
    return width * height
```

### 2.3 PEP 8 命名规范

| 类型 | 命名风格 | 示例 |
|------|---------|------|
| 变量名 | `snake_case`（全小写+下划线） | `user_name`, `total_count` |
| 函数名 | `snake_case` | `get_user_info()`, `calculate_total()` |
| 类名 | `PascalCase`（每个单词首字母大写） | `StudentManager`, `HttpClient` |
| 常量 | `UPPER_SNAKE_CASE`（全大写+下划线） | `MAX_SIZE`, `DEFAULT_TIMEOUT` |
| 模块/文件名 | `snake_case` | `data_processor.py` |
| 私有属性/方法 | 单下划线前缀 | `_internal_method()`, `_cache` |

### 2.4 其他关键规则

```python
# 1. 缩进：用 4 个空格，不要用 Tab
def greet(name):
    if name:
        print(f"Hello, {name}!")

# 2. 运算符两侧各加一个空格
result = a + b * c      # 正确
result=a+b*c            # 错误

# 3. 逗号后面加空格
names = ["张三", "李四", "王五"]    # 正确

# 4. 函数参数中的等号不加空格
def func(param=1):      # 正确

# 5. 导入顺序：标准库 → 第三方库 → 本地模块
import os
import sys

import requests

from my_project.utils import helper
```

## 3. 重点说明

### 3.1 注释的黄金法则

```python
# 不好的注释（重复代码）
result = a + b  # 把 a 和 b 相加

# 好的注释（解释为什么）
result = a + b + 10  # 加上缓冲值 10 是为了防止浮点精度问题

# 更好的做法：用变量名代替注释
buffer_for_float_precision = 10
result = a + b + buffer_for_float_precision
```

### 3.2 什么时候需要注释

| 需要注释 | 不需要注释 |
|----------|-----------|
| 复杂的业务逻辑 | `x = 1` 这种不言自明的代码 |
| 非显而易见的算法选择 | 能用清晰变量名/函数名替代的场景 |
| TODO/FIXME 标记 | 代码本身已经充分说明了的操作 |
| 公开 API 的文档字符串 | 画蛇添足的废话注释 |

### 3.3 缩进：永远用 4 个空格

```python
# 正确：4 个空格缩进
def greet(name):
    if name:
        print(f"Hello, {name}!")

# 错误：不要用 Tab，不要混用 Tab 和空格
# 错误：不要用 2 个空格
```

> **规则**：永远不要混用 Tab 和空格。在 IDE 中设置"按 Tab 键时插入 4 个空格"。

## 4. Demo 样例

```python
# -*- coding: utf-8 -*-
"""
代码注释与编码规范演示脚本
全面展示 Python 的注释写法、命名规范和 PEP 8 核心规则
"""

# === 1. 单行注释 ===
user_name = "张三"            # 行内注释：存储用户名
user_age = 25                 # 行内注释：存储年龄
print(f"用户名: {user_name}, 年龄: {user_age}")


# === 2. 文档字符串（docstring） ===
class Student:
    """
    学生类 —— 演示 docstring 的用法。

    Attributes:
        name (str): 学生姓名
        age (int): 学生年龄
        grade (str): 年级
    """

    def __init__(self, name, age, grade):
        """初始化学生对象。

        Args:
            name (str): 学生姓名
            age (int): 学生年龄
            grade (str): 年级
        """
        self.name = name
        self.age = age
        self.grade = grade

    def get_info(self):
        """
        获取学生信息的格式化字符串。

        Returns:
            str: 包含姓名、年龄、年级的描述字符串
        """
        return f"姓名: {self.name}, 年龄: {self.age}, 年级: {self.grade}"


# 创建学生对象并展示
student = Student("李四", 20, "大二")
print(f"\n文档字符串演示: {student.get_info()}")


# === 3. PEP 8 命名规范 ===
# 常量：UPPER_SNAKE_CASE
MAX_STUDENTS = 100
PI = 3.14159
print(f"常量示例: MAX_STUDENTS = {MAX_STUDENTS}")
print(f"常量示例: PI = {PI}")


# snake_case 函数名
def calculate_average_score(scores):
    """
    计算平均分 —— 演示 snake_case 函数命名。

    Args:
        scores (list): 分数列表

    Returns:
        float: 平均分数
    """
    if not scores:
        return 0.0
    return sum(scores) / len(scores)


# snake_case 变量名
student_scores = [85, 90, 78, 92, 88]
average_score = calculate_average_score(student_scores)
print(f"考试成绩: {student_scores}")
print(f"平均分: {average_score}")


# === 4. 导入规范 ===
# 标准库（按字母顺序）
import math
import os
import sys

print(f"标准库导入成功: math.sqrt(16) = {math.sqrt(16)}")
print(f"当前工作目录: {os.getcwd()}")
print(f"Python 版本: {sys.version_info.major}.{sys.version_info.minor}")


if __name__ == "__main__":
    print("\n" + "=" * 50)
    print("演示完成！")
    print("=" * 50)
    print("\n提示: 用 help(函数名) 查看任意函数的 docstring，例如:")
    print("  help(calculate_average_score)")
```

## 5. 典型例题（附带答案）

### 例题 1

题目：

下面的代码违反了 PEP 8 命名规范，请指出所有问题并给出修正后的代码。

```python
MaxValue = 100
def GetUserName(UserID):
    user_name_list = []
    return user_name_list
class user_manager:
    def CreateUser(self):
        pass
```

答案：

```python
# 问题1：MaxValue → 常量应全大写
MAX_VALUE = 100

# 问题2：GetUserName 和 UserID → 函数和参数应用 snake_case
def get_user_name(user_id):
    user_name_list = []
    return user_name_list

# 问题3：user_manager → 类名应用 PascalCase
class UserManager:
    # 问题4：CreateUser → 方法名应用 snake_case
    def create_user(self):
        pass
```

### 例题 2

题目：

为下面这个函数编写符合 Google 风格的文档字符串（docstring），包含功能描述、Args、Returns 和 Raises。

```python
def safe_divide(a, b):
    if b == 0:
        raise ValueError("除数不能为零")
    return a / b
```

答案：

```python
def safe_divide(a, b):
    """
    安全地执行除法运算，避免除零错误。

    当除数为零时抛出异常而不是让程序崩溃。

    Args:
        a (float or int): 被除数（分子）
        b (float or int): 除数（分母），不能为零

    Returns:
        float: a 除以 b 的结果

    Raises:
        ValueError: 当 b 等于 0 时抛出
    """
    if b == 0:
        raise ValueError("除数不能为零")
    return a / b
```

## 6. 易错点

### 易错点 1：注释和代码不同步

错误现象：注释说"返回用户总数"，但代码实际返回的是"活跃用户数"。

错误原因：修改了代码逻辑，但忘记同步更新注释。

正确做法：
- 养成修改代码时同步修改注释的习惯。
- **更好的做法**：写"干净"的代码，让代码自解释。优先用清晰的变量名和函数名。

### 易错点 2：Tab 和空格混用

错误现象：代码在自己电脑上运行正常，发给别人或放到服务器上后就报 `IndentationError: unexpected indent`。

错误原因：混用了 Tab 和空格。Python 3 不允许混用。

正确做法：
- 设置 IDE：将 Tab 键转换为 4 个空格。
- VS Code：`Cmd/Ctrl+Shift+P` → `Convert Indentation to Spaces` 来修复已有文件。

### 易错点 3：命名使用拼音或缩写

错误现象：变量名是 `xm`（姓名）、`nl`（年龄）、`sz`（数字），一段时间后自己都看不懂。

错误原因：图省事用了拼音首字母或英文缩写。

正确做法：坚决不用拼音命名。用英文单词，不会的单词查词典。

```python
# 糟糕的命名
def cal_avg(num_list):
    s = 0
    for n in num_list:
        s += n
    return s / len(num_list)

# 好的命名（完全自解释）
def calculate_average(numbers):
    total = 0
    for number in numbers:
        total += number
    return total / len(numbers)
```

### 易错点 4：导入语句混乱

错误现象：`import` 语句散落在文件各处，标准库和第三方库混在一起。

错误原因：写代码时随用随导入，没有整理。

正确做法：所有导入都放在文件顶部，按标准库 → 第三方库 → 本地模块的顺序分组。

## 7. 课后练习（附带答案）

### 练习 1：注释改写

题目：

以下代码的注释大多是"废话注释"。请改写这些注释，使其更有价值。

```python
x = 10      # 设置 x 为 10
y = 3.14    # 设置 y 为 3.14
result = x + y  # x 加上 y
print(result)   # 打印 result
```

答案：

```python
# 安全缓冲区大小（通过实验确定，小于 10 可能触发溢出）
BUFFER_SIZE = 10

# 最大允许的误差范围（产品需求文档 v2.3 指定）
TOLERANCE = 3.14

# 容错阈值：当检测值超过缓冲值 + 误差范围时触发告警
alert_threshold = BUFFER_SIZE + TOLERANCE
print(f"告警阈值: {alert_threshold}")
```

### 练习 2：整理导入语句

题目：

下面的导入语句杂乱无章，请按照 PEP 8 规范重新排列。

```python
from myapp.utils import format_date
import os
import requests
from datetime import datetime
import sys
from myapp.models import User
import numpy as np
```

答案：

```python
# 标准库（按字母顺序）
import os
import sys
from datetime import datetime

# 第三方库（按字母顺序）
import numpy as np
import requests

# 本地模块（按字母顺序）
from myapp.models import User
from myapp.utils import format_date
```

### 练习 3：找出 PEP 8 违规

题目：

下面的代码至少违反了 5 条 PEP 8 规范，请逐一找出并修正。

```python
def calc( a,b,c ):
    result=a+b*c
    return result
def anotherFunc(x,  y):
    return x-y
MyList= [1,2,3,4,5]
```

答案：

| # | 违规 | 修正 |
|---|------|------|
| 1 | `( a,b,c )` 括号和参数间多余空格 | `(a, b, c)` |
| 2 | `def anotherFunc` 函数名应为 snake_case | `def another_func` |
| 3 | `result=a+b*c` 运算符两侧缺空格 | `result = a + b * c` |
| 4 | `(x,  y)` 逗号后多余空格 | `(x, y)` |
| 5 | `MyList` 变量名应为 snake_case | `my_list = [1, 2, 3, 4, 5]` |

## 8. 答题总结

- 注释回答"为什么这样做"，而不是"做了什么"（代码本身已说明做什么）。
- 单行注释用 `#`，文档字符串（docstring）用 `"""..."""`，放定义体第一行。
- PEP 8 核心规则：4 空格缩进、snake_case 变量/函数名、PascalCase 类名、UPPER_CASE 常量名。
- **永远不要混用 Tab 和空格**，在 IDE 中设置 Tab 键转换为 4 个空格。
- 导入语句按"标准库 → 第三方 → 本地"顺序排列，放在文件顶部。
- 好的命名比注释更重要——代码本身就是最好的文档。
