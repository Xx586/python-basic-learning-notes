# pip 包管理

## 1. 知识点定义

pip 是 Python 的官方包管理工具（Pip Installs Packages），从 Python 3.4 开始随安装包自动附带。它用于从 [PyPI](https://pypi.org/)（Python Package Index，拥有超过 40 万个第三方库）下载、安装、升级和卸载 Python 包。

你可以把 pip 理解为 Python 生态的"应用商店"——需要什么功能，用 pip 安装对应的包即可，无需自己从零编写。

掌握 pip 的重要性：
- **不重复造轮子**：社区已经实现了绝大多数通用功能（HTTP 请求、数据分析、图像处理）。
- **依赖管理**：项目可能依赖十几个甚至上百个包，手动管理不现实。
- **环境复现**：通过 `pip freeze` 导出依赖列表，让他人或服务器能精确还原你的开发环境。
- **版本控制**：不同项目可能需要同一个包的不同版本，pip 可以精确安装指定版本。

> 小白理解：
> pip 就像手机上的"应用商店"。你想用计算器功能，不用自己写一个——去商店下载"计算器 App"就行。写 Python 也是一样，需要什么功能，`pip install` 一下就有了。

## 2. 语法格式

### 2.1 安装包

```bash
# 安装最新版本
pip install requests

# 安装指定版本
pip install requests==2.28.0

# 安装版本范围（大于等于某版本）
pip install "requests>=2.28.0"

# 同时安装多个包
pip install requests numpy pandas

# 从 requirements.txt 批量安装
pip install -r requirements.txt
```

### 2.2 卸载包

```bash
pip uninstall requests
# 会提示确认，输入 y 确认卸载
```

### 2.3 查看已安装的包

```bash
# 列出所有已安装的包及其版本
pip list

# 查看某个包的详细信息
pip show requests

# 检查哪些包有可用更新
pip list --outdated
```

### 2.4 升级包

```bash
# 升级 pip 自身
pip install --upgrade pip

# 升级指定包
pip install --upgrade requests
```

### 2.5 导出依赖

```bash
# 查看所有依赖
pip freeze

# 导出到文件（项目根目录下）
pip freeze > requirements.txt
```

`requirements.txt` 文件内容示例：

```
requests==2.28.0
numpy==1.24.3
pandas==2.0.1
```

其他开发者拿到这个文件后，只需一行命令即可安装完全相同的依赖：

```bash
pip install -r requirements.txt
```

### 2.6 换源加速

```bash
# 临时换源（单次安装）
pip install -i https://pypi.tuna.tsinghua.edu.cn/simple requests

# 永久换源（推荐）
pip config set global.index-url https://pypi.tuna.tsinghua.edu.cn/simple
```

## 3. 重点说明

### 3.1 常用国内镜像源

| 镜像源 | URL |
|--------|-----|
| 清华大学 | `https://pypi.tuna.tsinghua.edu.cn/simple` |
| 阿里云 | `https://mirrors.aliyun.com/pypi/simple` |
| 中国科技大学 | `https://pypi.mirrors.ustc.edu.cn/simple` |

### 3.2 --user 安装与权限问题

当在系统级 Python 环境中安装包时可能遇到权限错误：

```
ERROR: Could not install packages due to an OSError: [Errno 13] Permission denied
```

**解决方案：**

| 方案 | 命令 | 推荐度 |
|------|------|:---:|
| 用户目录安装 | `pip install --user requests` | ⚠️ 临时方案 |
| 虚拟环境 | `python3 -m venv venv` + 激活后 `pip install` | ✅ 强烈推荐 |
| sudo（危险） | `sudo pip install requests` | ❌ 永远不要用 |

> **警告**：永远不要用 `sudo pip install`！这可能覆盖系统级别的包，导致操作系统工具故障。

### 3.3 版本号语法

| 语法 | 含义 |
|------|------|
| `requests==2.28.0` | 精确匹配 2.28.0 |
| `requests>=2.28.0` | 安装 2.28.0 及以上任意版本 |
| `requests>=2.28.0,<3.0.0` | 2.28.0 到 3.0.0 之间 |
| `requests~=2.28.0` | 兼容版本，等价于 `>=2.28.0, <2.29.0` |

## 4. Demo 样例

```python
# -*- coding: utf-8 -*-
"""
pip 包管理演示脚本
本脚本演示如何在 Python 代码中检查已安装的包信息
"""

import sys
import importlib


def check_installed_packages():
    """检查常用第三方包是否已安装"""
    print("=" * 50)
    print("检查常用第三方包安装状态")
    print("=" * 50)

    packages = [
        ("requests", "HTTP 请求库"),
        ("numpy", "数值计算库"),
        ("pandas", "数据分析库"),
        ("flask", "Web 框架"),
        ("pytest", "测试框架"),
    ]

    for pkg_name, description in packages:
        try:
            module = importlib.import_module(pkg_name)
            version = getattr(module, "__version__", "未知版本")
            print(f"[已安装] {pkg_name} ({description}) - 版本: {version}")
        except ImportError:
            print(f"[未安装] {pkg_name} ({description})")


def show_pip_commands():
    """以注释形式展示常用 pip 命令"""
    print("\n" + "=" * 50)
    print("常用 pip 命令速查（请在终端中执行）")
    print("=" * 50)

    commands = """
# --- 安装包 ---
pip install requests                    # 安装最新版
pip install requests==2.28.0            # 安装指定版本
pip install -r requirements.txt         # 批量安装

# --- 卸载包 ---
pip uninstall requests                  # 卸载指定包

# --- 查看包 ---
pip list                                # 列出所有已安装的包
pip show requests                       # 查看包的详细信息
pip list --outdated                     # 查看可更新的包

# --- 升级包 ---
pip install --upgrade pip               # 升级 pip 自身
pip install --upgrade requests          # 升级指定包

# --- 导出依赖 ---
pip freeze > requirements.txt           # 导出到文件

# --- 换源加速（国内用户） ---
pip install -i https://pypi.tuna.tsinghua.edu.cn/simple requests
"""
    print(commands)


def show_pip_config():
    """显示当前 pip 基本信息"""
    print("=" * 50)
    print("当前 Python 环境信息")
    print("=" * 50)
    print(f"Python 版本: {sys.version}")
    print(f"Python 可执行文件: {sys.executable}")

    try:
        import pip
        print(f"pip 版本: {pip.__version__}")
    except Exception:
        print("pip 版本: 无法获取（请在终端执行 pip --version）")

    print(f"\n包的安装位置（site-packages）: ")
    for path in sys.path:
        if "site-packages" in path:
            print(f"  - {path}")


if __name__ == "__main__":
    check_installed_packages()
    show_pip_commands()
    show_pip_config()

    print("\n" + "=" * 50)
    print("提示：请在终端中执行 pip list 查看完整包列表！")
    print("=" * 50)
```

运行结果示例：

```
==================================================
检查常用第三方包安装状态
==================================================
[未安装] requests (HTTP 请求库)
[未安装] numpy (数值计算库)
...

==================================================
当前 Python 环境信息
==================================================
Python 版本: 3.12.3
Python 可执行文件: /usr/local/bin/python3
pip 版本: 24.0
...
```

## 6. 易错点

### 易错点 1：pip 命令找不到

错误现象：终端输入 `pip` 显示 `command not found`。

错误原因：Python 安装时未勾选 "Add Python to PATH"（Windows），或系统有 Python 2 和 Python 3 但 pip 指向了 Python 2。

正确做法：
- 使用 `pip3` 替代 `pip`。
- 通用方案：始终使用 `python3 -m pip` 代替裸 `pip` 命令（更可靠）。
- Windows：重新运行安装程序选 "Modify"，勾选 PATH。

### 易错点 2：用 sudo pip install 导致系统问题

错误现象：使用 `sudo pip install` 后，系统工具报错或版本冲突。

错误原因：`sudo` 以 root 身份安装包到系统级目录，可能覆盖系统依赖的包版本。

正确做法：
- **永远不要使用 `sudo pip install`**。
- 正确做法：使用虚拟环境（`python3 -m venv`）隔离项目依赖。

### 易错点 3：pip freeze 导出过多无关包

错误现象：`pip freeze > requirements.txt` 后，文件中包含了几十个自己从未安装过的包。

错误原因：`pip freeze` 导出的是当前 Python 环境中**所有**已安装的包，包括依赖的依赖。

正确做法：
- 用虚拟环境隔离项目（最佳方案），每个项目只有自己需要的包。
- 手动维护 `requirements.txt`，只列出直接依赖的顶层包。

### 易错点 4：换源后下载仍慢或报 SSL 错误

错误现象：配置了国内镜像源，下载仍然很慢或报 SSL 证书错误。

错误原因：镜像源的 `trusted-host` 未配置。

正确做法：确保同时配置了 `index-url` 和 `trusted-host`：

```bash
pip config set global.index-url https://pypi.tuna.tsinghua.edu.cn/simple
pip config set global.trusted-host pypi.tuna.tsinghua.edu.cn
```

## 7. 课后练习（附带答案）

### 练习 1：导出和查看依赖

题目：

使用 `pip freeze` 导出当前环境的依赖，然后查看生成的文件中有多少个包。思考为什么有些包你并没有手动安装过？

答案：

```bash
# 导出依赖到文件
$ pip3 freeze > my_env_packages.txt

# 查看文件行数（即包的个数）
$ wc -l my_env_packages.txt   # macOS/Linux

# 原因：pip freeze 会列出所有包，包括依赖的依赖。
# 例如安装 pandas 时会自动安装 numpy、pytz 等十几个依赖包。
```

### 练习 2：配置国内镜像源

题目：

为你的 pip 配置清华大学镜像源作为默认源。配置完成后，用 `pip config list` 验证配置是否生效。

答案：

```bash
# 命令行配置
$ pip config set global.index-url https://pypi.tuna.tsinghua.edu.cn/simple
$ pip config set global.trusted-host pypi.tuna.tsinghua.edu.cn

# 验证配置
$ pip config list
global.index-url='https://pypi.tuna.tsinghua.edu.cn/simple'
global.trusted-host='pypi.tuna.tsinghua.edu.cn'
```

### 练习 3：模拟完整的依赖管理流程

题目：

假设你创建了一个新项目，需要用到 `requests` 和 `pytest`。请写出从安装 → 开发 → 导出依赖 → 在其他机器上还原的完整命令流程。

答案：

```bash
# 1. 创建虚拟环境
$ python3 -m venv venv
$ source venv/bin/activate  # macOS/Linux
# 或 venv\Scripts\activate  # Windows

# 2. 安装开发需要的包
$ pip install requests pytest

# 3. 导出精确版本的依赖清单
$ pip freeze > requirements.txt

# 4. 在其他机器上还原环境
$ python3 -m venv venv
$ source venv/bin/activate
$ pip install -r requirements.txt
```

## 8. 答题总结

- pip 是 Python 的官方包管理工具，从 Python 3.4 开始自动附带。
- 常用命令：`pip install`（安装）、`pip uninstall`（卸载）、`pip list`（查看）、`pip freeze > requirements.txt`（导出依赖）。
- 国内用户建议配置镜像源加速下载（清华/阿里/中科大）。
- **永远不要用 `sudo pip install`**，使用虚拟环境隔离项目依赖。
- 始终使用 `python3 -m pip` 替代裸 `pip` 命令，避免多版本混淆。
- `requirements.txt` 记录精确版本号（`==`），用于环境复现。
