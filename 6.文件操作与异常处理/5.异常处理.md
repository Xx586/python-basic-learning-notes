# 异常处理

## 1. 知识点定义

异常（Exception）是程序运行时发生的**错误信号**。如果不对异常进行处理，程序会**崩溃终止**。异常处理就是给代码加"保险"，让程序在遇到错误时不崩溃，而是优雅地处理错误并继续运行。

> 小白理解：
> 异常就像开车时遇到的突发情况——爆胎了（文件不存在）、油不够了（除以零）。异常处理就是你的"备用方案"：爆胎了换备胎，油不够了找加油站，而不是直接弃车走人。

它主要解决的问题是：
- 用户输入了不合法数据导致程序崩溃
- 文件不存在、网络断开等外部因素导致程序中断
- 需要区分不同类型的错误并分别处理

在实际写 Python 代码时，它常用于：
- 校验用户输入（防止 `int('abc')` 崩溃）
- 安全地打开文件（文件不存在时给友好提示）
- 数据库操作失败时的回滚处理
- API 请求失败时的重试机制

## 2. 语法格式

### 2.1 try/except/else/finally 完整语法

```python
try:
    # 可能发生异常的代码
    result = 10 / 0
except ZeroDivisionError:
    # 捕获特定异常，执行处理逻辑
    print("除数不能为 0！")
except (ValueError, TypeError) as e:
    # 捕获多个异常，用 as e 获取异常详情
    print(f"数据错误: {e}")
else:
    # try 块没有发生异常时执行（可选）
    print(f"计算成功，结果: {result}")
finally:
    # 无论是否发生异常，一定执行（可选）
    print("清理资源，关闭连接")
```

### 2.2 常见异常类型

| 异常类 | 触发场景 | 示例 |
|--------|---------|------|
| `ValueError` | 传入的值的类型正确但值不正确 | `int('abc')` |
| `TypeError` | 对不兼容的类型进行操作 | `'a' + 1` |
| `KeyError` | 访问字典中不存在的键 | `d = {}; d['x']` |
| `IndexError` | 访问列表中不存在的索引 | `[1,2][5]` |
| `FileNotFoundError` | 打开不存在的文件 | `open('不存在.txt')` |
| `ZeroDivisionError` | 除以零 | `1 / 0` |
| `AttributeError` | 访问对象不存在的属性 | `'abc'.not_exist` |
| `NameError` | 使用未定义的变量 | `print(未定义变量)` |
| `ImportError` | 导入模块失败 | `import 不存在的模块` |

### 2.3 raise：主动抛出异常

```python
def set_age(age):
    if age < 0:
        raise ValueError(f"年龄不能为负数，传入的值: {age}")
    if age > 150:
        raise ValueError(f"年龄太大，不合理: {age}")
    print(f"年龄设置为: {age}")

# set_age(-5)  # 触发 ValueError
```

### 2.4 自定义异常

```python
class InvalidAgeError(Exception):
    """年龄无效异常"""
    def __init__(self, age, message="年龄不符合要求"):
        self.age = age
        self.message = f"{message}: {age}"
        super().__init__(self.message)

# 使用
raise InvalidAgeError(200, "年龄超出合理范围")
```

### 2.5 assert 断言

```python
def calculate_average(scores):
    assert len(scores) > 0, "列表不能为空"   # 条件为 False 时抛出 AssertionError
    return sum(scores) / len(scores)

# calculate_average([])  # AssertionError: 列表不能为空
```

> **注意**：`assert` 主要用于**开发调试**，Python 用 `-O` 模式运行时所有 `assert` 会被跳过。不要用 `assert` 校验用户输入。

## 3. 重点说明

### 3.1 try/except/else/finally 执行流程

| 场景 | try | except | else | finally |
|------|-----|--------|------|---------|
| try 正常执行 | 全部执行 | 跳过 | 执行 | 执行 |
| try 异常被捕获 | 中断 | 执行匹配的 except | 跳过 | 执行 |
| try 异常未被捕获 | 中断 | 跳过 | 跳过 | 执行后向上抛出 |

> `finally` 无论如何都会执行，适合放清理代码（关闭文件、释放锁等）。

### 3.2 捕获异常的最佳实践

```python
# ❌ 恶劣做法：空 except 吞掉所有异常（连 Ctrl+C 都吞了！）
try:
    do_something()
except:
    pass

# ❌ 也很差：捕获太宽泛，不知道具体什么错
try:
    do_something()
except Exception:
    print("出错了")  # 不知道什么错，也不知道哪错了

# ✅ 正确：捕获具体的异常类型
try:
    num = int(input("请输入数字: "))
    result = 100 / num
except ValueError:
    print("请输入有效的数字！")
except ZeroDivisionError:
    print("不能输入 0！")
```

### 3.3 assert 和 raise 的区别

| 对比项 | `assert` | `raise` |
|-------|---------|---------|
| 用途 | 调试时检查程序逻辑 | 处理运行时错误 |
| 是否可被禁用 | 是（`python -O` 模式跳过） | 否 |
| 适用场景 | 开发阶段自检 | 正式的业务逻辑校验 |
| 示例 | `assert x > 0` | `raise ValueError("x 必须大于 0")` |

### 3.4 异常继承关系

```
BaseException
 ├── KeyboardInterrupt      (用户按 Ctrl+C，一般不捕获)
 ├── SystemExit             (sys.exit()，一般不捕获)
 └── Exception              (所有常规异常的基类)
      ├── ValueError
      ├── TypeError
      ├── KeyError
      ├── IndexError
      ├── FileNotFoundError (继承自 OSError)
      ├── ZeroDivisionError
      └── ...（自定义异常也放这里）
```

> **注意**：不要用 `except BaseException` 或裸 `except:`，这会把 `KeyboardInterrupt` 也捕获，导致用户按 Ctrl+C 关不掉程序。

## 4. Demo 样例

```python
# ========== 1. 基本 try/except ==========
print("=" * 40)
print("1. 基本 try/except")

try:
    num = int(input("请输入一个数字: "))  # 用户输入 "abc" 会触发 ValueError
    print(f"你输入的是: {num}")
except ValueError:
    print("错误：请输入有效的数字！")      # 捕获后给友好提示，程序不会崩溃

# ========== 2. 捕获多个异常 + else + finally ==========
print("\n" + "=" * 40)
print("2. 捕获多个异常")

def safe_divide(data, index, divisor_index):
    """安全地进行除法运算，处理多种可能的异常"""
    try:
        a = data[index]                     # 可能触发 IndexError
        b = data[divisor_index]             # 可能触发 IndexError
        result = a / b                      # 可能触发 ZeroDivisionError
    except IndexError as e:
        print(f"索引越界: {e}")             # 捕获索引错误
    except ZeroDivisionError:
        print("除数不能为零")                # 捕获除零错误
    except TypeError as e:
        print(f"类型错误: {e}")             # 捕获类型错误
    else:
        print(f"计算结果: {a} / {b} = {result}")  # 没异常才执行
    finally:
        print("计算操作结束\n")              # 无论是否异常都执行

safe_divide([10, 20, 0], 0, 2)   # 10/0 → 除数不能为零
safe_divide([10, 20], 0, 5)       # 索引越界
safe_divide([10, 20, 5], 0, 2)    # 正常计算: 10/5 = 2.0

# ========== 3. raise 主动抛出异常 ==========
print("=" * 40)
print("3. raise 主动抛出异常")

def validate_password(password):
    """校验密码强度"""
    if len(password) < 6:
        raise ValueError(f"密码长度至少6位，当前只有{len(password)}位")
    if password.isalpha():                       # 全是字母，没有数字或符号
        raise ValueError("密码不能全是字母，需要包含数字或符号")
    print("密码验证通过！")

try:
    validate_password("abc")                     # 长度不够
except ValueError as e:
    print(f"密码校验失败: {e}")

# ========== 4. 自定义异常 ==========
print("\n" + "=" * 40)
print("4. 自定义异常类")

class BalanceInsufficientError(Exception):
    """余额不足异常"""
    def __init__(self, balance, amount):
        self.balance = balance
        self.amount = amount
        super().__init__(f"余额 {balance} 元，无法支付 {amount} 元")

def withdraw(balance, amount):
    """取款，余额不足时抛自定义异常"""
    if amount > balance:
        raise BalanceInsufficientError(balance, amount)
    balance -= amount
    print(f"取款成功！余额: {balance}")
    return balance

try:
    withdraw(100, 200)                           # 余额不足
except BalanceInsufficientError as e:
    print(f"取款失败: {e}")
    print(f"差额: {e.amount - e.balance} 元")     # 可以访问自定义属性

# ========== 5. assert 断言 ==========
print("\n" + "=" * 40)
print("5. assert 断言")

def get_grade(score):
    """根据分数返回等级，分数必须在 0-100 之间"""
    assert 0 <= score <= 100, f"分数必须在0-100之间，传入: {score}"
    if score >= 90:
        return 'A'
    elif score >= 80:
        return 'B'
    else:
        return 'C'

print(f"85分的等级: {get_grade(85)}")            # 正常

try:
    get_grade(150)                               # 超出范围
except AssertionError as e:
    print(f"断言失败: {e}")

print("\n异常处理演示完成！")
```

## 5. 典型例题（附带答案）

### 例题 1

**题目：**

编写函数 `safe_input_int(prompt)`，反复提示用户输入直到输入一个有效整数。如果用户输入 `'q'`，退出循环返回 `None`。

**答案：**

```python
def safe_input_int(prompt):
    """安全地获取用户输入的整数，输入 'q' 退出"""
    while True:
        user_input = input(prompt)
        if user_input.lower() == 'q':       # 检查是否要退出
            return None
        try:
            return int(user_input)           # 尝试转为整数
        except ValueError:
            print(f"'{user_input}' 不是有效整数，请重新输入（输入 q 退出）")

# 测试
age = safe_input_int("请输入年龄: ")
if age is not None:
    print(f"你的年龄是: {age}")
```

**解析：** 用无限循环 + `try/except` 实现"不输入正确就不放过"的效果。输入 `q` 时正常退出，避免死循环。

### 例题 2

**题目：**

创建一个简单的银行账户类，实现存款和取款功能。取款时余额不足抛出自定义异常 `InsufficientFundsError`，取款金额为负数也抛异常。

**答案：**

```python
class InsufficientFundsError(Exception):
    """余额不足异常"""
    def __init__(self, balance, amount):
        self.balance = balance
        self.amount = amount
        super().__init__(f"余额不足：当前余额 {balance}，取款金额 {amount}")

class InvalidAmountError(Exception):
    """金额无效异常"""
    def __init__(self, amount):
        self.amount = amount
        super().__init__(f"金额无效: {amount}")

class BankAccount:
    def __init__(self, owner, balance=0):
        self.owner = owner
        self.balance = balance

    def deposit(self, amount):
        """存款"""
        if amount <= 0:
            raise InvalidAmountError(amount)
        self.balance += amount
        print(f"存入 {amount}，余额: {self.balance}")

    def withdraw(self, amount):
        """取款"""
        if amount <= 0:
            raise InvalidAmountError(amount)
        if amount > self.balance:
            raise InsufficientFundsError(self.balance, amount)
        self.balance -= amount
        print(f"取出 {amount}，余额: {self.balance}")

# 测试
account = BankAccount("张三", 1000)
try:
    account.deposit(500)
    account.withdraw(2000)       # 余额不足，抛自定义异常
except InsufficientFundsError as e:
    print(f"取款失败: {e}")
except InvalidAmountError as e:
    print(f"金额无效: {e}")
```

**解析：** 自定义异常类可以携带额外的上下文信息（如 `balance`、`amount`），方便外部捕获后做更细致的处理。

## 6. 易错点

### 易错点 1：空 except 吞掉所有异常

错误写法：
```python
try:
    risky_operation()
except:         # 什么都不写，连 Ctrl+C 都吞了！
    pass
```

错误原因：裸 `except:` 会捕获所有异常，包括 `KeyboardInterrupt` 和 `SystemExit`，导致程序关不掉。

正确写法：
```python
try:
    risky_operation()
except FileNotFoundError as e:
    print(f"文件不存在: {e}")
except ValueError as e:
    print(f"数据错误: {e}")
except Exception as e:
    # 用 Exception 兜底，但至少记录日志
    print(f"未预期的错误: {type(e).__name__}: {e}")
```

### 易错点 2：try 块包裹太多代码

错误写法：
```python
try:
    data = load_file("a.txt")
    config = parse_config(data)
    result = connect_db(config)    # 一大坨代码塞一个 try
    process_data(result)
except Exception as e:
    print(f"出错了: {e}")          # 到底哪一步错了？
```

错误原因：异常发生后不知道是哪一行出的问题，排查困难。

正确写法：
```python
try:
    data = load_file("a.txt")
except FileNotFoundError:
    print("文件 a.txt 不存在")
    return

config = parse_config(data)         # 这行不太可能出错，移到 try 外面

try:
    db = connect_db(config)
except ConnectionError:
    print("数据库连接失败")
    return
```

### 易错点 3：用 assert 做业务校验

错误写法：
```python
def process_order(amount):
    assert amount > 0, "金额必须大于0"   # 生产环境 -O 模式会被跳过！
    # ...
```

错误原因：Python 用 `-O`（优化模式）运行时所有 `assert` 语句被忽略。

正确写法：
```python
def process_order(amount):
    if amount <= 0:
        raise ValueError("金额必须大于0")  # raise 不会被跳过
    # ...
```

### 易错点 4：finally 中写 return

错误写法：
```python
def bad_func():
    try:
        return 1
    finally:
        return 2   # finally 中的 return 会覆盖 try 中的 return！
print(bad_func())   # 输出 2，不是 1！
```

错误原因：`finally` 中的 `return` 会覆盖 `try` 块的 `return` 值。

正确写法：
```python
def good_func():
    try:
        return 1
    finally:
        print("清理资源")   # finally 只做清理，不写 return
```

### 易错点 5：捕获了异常却不处理就重抛

错误写法：
```python
try:
    do_something()
except Exception:
    raise    # 相当于没写 try/except，毫无意义
```

错误原因：捕获后什么都不做就重抛，跟不捕获效果一样。

正确写法：
```python
# 要么别捕获
do_something()

# 要么捕获后做有意义的处理再重抛
try:
    do_something()
except Exception:
    logging.exception("操作失败，重新抛出")   # 至少记录日志
    raise
```

## 7. 课后练习（附带答案）

### 练习 1

**题目：**

编写函数 `safe_read_file(filepath)`，尝试用多种编码（utf-8, utf-8-sig, gbk, gb2312）读取文件，找到能成功打开的编码后返回文件内容。所有编码都失败则返回 `None`。

**答案：**

```python
def safe_read_file(filepath):
    """尝试多种编码读取文件"""
    encodings = ['utf-8', 'utf-8-sig', 'gbk', 'gb2312', 'gb18030']

    for enc in encodings:
        try:
            with open(filepath, 'r', encoding=enc) as f:
                content = f.read()
            print(f"✓ 用 {enc} 编码读取成功")
            return content
        except UnicodeDecodeError:       # 编码不对，继续尝试下一种
            continue
        except FileNotFoundError:        # 文件不存在，直接返回
            print(f"文件不存在: {filepath}")
            return None

    print(f"所有编码均无法读取: {filepath}")
    return None
```

**说明：** 编码问题只抛出 `UnicodeDecodeError`，文件不存在抛出 `FileNotFoundError`，需要分开处理。

### 练习 2

**题目：**

编写装饰器 `retry_on_exception(max_retries=3)`，被装饰的函数抛出异常时自动重试，达到最大重试次数后仍失败则抛出异常。

**答案：**

```python
import functools

def retry_on_exception(max_retries=3, exceptions=(Exception,)):
    """自动重试装饰器"""
    def decorator(func):
        @functools.wraps(func)
        def wrapper(*args, **kwargs):
            for attempt in range(1, max_retries + 1):
                try:
                    return func(*args, **kwargs)
                except exceptions as e:
                    if attempt == max_retries:    # 最后一次也失败了
                        raise                     # 不再重试，向上抛出
                    print(f"第 {attempt} 次失败: {e}，重试中...")
        return wrapper
    return decorator

# 测试
import random

@retry_on_exception(max_retries=3, exceptions=(ValueError,))
def unstable_operation():
    if random.random() < 0.7:
        raise ValueError("随机失败")
    return "操作成功"

for i in range(3):
    try:
        result = unstable_operation()
        print(f"结果: {result}")
    except ValueError as e:
        print(f"最终失败: {e}")
```

**说明：** 装饰器结构较复杂，本练习可以让学生了解"异常处理 + 装饰器"的组合用法。

### 练习 3

**题目：**

编写函数 `clean_data(data_list, required_fields)`。接收一个字典列表和必填字段列表，清理无效数据：缺少必填字段或字段值为空的记录被跳过并打印警告，返回清洗后的数据列表。

**答案：**

```python
class DataValidationError(Exception):
    """数据校验异常"""
    def __init__(self, index, field):
        self.index = index
        self.field = field
        super().__init__(f"记录 #{index} 缺少必填字段 '{field}' 或值为空")

def clean_data(data_list, required_fields):
    """清洗数据，移除不符合要求的记录"""
    cleaned = []
    for i, record in enumerate(data_list):
        try:
            for field in required_fields:
                if field not in record or not str(record[field]).strip():
                    raise DataValidationError(i, field)
            cleaned.append(record)           # 所有检查通过，保留
        except DataValidationError as e:
            print(f"⚠ 跳过: {e}")

    print(f"\n清洗结果: {len(cleaned)}/{len(data_list)} 条有效数据")
    return cleaned

# 测试
raw_data = [
    {"name": "张三", "age": "25", "city": "北京"},    # 有效
    {"name": "", "age": "30", "city": "上海"},        # name 为空
    {"name": "王五", "city": "广州"},                  # 缺少 age
    {"name": "赵六", "age": "28", "city": "深圳"},     # 有效
]

cleaned_data = clean_data(raw_data, ["name", "age", "city"])
print(cleaned_data)
```

**说明：** 自定义异常让数据清洗逻辑更清晰，能精确定位哪条记录的哪个字段有问题。

## 8. 答题总结

- 异常是程序运行时的错误信号，`try/except` 可以捕获异常防止程序崩溃。
- `except 具体异常类型` 优于裸 `except:`，能精确定位和处理问题。
- `else` 在无异常时执行，`finally` 无论有无异常都执行（适合清理资源）。
- `raise` 主动抛出异常，`assert` 用于开发调试（不要用于业务校验）。
- 自定义异常继承 `Exception`，可以添加额外属性携带上下文信息。
- 常见异常类型：`ValueError`、`TypeError`、`KeyError`、`IndexError`、`FileNotFoundError`、`ZeroDivisionError`。
- 不要用空 `except:` 吞异常，不要用 `assert` 校验用户输入，不要在 `finally` 中 `return`。
