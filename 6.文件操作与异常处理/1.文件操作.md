# 文件操作

## 1. 知识点定义

文件操作是指 Python 程序对电脑硬盘上的文件进行**读取**和**写入**的功能。

> 小白理解：
> 程序运行时产生的数据存在内存里，程序关了数据就没了。文件操作就像把数据"存档"到硬盘上——和游戏存档是一个道理。下次再打开程序，可以把存档读回来继续用。

它主要解决的问题是：
- 程序数据需要**持久化保存**（不能程序一关数据就没了）
- 需要读取外部文件（配置文件、日志文件、用户上传的文件等）
- 需要批量处理文本数据（分析小说、处理 CSV 表格、清洗数据等）

在实际写 Python 代码时，它常用于：
- 保存用户配置（如游戏存档、软件设置）
- 读取日志文件进行分析
- 批量处理文本数据
- 读写图片、音频等二进制数据

## 2. 语法格式

### 2.1 打开文件：open()

```python
f = open(file, mode='r', encoding=None)
```

**参数说明：**

| 参数 | 说明 |
|------|------|
| `file` | 文件路径（可以是相对路径或绝对路径） |
| `mode` | 打开模式，默认为 `'r'`（只读） |
| `encoding` | 编码方式，文本文件推荐始终指定 `encoding='utf-8'` |

### 2.2 文件打开模式

#### 基本读写模式

| 模式 | 含义 | 文件不存在时 | 文件存在时 |
|------|------|-------------|-----------|
| `'r'` | 只读（默认） | 报错 FileNotFoundError | 从头读取 |
| `'w'` | 只写 | 创建新文件 | **清空**原有内容，从头写入 |
| `'a'` | 追加写 | 创建新文件 | 在末尾追加，不清空 |
| `'x'` | 排他创建 | 创建新文件 | 报错 FileExistsError |

#### 二进制模式

| 模式 | 含义 |
|------|------|
| `'rb'` | 二进制只读 |
| `'wb'` | 二进制只写 |
| `'ab'` | 二进制追加写 |

> `'rb'` 和 `'wb'` 用于处理图片、PDF、音频等非文本文件。

### 2.3 读取文件

```python
# read() —— 一次性读取全部内容
with open('文件.txt', 'r', encoding='utf-8') as f:
    content = f.read()          # 返回整个文件的字符串

# readline() —— 每次读取一行
with open('文件.txt', 'r', encoding='utf-8') as f:
    line1 = f.readline()        # 读取第一行（包含末尾的 \n）
    line2 = f.readline()        # 读取第二行

# readlines() —— 返回所有行的列表
with open('文件.txt', 'r', encoding='utf-8') as f:
    lines = f.readlines()       # ['第一行\n', '第二行\n', ...]

# for line in f —— 逐行迭代（推荐！）
with open('文件.txt', 'r', encoding='utf-8') as f:
    for line in f:              # 惰性迭代，不会一次性加载全部到内存
        print(line.strip())     # strip() 去掉末尾的换行符
```

### 2.4 写入文件

```python
# write() —— 写入字符串（不会自动加换行符）
with open('输出.txt', 'w', encoding='utf-8') as f:
    f.write('Hello\n')          # 需要手动加 \n
    f.write('World\n')

# writelines() —— 写入字符串列表（也不会自动加换行符）
lines = ['第一行\n', '第二行\n', '第三行\n']
with open('输出.txt', 'w', encoding='utf-8') as f:
    f.writelines(lines)

# 追加写入 —— 使用 'a' 模式
with open('输出.txt', 'a', encoding='utf-8') as f:
    f.write('这是追加的内容\n')
```

### 2.5 with 语句（推荐写法）

```python
# ✅ 推荐写法：自动关闭文件
with open('文件.txt', 'r', encoding='utf-8') as f:
    content = f.read()
# 离开 with 块后，文件自动关闭，即使中间发生异常也不会漏关

# ❌ 不推荐：需要手动关闭，如果中间发生异常可能关不掉
f = open('文件.txt', 'r', encoding='utf-8')
content = f.read()
f.close()
```

## 3. 重点说明

### 3.1 什么时候用哪种读取方式？

| 读取方式 | 使用场景 | 内存占用 |
|---------|---------|---------|
| `read()` | 小文件（几十KB ~ 几MB） | 整个文件加载到内存 |
| `readline()` | 需要逐行控制读取进度 | 只有当前行 |
| `readlines()` | 需要以列表形式操作所有行 | 整个文件加载到内存 |
| `for line in f` | **大文件首选**（几百MB ~ GB级） | 只有当前行 |

> **推荐理由**：`for line in f` 是惰性迭代，不会一次性把所有内容加载到内存，处理大文件时最安全。

### 3.2 w 和 a 的区别（初学者最容易搞混！）

| 对比项 | `w` 模式 | `a` 模式 |
|-------|---------|---------|
| 文件已存在 | **清空**原内容 | **保留**原内容，在末尾追加 |
| 文件不存在 | 创建新文件 | 创建新文件 |
| 常见误用 | 想追加却用了 `w`，数据全丢了！ | — |

> 小白理解：`w` = write（重写），像用新本子盖掉旧本子；`a` = append（附加），像在本子最后一页接着写。

### 3.3 文本模式 vs 二进制模式

```python
# 文本模式：处理文字（自动处理编码）
with open('日记.txt', 'r', encoding='utf-8') as f:
    content = f.read()

# 二进制模式：复制图片/PDF/音频（不涉及编码，逐字节复制）
with open('原图.jpg', 'rb') as src:
    data = src.read()
with open('副本.jpg', 'wb') as dst:
    dst.write(data)
```

> **关键**：如果误用文本模式打开图片，Python 会尝试按编码方式解码字节数据，导致文件损坏。

### 3.4 with 语句的好处

1. 离开代码块时**自动调用** `f.close()`，即使发生异常也不会漏关
2. 代码更简洁，作用域更清晰
3. 是 Python 官方推荐的最佳实践

## 4. Demo 样例

```python
# ========== 写入文件 ==========
# 使用 'w' 模式创建并写入文件，如果文件已存在则清空后重新写入
with open('测试.txt', 'w', encoding='utf-8') as f:
    f.write('第一行内容\n')   # write() 不会自动加换行，需要手动写 \n
    f.write('第二行内容\n')
    f.write('第三行内容\n')
print("文件写入完成！")

# ========== 逐行读取（推荐方式，处理大文件也安全） ==========
print("\n读取文件内容：")
with open('测试.txt', 'r', encoding='utf-8') as f:
    for line in f:                        # 逐行迭代，一次只读一行到内存
        print(f"读到: {line.strip()}")    # strip() 去掉行尾的换行符

# ========== 追加内容 ==========
# 使用 'a' 模式在文件末尾追加，不会删除原有内容
with open('测试.txt', 'a', encoding='utf-8') as f:
    f.write('这是追加的内容\n')
print("\n追加了一行新内容！")

# ========== 验证追加结果 ==========
print("\n追加后的完整内容：")
with open('测试.txt', 'r', encoding='utf-8') as f:
    print(f.read())  # 小文件可以用 read() 一次性读完

# ========== 二进制模式复制文件 ==========
# 'rb' 读取二进制数据，'wb' 写入二进制数据，用于非文本文件
with open('测试.txt', 'rb') as src:       # 以二进制方式读
    data = src.read()                     # 得到 bytes 类型数据
with open('测试_副本.txt', 'wb') as dst:   # 以二进制方式写
    dst.write(data)
print("二进制复制完成：测试_副本.txt")
```

**运行结果：**

```
文件写入完成！

读取文件内容：
读到: 第一行内容
读到: 第二行内容
读到: 第三行内容

追加了一行新内容！

追加后的完整内容：
第一行内容
第二行内容
第三行内容
这是追加的内容

二进制复制完成：测试_副本.txt
```

## 5. 典型例题（附带答案）

### 例题 1

**题目：**

编写程序，读取一个文本文件，统计并输出：
- 总行数
- 总字符数（不含换行符）
- 总单词数（以空格分隔）

**答案：**

```python
def analyze_file(filepath):
    """统计文件的行数、字符数和单词数"""
    with open(filepath, 'r', encoding='utf-8') as f:
        lines = f.readlines()  # 读取所有行到列表

    # 统计总字符数：去掉每行的换行符后累加长度
    total_chars = sum(len(line.rstrip('\n')) for line in lines)
    # 统计总单词数：每行按空格分割后统计单词数量
    total_words = sum(len(line.split()) for line in lines)

    print(f"总行数: {len(lines)}")
    print(f"总字符数（不含换行符）: {total_chars}")
    print(f"总单词数: {total_words}")

# 测试
# analyze_file('测试.txt')
```

**解析：**
- `f.readlines()` 一次性读取所有行，返回列表（小文件适用）
- `rstrip('\n')` 去掉行尾的换行符，避免统计字符数时多算
- `split()` 按空白字符分割字符串，返回单词列表

### 例题 2

**题目：**

编写函数 `search_keyword(filepath, keyword)`，读取文件并打印所有包含指定关键词的行，要求带行号显示。

**答案：**

```python
def search_keyword(filepath, keyword):
    """搜索文件中包含关键词的行"""
    with open(filepath, 'r', encoding='utf-8') as f:
        for line_num, line in enumerate(f, start=1):  # start=1 让行号从 1 开始
            if keyword in line:                        # 判断关键词是否在当前行
                print(f"第{line_num}行: {line.strip()}")

# 测试
# search_keyword('测试.txt', '内容')
```

**解析：**
- `enumerate(f, start=1)` 在遍历文件的同时生成行号，`start=1` 使行号从 1 开始
- `keyword in line` 是最简单的字符串包含判断，适合零基础学生

## 6. 易错点

### 易错点 1：w 模式会清空原文件

错误写法：
```python
# 想追加内容却用了 'w'，原内容全部丢失！
with open('data.txt', 'w', encoding='utf-8') as f:
    f.write('新内容')
```

错误原因：`'w'` 模式在打开文件时会立即清空所有原有内容。

正确写法：
```python
# 追加内容用 'a' 模式
with open('data.txt', 'a', encoding='utf-8') as f:
    f.write('追加内容')
```

### 易错点 2：忘记关闭文件 / 没有使用 with

错误写法：
```python
f = open('data.txt', 'r', encoding='utf-8')
data = f.read()
# 忘了 f.close()，文件一直开着，可能造成资源泄漏
```

错误原因：如果 `read()` 过程中发生异常，`f.close()` 根本不会执行。

正确写法：
```python
# 始终使用 with，自动关闭
with open('data.txt', 'r', encoding='utf-8') as f:
    data = f.read()
```

### 易错点 3：Windows 路径中的反斜杠

错误写法：
```python
path = "C:\Users\name\test.txt"  # \n 被当成换行符，\t 被当成 Tab！
```

错误原因：字符串中 `\` 是转义字符，`\n`、`\t` 等有特殊含义。

正确写法：
```python
# 方法1：使用原始字符串（推荐）
path = r"C:\Users\name\test.txt"

# 方法2：使用正斜杠（Python 跨平台都能识别）
path = "C:/Users/name/test.txt"

# 方法3：使用双反斜杠
path = "C:\\Users\\name\\test.txt"
```

### 易错点 4：不指定 encoding 导致中文乱码

错误写法：
```python
# Windows 上默认编码可能是 gbk，不是 utf-8
with open('文件.txt', 'r') as f:  # 没有指定 encoding
    content = f.read()
```

错误原因：不同操作系统的默认编码不同，不显式指定极易乱码。

正确写法：
```python
# 始终显式指定 encoding='utf-8'
with open('文件.txt', 'r', encoding='utf-8') as f:
    content = f.read()
```

## 7. 课后练习（附带答案）

### 练习 1

**题目：**

编写程序，将多个文本文件的内容合并到一个新文件中，每个文件内容之间用分隔线 `---` 隔开。

**答案：**

```python
def merge_files(input_files, output_file):
    """将多个文件合并为一个文件"""
    with open(output_file, 'w', encoding='utf-8') as out:
        for i, filepath in enumerate(input_files):
            with open(filepath, 'r', encoding='utf-8') as inf:
                out.write(inf.read())            # 写入当前文件内容
            if i < len(input_files) - 1:         # 最后一个文件后面不加分隔线
                out.write('\n---\n')

# merge_files(['a.txt', 'b.txt', 'c.txt'], 'merged.txt')
```

**说明：** 关键是判断是否为最后一个文件（`i < len(input_files) - 1`），最后一个文件末尾不加分隔线。

### 练习 2

**题目：**

编写程序，以二进制模式复制任意类型文件（图片、PDF 等），并显示复制进度（百分比）。
提示：使用 `os.path.getsize()` 获取文件总大小，分块读取写入。

**答案：**

```python
import os

def copy_binary(src_path, dst_path, buffer_size=8192):
    """二进制方式复制文件，显示进度"""
    total_size = os.path.getsize(src_path)   # 获取源文件大小（字节）
    copied = 0

    with open(src_path, 'rb') as src, open(dst_path, 'wb') as dst:
        while True:
            chunk = src.read(buffer_size)    # 每次读一块（8KB）
            if not chunk:                    # 读到文件末尾，退出
                break
            dst.write(chunk)                 # 写入目标文件
            copied += len(chunk)             # 累加已复制字节数
            percent = (copied / total_size) * 100
            print(f"\r复制进度: {percent:.1f}%", end='')  # \r 回到行首刷新显示
    print()  # 换行

# copy_binary('原图.jpg', '副本.jpg')
```

**说明：** `buffer_size=8192`（8KB）是常用的缓冲大小，分块读写避免一次加载整个大文件到内存。

### 练习 3

**题目：**

编写程序，读取文件内容，将每一行倒序后写入新文件（行的顺序不变，每行内的字符顺序反转）。

**答案：**

```python
def reverse_lines(src_path, dst_path):
    """将每行字符反转后写入新文件"""
    with open(src_path, 'r', encoding='utf-8') as src, \
         open(dst_path, 'w', encoding='utf-8') as dst:
        for line in src:
            reversed_line = line.rstrip('\n')[::-1]  # 去掉换行 → 反转字符串
            dst.write(reversed_line + '\n')          # 写回时补上换行

# reverse_lines('input.txt', 'reversed.txt')
```

**说明：** `[::-1]` 是 Python 字符串反转的切片写法；注意要先用 `rstrip('\n')` 去掉换行再反转，写回时手动加 `\n`。

## 8. 答题总结

- `open()` 是 Python 打开文件的核心函数，需要指定文件路径、模式和编码。
- 文件模式中，`r` 读、`w` 写（覆盖）、`a` 追加，`rb`/`wb` 用于二进制文件。
- `with open() as f:` 是最推荐的写法，可以自动关闭文件，即使发生异常也不会漏关。
- 读取大文件时用 `for line in f` 逐行迭代，不要用 `read()` 一次性加载全部内容。
- `write()` 和 `writelines()` 都不会自动添加换行符，需要手动写 `\n`。
- 处理中文文件时，始终显式写上 `encoding='utf-8'`，避免乱码。
- Windows 路径中的反斜杠要用原始字符串 `r"..."` 或正斜杠 `/` 替代。
- 初学者最容易出错的是：模式选错（`w` vs `a`）、忘记关闭文件、路径写错、编码不一致。
