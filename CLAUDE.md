# Python 基础资料整理项目规则

## 1. 总目标

本项目是整理 Python 基础语法学习资料，面向零基础学生。

必须先阅读现有所有资料，再生成理论讲解文档。

## 2. 必须读取的资料

每次开始任务前，必须优先阅读：

1. `Python基础学习路线(5).md`
2. `2.AI应用学习方式(2).md`
3. `思维导图.png`
4. 现有 1-10 章节文件夹中的资料
5. 所有截图中的知识点结构

## 3. 知识点文档固定框架

每个知识点 `.md` 文件必须使用以下结构：

# 知识点名称

## 1. 知识点定义
## 2. 语法格式
## 3. 重点说明
## 4. Demo 样例
## 5. 典型例题（附带答案）
## 6. 易错点
## 7. 课后练习（附带答案）
## 8. 答题总结

## 4. 生成原则

- 每个知识点一个独立 `.md` 文件。
- 不能遗漏学习路线、思维导图、截图中的知识点。
- 理论讲解要适合 Python 零基础学生。
- Demo 必须可运行。
- Demo 代码讲解必须写成代码注释。
- 理论补充内容直接加到原文对应位置，不另建解释文件。
- 不要误删原始资料。
- 修改前必须先备份。
- 每次只处理一个章节，不要一次性生成全部章节。

## 5. 执行流程

每次处理章节时必须按顺序执行：

1. 阅读本章节已有资料。
2. 阅读学习路线和思维导图中与本章相关内容。
3. 输出本章知识点清单。
4. 输出本章整理方案。
5. 等待用户确认。
6. 创建或更新本章 `.md` 文件。
7. 检查是否遗漏知识点。
8. 输出本章完成报告。
