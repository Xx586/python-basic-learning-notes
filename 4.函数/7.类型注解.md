# 类型注解

## 1. 知识点定义

**类型注解（Type Hints）** 是 Python 3.5+ 引入的语法，用来**标注变量、函数参数和返回值的预期类型**。最重要的是：**类型注解不影响代码运行**，Python 解释器完全忽略它们。

它们的作用：提高代码**可读性**（一眼知道参数该传什么）、配合 IDE 实现**智能提示**、配合 **mypy** 工具做**静态类型检查**。

> 小白理解：
> 类型注解就像快递单上的"易碎品"标签——贴不贴包裹都能寄到，但贴了快递员会更小心，收件人也知道里面是什么。Python 不会因为注解标错了就不运行，但它能帮人和 IDE 更好地理解代码。

## 2. 语法格式

```python
# 最基础的函数类型注解
def add(a: int, b: int) -> int:
    return a + b

def greet(name: str = "World") -> str:
    return f"Hello, {name}!"

def log(msg: str) -> None:    # 无返回值标注为 None
    print(msg)

# 变量注解（Python 3.6+）
name: str = "张三"
age: int = 25
scores: list[int] = [85, 92, 78]     # Python 3.9+
# 等价于：
from typing import List
scores: List[int] = [85, 92, 78]     # Python 3.5-3.8

# 复杂类型
from typing import Optional, Union
def find_user(name: str) -> Optional[dict]:  # 返回 dict 或 None
    pass
```

## 3. 重点说明

### 3.1 类型注解不强制检查

```python
def double(x: int) -> int:
    return x * 2

print(double("hello"))  # "hellohello" — 不会报错！注解只是提示
```

真正的类型检查需要 `mypy` 工具：`pip install mypy && mypy script.py`。

### 3.2 常用 typing 类型

| typing | 3.9+ 简化 | 含义 |
|--------|----------|------|
| `List[str]` | `list[str]` | 字符串列表 |
| `Dict[str, int]` | `dict[str, int]` | 键str值int字典 |
| `Optional[int]` | `int \| None` | int 或 None |
| `Union[int, str]` | `int \| str` | int 或 str |
| `Any` | — | 任意类型 |

> 更多 advanced typing 用法（`Callable`, `TypeVar`, `Protocol` 等）见 [10.进阶语法/6.类型注解.md](../10.进阶语法/6.类型注解.md)。

## 4. Demo 样例

```python
from typing import List, Optional

# ===== 1. 基本函数注解 =====
def calculate_average(scores: List[int]) -> float:
    """参数是整数列表，返回浮点数"""
    return sum(scores) / len(scores)

scores: List[int] = [85, 92, 78, 90, 88]
avg: float = calculate_average(scores)
print(f"平均分: {avg:.1f}")  # 86.6

# ===== 2. Optional 类型 =====
def find_score(name: str, db: dict) -> Optional[int]:
    return db.get(name)

db = {"张三": 85, "李四": 92}
result = find_score("王五", db)
print(result)  # None（Optional 类型需判空再使用）
```

## 5. 典型例题（附带答案）

### 例题 1

题目：给函数添加类型注解：

```python
def get_top(students, n):
    sorted_s = sorted(students.items(), key=lambda x: x[1], reverse=True)
    return [name for name, _ in sorted_s[:n]]
```

答案：`def get_top(students: dict[str, int], n: int) -> list[str]:`

### 例题 2

题目：定义 `safe_divide(a, b)`，b 可能为 0 返回 None，写出带注解的版本。

答案：`def safe_divide(a: float, b: float) -> Optional[float]:`

## 6. 易错点

### 易错点 1：以为注解会强制检查

Python 不会因类型注解报错。类型检查需配合 mypy 等工具。

### 易错点 2：Optional 需判空

返回 `Optional[int]` 后，使用前务必 `if result is not None:`。

### 易错点 3：Python 版本兼容

`list[int]` (3.9+)、`int | None` (3.10+)。老项目用 `List[int]`, `Optional[int]`。

## 7. 课后练习（附带答案）

### 练习 1

题目：给函数 `def merge_dicts(d1, d2): return {**d1, **d2}` 加类型注解。

答案：`def merge_dicts(d1: dict, d2: dict) -> dict:`

### 练习 2

题目：写出 `list[int | str]` 和 `dict[str, tuple[int, float]]` 的含义。

答案：元素可以是 int 或 str 的列表；键为 str、值为 (int, float) 元组的字典。

## 8. 答题总结

- 类型注解标注参数和返回值的预期类型，不影响运行，是给人/IDE 看的说明书。
- Python 3.9+ 可用 `list[int]`、`dict[str, int]` 简化写法。
- `Optional[X]` = `X | None`，表示可以是指定类型或 None。
- `typing` 模块提供 `List`、`Dict`、`Optional`、`Union`、`Any` 等泛型类型。
- 真正类型检查需 mypy：`pip install mypy && mypy script.py`。
