# 协程基础

## 1. 知识点定义

**协程（Coroutine）** 是 Python 中一种轻量级的并发编程方式，通过 `async`/`await` 语法实现单线程内的"并发"。与多线程不同，协程的切换由程序员显式控制（协作式），而非操作系统强制切换（抢占式）。

> 小白理解：
> 协程就像一个人同时做多件事——煮水的时候去切菜，切完菜水也开了。虽然只有一个人（单线程），但通过合理安排时间，看起来像同时在做多件事。

它主要解决的问题是：
- 高并发 I/O 场景（如 Web 服务器同时处理数千请求）
- 比多线程更轻量，可轻松创建上万个协程
- 代码写起来像同步代码，运行起来是异步的

## 2. 语法格式

### 2.1 async / await 基本语法

```python
import asyncio

# async def 定义协程函数
async def fetch_data():
    print("开始获取数据...")
    await asyncio.sleep(1)    # 非阻塞等待，让出控制权
    return "数据内容"

# await 只能在 async def 函数内部使用
async def process():
    data = await fetch_data()  # 挂起当前协程，等待 fetch_data 完成
    print(f"获取到: {data}")

# asyncio.run() 启动协程（Python 3.7+）
asyncio.run(process())
```

### 2.2 asyncio.gather() —— 并发运行多个协程

```python
async def fetch(x):
    await asyncio.sleep(0.1)
    return x * 2

async def main():
    # 并发执行，总耗时 ≈ 最慢的那个
    results = await asyncio.gather(
        fetch(1), fetch(2), fetch(3),
    )
    print(results)  # [2, 4, 6]

asyncio.run(main())
```

### 2.3 asyncio.create_task() —— 后台任务

```python
async def background_work():
    await asyncio.sleep(1)
    return "done"

async def main():
    task = asyncio.create_task(background_work())  # 提交到事件循环
    print("主协程做其他事...")
    result = await task   # 等后台任务完成
    print(f"结果: {result}")
```

## 3. 重点说明

### 3.1 同步 vs 异步对比

```python
# 同步：总耗时 = 2 + 2 = 4 秒
time.sleep(2)   # 阻塞！CPU 空等
time.sleep(2)

# 异步：总耗时 ≈ max(2, 2) = 2 秒
await asyncio.sleep(2)   # 非阻塞！让出控制权
await asyncio.sleep(2)
```

### 3.2 time.sleep vs asyncio.sleep

| 对比 | `time.sleep()` | `asyncio.sleep()` |
|------|---------------|-------------------|
| 行为 | 阻塞整个线程 | 只挂起当前协程 |
| 在协程中使用 | ❌ 阻塞事件循环 | ✅ 其他协程可继续运行 |

### 3.3 协程 vs 线程

| 维度 | 协程 | 线程 |
|------|------|------|
| 调度 | 用户态（协作式） | 系统态（抢占式） |
| 切换开销 | 极小 | 较大 |
| GIL | 不受限制（单线程） | 受 GIL 限制 |
| 并发量 | 可轻松上万 | 受内存限制 |

## 4. Demo 样例

```python
import asyncio
import time

# ========== 1. 同步 vs 异步对比 ==========
print("=" * 40)
print("1. 同步 vs 异步耗时对比")

async def async_task(name, delay):
    print(f"{name} 开始")
    await asyncio.sleep(delay)       # 非阻塞
    print(f"{name} 结束")

def sync_task(name, delay):
    print(f"{name} 开始")
    time.sleep(delay)                # 阻塞
    print(f"{name} 结束")

# 同步：顺序执行，总耗时 = 1+1+1 = 3s
print("\n--- 同步执行 ---")
start = time.time()
sync_task("A", 1)
sync_task("B", 1)
sync_task("C", 1)
print(f"同步总耗时: {time.time() - start:.1f}s")

# 异步：并发执行，总耗时 ≈ 1s
async def run_async():
    await asyncio.gather(
        async_task("A", 1),
        async_task("B", 1),
        async_task("C", 1),
    )

print("\n--- 异步执行 ---")
start = time.time()
asyncio.run(run_async())
print(f"异步总耗时: {time.time() - start:.1f}s")

# ========== 2. gather 收集返回值 ==========
print("\n" + "=" * 40)
print("2. gather 收集返回值")

async def return_num(n):
    await asyncio.sleep(0.1)
    return n * 10

async def main():
    results = await asyncio.gather(
        return_num(1), return_num(2), return_num(3),
        return_num(4), return_num(5),
    )
    print(f"结果: {results}")
    print(f"总和: {sum(results)}")

asyncio.run(main())

# ========== 3. create_task 并发 ==========
print("\n" + "=" * 40)
print("3. create_task 后台任务")

async def download(filename):
    print(f"  开始下载 {filename}")
    await asyncio.sleep(1)
    print(f"  完成下载 {filename}")
    return f"{filename} 的内容"

async def main2():
    # 同时启动两个下载
    task1 = asyncio.create_task(download("file1.txt"))
    task2 = asyncio.create_task(download("file2.txt"))

    print("主协程做其他事情...")
    await asyncio.sleep(0.5)
    print("等待下载完成...")

    r1 = await task1
    r2 = await task2
    print(f"全部完成: {r1}, {r2}")

asyncio.run(main2())

print("\n协程基础演示完成！")
```

## 5. 典型例题（附带答案）

### 例题 1

**题目：**

用 `gather` 并发执行 3 个协程，收集它们的返回值并求和。

**答案：**

```python
import asyncio

async def work(n):
    await asyncio.sleep(0.1)
    return n * 10

async def main():
    results = await asyncio.gather(work(1), work(2), work(3))
    print(f"总和: {sum(results)}")   # 60

asyncio.run(main())
```

### 例题 2

**题目：**

在一个 `gather` 调用中有多个协程，其中一个抛出异常。用 `return_exceptions=True` 观察效果。

**答案：**

```python
import asyncio

async def safe():
    return "success"

async def fail():
    raise ValueError("失败了")

async def main():
    # return_exceptions=True：异常作为结果返回
    results = await asyncio.gather(safe(), fail(), safe(), return_exceptions=True)
    print(results)  # ['success', ValueError('失败了'), 'success']

asyncio.run(main())
```

## 6. 易错点

### 易错点 1：在协程中使用 time.sleep()

错误：`time.sleep()` 阻塞整个事件循环，所有协程都被卡住。

正确：协程中使用 `await asyncio.sleep()`。

### 易错点 2：忘记 await

错误：调用协程函数但不 `await`，只返回协程对象而不执行。

正确：`await` 协程调用，或用 `create_task` 显式调度。

### 易错点 3：asyncio.run() 嵌套调用

错误：在已有事件循环内调用 `asyncio.run()`。

正确：内部使用 `await` 而非 `asyncio.run()`。

### 易错点 4：CPU 密集操作阻塞事件循环

错误：在协程中执行大量计算，阻塞整个事件循环。

正确：将 CPU 密集任务放到线程池：`await loop.run_in_executor(None, cpu_task)`。

## 8. 答题总结

- `async def` 定义协程函数，`await` 挂起等待可等待对象。
- `asyncio.run()` 是 Python 3.7+ 的入口函数，创建事件循环并执行协程。
- `asyncio.gather()` 并发运行多个协程，`asyncio.create_task()` 创建后台任务。
- 协程中必须用 `await asyncio.sleep()` 而非 `time.sleep()`。
- 协程适合 I/O 密集型高并发场景，比线程更轻量。
- 忘记 `await` 是最常见的错误，Python 3.11+ 会发出 RuntimeWarning 提醒。
