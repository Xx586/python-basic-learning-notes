# 深拷贝与浅拷贝

## 1. 知识点定义

Python 中对象的复制分为三种层级：**赋值（=）**、**浅拷贝（copy.copy）**、**深拷贝（copy.deepcopy）**。它们之间的区别在于复制了多少层。

> 小白理解：
> - 赋值 `=` = 给同一个盒子再贴一个标签（两个名字指向同一个盒子）
> - 浅拷贝 = 复制了外层盒子，但内层的盒子还是共享的
> - 深拷贝 = 把所有盒子（外层、内层、内内层...）全都复制了一遍

它主要解决的问题是：
- 复制一份数据来进行修改，同时保持原数据不变
- 理解为什么修改嵌套列表的内层会影响"副本"
- 在函数间传递配置数据时防止意外修改

## 2. 语法格式

### 2.1 赋值（=）—— 只是引用

```python
a = [1, 2, 3]
b = a              # b 和 a 指向同一个列表对象

b.append(4)
print(a)           # [1, 2, 3, 4] —— a 也被影响了！
print(a is b)      # True —— 同一个对象
```

### 2.2 浅拷贝 copy.copy() —— 只复制第一层

```python
import copy

# 一维列表：浅拷贝后看起来"独立"
a = [1, 2, 3]
b = copy.copy(a)
b.append(4)
print(a)           # [1, 2, 3] —— 不受影响（元素不可变）

# 嵌套列表：浅拷贝只复制第一层！
a = [[1, 2], [3, 4]]
b = copy.copy(a)
b[0].append(999)
print(a)           # [[1, 2, 999], [3, 4]] —— 内层共享，a 也被改了！

# 常见的浅拷贝操作（以下都是浅拷贝！）
b = a[:]           # 切片
b = list(a)        # list() 构造
b = a.copy()       # .copy() 方法
b = [*a]           # 解包
```

### 2.3 深拷贝 copy.deepcopy() —— 完全独立

```python
import copy

a = [[1, 2], [3, 4]]
b = copy.deepcopy(a)
b[0].append(999)
print(a)           # [[1, 2], [3, 4]] —— 不受影响！
print(b)           # [[1, 2, 999], [3, 4]]
```

### 2.4 对比总结

| 类型 | 赋值（=） | 浅拷贝 copy.copy() | 深拷贝 copy.deepcopy() |
|------|----------|-------------------|----------------------|
| 不可变对象 | 同一对象 | **同一对象** | 同一对象 |
| 可变顶层 | 同一对象 | 新对象 | 新对象 |
| 可变内层 | 同一对象 | **同一对象（共享！）** | 新对象 |

## 3. 重点说明

### 3.1 图解三种复制方式

```
赋值 (=):
a ──→ [1, 2, 3]  ←── b
       同一个对象，两个名称

浅拷贝 (copy.copy):
a ──→ [ □ , □ ]
        ↓    ↓
       [1,2] [3,4]     ←── 内层共享
        ↑    ↑
b ──→ [ □ , □ ]        ←── 外层是新的

深拷贝 (copy.deepcopy):
a ──→ [ □ , □ ]       [1,2]  ←── 完全独立
        ↓    ↓
       [1,2] [3,4]
                      [3,4]  ←── 完全独立
b ──→ [ □ , □ ]
        ↓    ↓
       [1,2] [3,4]
```

### 3.2 不可变类型的特殊行为

```python
import copy

# 不可变对象（int, str, tuple）：浅拷贝返回自身
a = (1, 2, 3)
b = copy.copy(a)
print(a is b)  # True！不可变对象浅拷贝返回自身

# 但嵌套不可变中带可变对象时，浅拷贝仍然共享内层
a = ([1, 2], [3, 4])    # 元组不可变，但包含可变列表
b = copy.copy(a)
print(a is b)            # True —— 元组本身返回自身
# 但 b[0] 和 a[0] 是同一个列表！修改 b[0] 会影响 a[0]
```

### 3.3 深拷贝处理循环引用

```python
import copy

a = []
b = [a]
a.append(b)      # a → b → a，形成循环引用

c = copy.deepcopy(a)    # 不会无限递归！memo 机制处理了循环
print(c)         # [[[...]]]
```

## 4. Demo 样例

```python
import copy

# ========== 1. 赋值 vs 浅拷贝 vs 深拷贝 ==========
print("=" * 40)
print("1. 赋值 vs 浅拷贝 vs 深拷贝")

original = [[1, 2], [3, 4]]
assigned = original               # 赋值
shallow = copy.copy(original)     # 浅拷贝
deep = copy.deepcopy(original)    # 深拷贝

# 修改内层元素
assigned[0][0] = 999

print(f"原始列表: {original}")    # [[999, 2], [3, 4]] —— 赋值影响了原列表
print(f"浅拷贝:   {shallow}")     # [[999, 2], [3, 4]] —— 内层共享，也被改了
print(f"深拷贝:   {deep}")        # [[1, 2], [3, 4]]   —— 完全独立

# ========== 2. id 对比 ==========
print("\n" + "=" * 40)
print("2. 内存地址（id）对比")

a = [[1, 2], [3, 4]]
b = a                   # 赋值
c = copy.copy(a)        # 浅拷贝
d = copy.deepcopy(a)    # 深拷贝

print(f"a 的外层 id:    {id(a)}")
print(f"b 的外层 id:    {id(b)}     (== a)")
print(f"c 的外层 id:    {id(c)}     (!= a, 新对象)")
print(f"d 的外层 id:    {id(d)}     (!= a, 新对象)")

print(f"\na[0] 的内层 id: {id(a[0])}")
print(f"c[0] 的内层 id: {id(c[0])}     (== a[0], 共享!)")
print(f"d[0] 的内层 id: {id(d[0])}     (!= a[0], 独立!)")

# ========== 3. 实际场景：配置防篡改 ==========

}

def make_request(user_config=None):

    config = copy.deepcopy(DEFAULT_CONFIG)    # 深拷贝保护默认值
    if user_config:
        config["options"].update(user_config.get("options", {}))
    return config

config1 = make_request({"options": {"timeout": 60}})

print(f"配置1:    {config1['options']}")          # timeout=60
print(f"配置2:    {config2['options']}")          # retry=5

# ========== 4. 浅拷贝的常见形式 ==========
print("\n" + "=" * 40)
print("4. 以下操作都是浅拷贝")

a = [[1, 2], [3, 4]]
b1 = a[:]          # 切片
b2 = list(a)       # list() 构造
b3 = a.copy()      # .copy() 方法
b4 = [*a]          # 解包

# 修改原列表的内层
a[0][0] = 999
for name, b in [("切片", b1), ("list()", b2), (".copy()", b3), ("解包", b4)]:
    print(f"{name}: {b}")  # 全都是 [[999, 2], [3, 4]]！
```

## 5. 典型例题（附带答案）

### 例题 1

**题目：**

解释为什么下面的代码中，修改 `b` 后 `a` 也变了？

```python
a = [[1, 2], [3, 4]]
b = a[:]
b[0][0] = 999
print(a[0][0])  # 999 —— 为什么 a 也被改了？
```

**答案：**

因为 `a[:]`（切片）是**浅拷贝**，只复制了外层列表。内层的 `[1, 2]` 和 `[3, 4]` 在两个列表中仍然是**同一个对象**。修改 `b[0][0]` 实际上修改的是共享的那个内层列表，所以 `a` 也会变。

**正确做法**（如果需要完全独立）：
```python
import copy
b = copy.deepcopy(a)
b[0][0] = 999
print(a[0][0])  # 1 —— 不受影响
```

### 例题 2

**题目：**

编写函数 `load_config(overrides)`，基于默认配置字典，用深拷贝合并用户覆盖项，保证多次调用互不影响。

**答案：**

```python
import copy

DEFAULTS = {
    "host": "0.0.0.0",
    "port": 8080,
    "cache": {"enabled": True, "ttl": 300}
}

def load_config(overrides=None):
    """基于默认配置创建新配置，不影响原默认值"""
    config = copy.deepcopy(DEFAULTS)    # 深拷贝保护默认值
    if overrides:
        for key, value in overrides.items():
            if isinstance(value, dict) and isinstance(config.get(key), dict):
                config[key].update(value)   # 嵌套字典：合并
            else:
                config[key] = value         # 普通值：覆盖
    return config

c1 = load_config({"cache": {"ttl": 600}})
c2 = load_config({"port": 9090})

print(DEFAULTS["cache"]["ttl"])   # 300 —— 默认值没被改
print(c1["cache"]["ttl"])         # 600 —— 配置 1 独立
print(c2["port"])                 # 9090 —— 配置 2 独立
```

## 6. 易错点

### 易错点 1：误以为 list[:] 是深拷贝

错误写法：
```python
a = [[1, 2], [3, 4]]
b = a[:]           # 以为完全独立，实际只是浅拷贝
b[0][0] = 999
print(a[0][0])     # 999 —— 原列表也被改了！
```

错误原因：切片、`list()`、`.copy()` 都是**浅拷贝**，只复制第一层。

正确写法：
```python
import copy
b = copy.deepcopy(a)   # 嵌套结构用深拷贝
```

### 易错点 2：不可变对象中包含可变元素

错误写法：
```python
import copy
a = (1, 2, [3, 4])     # 元组本身不可变，但包含可变列表
b = copy.copy(a)
print(a is b)           # True！以为复制了，实际还是同一个元组
b[2].append(5)
print(a)                # (1, 2, [3, 4, 5]) —— 还是被影响了！
```

错误原因：不可变对象的浅拷贝返回自身，但内部可变元素仍然共享。

正确写法：
```python
b = copy.deepcopy(a)    # 包含可变元素的不可变对象用深拷贝
```

### 易错点 3：函数默认参数用可变对象

错误写法：
```python
def add_item(item, lst=[]):    # 危险！默认参数在多次调用间共享
    lst.append(item)
    return lst

print(add_item(1))  # [1]
print(add_item(2))  # [1, 2] —— 不是 [2]！
```

错误原因：Python 的默认参数在函数定义时只创建一次，多次调用共享同一个列表。

正确写法：
```python
def add_item(item, lst=None):
    if lst is None:
        lst = []
    lst.append(item)
    return lst
```

## 7. 课后练习（附带答案）

### 练习 1

**题目：**

创建一个二维列表，分别做赋值、浅拷贝、深拷贝。打印三层 id（外层、`[0]`、`[1]`）并说明哪种方式产生了完全独立的对象。

**答案：**

```python
import copy

a = [[1, 2], [3, 4]]
b = a                   # 赋值
c = copy.copy(a)        # 浅拷贝
d = copy.deepcopy(a)    # 深拷贝

print("外层 id:", id(a), id(b), id(c), id(d))
# id(a) == id(b), id(c) 不同, id(d) 不同

print("a[0] id:", id(a[0]), id(c[0]), id(d[0]))
# id(a[0]) == id(c[0])（浅拷贝共享内层！）
# id(d[0]) 不同（深拷贝完全独立）

# 结论：赋值全共享 → 浅拷贝独立外层 → 深拷贝完全独立
```

### 练习 2

**题目：**

创建两个列表互相包含形成循环引用，验证 `deepcopy` 能正确处理而不会死循环。

**答案：**

```python
import copy

a = [1, 2]
b = [3, 4, a]
a.append(b)          # a → b → a，形成循环

c = copy.deepcopy(a)
print(c is a)        # False —— 是完全独立的新对象
print(c[2] is c)     # True —— 内部保持了循环引用关系

# deepcopy 通过 memo 字典追踪已复制的对象，正确处理循环引用
```

### 练习 3

**题目：**

对比赋值、浅拷贝、深拷贝对一个大列表（含 10000 个嵌套列表）的耗时。

**答案：**

```python
import copy
import time

data = [[i, i+1, i+2] for i in range(10000)]

# 赋值（最快）
start = time.perf_counter()
b = data
print(f"赋值耗时:   {time.perf_counter() - start:.6f}s")

# 浅拷贝
start = time.perf_counter()
c = copy.copy(data)
print(f"浅拷贝耗时: {time.perf_counter() - start:.6f}s")

# 深拷贝（最慢，递归复制所有内层）
start = time.perf_counter()
d = copy.deepcopy(data)
print(f"深拷贝耗时: {time.perf_counter() - start:.6f}s")
```

## 8. 答题总结

- 赋值 `=` 不创建新对象，只是新增一个引用（标签）。
- 浅拷贝 `copy.copy()` 创建新的外层容器，但内层元素仍是原对象的引用。
- 深拷贝 `copy.deepcopy()` 递归复制所有层级，生成完全独立的对象。
- 切片 `[:]`、`list()`、`.copy()`、`[*a]` 都是**浅拷贝**。
- 不可变类型的浅拷贝返回自身，但如果内部包含可变元素仍会共享。
- `deepcopy` 通过 `memo` 字典处理循环引用，不会无限递归。
- 修改嵌套结构时，务必想清楚用的是浅拷贝还是深拷贝。
