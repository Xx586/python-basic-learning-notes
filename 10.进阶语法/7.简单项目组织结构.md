# 简单项目组织结构

## 1. 知识点定义

**项目组织结构**是指 Python 项目中文件和目录的布局方式。一个清晰的项目结构能让代码易于维护、便于团队协作、方便打包发布。

> 小白理解：
> 就像整理房间——袜子放抽屉、衣服挂衣柜、书放书架。项目结构就是把代码分门别类放好，让人一眼就能找到想要的东西。

它主要解决的问题是：
- "所有代码放一个文件太长了，怎么拆分？"
- "别人拿到我的项目，怎么知道从哪开始看？"
- "怎么让项目看起来专业，而不是一堆文件乱堆？"

## 2. 语法格式

### 2.1 最小项目结构

```
my_project/
├── main.py              # 程序入口（或 app.py / run.py）
├── utils.py             # 工具函数
├── models.py            # 数据模型
├── config.py            # 配置常量
└── requirements.txt     # 依赖列表
```

### 2.2 标准项目结构

```
my_project/
├── src/                 # 源代码目录（或直接用包名）
│   ├── __init__.py
│   ├── main.py          # 入口文件
│   ├── config.py        # 配置
│   ├── models/          # 数据模型包
│   │   ├── __init__.py
│   │   └── user.py
│   ├── services/        # 业务逻辑包
│   │   ├── __init__.py
│   │   └── user_service.py
│   └── utils/           # 工具函数包
│       ├── __init__.py
│       └── helpers.py
├── tests/               # 测试目录
│   ├── __init__.py
│   └── test_user.py
├── docs/                # 文档
├── requirements.txt     # 项目依赖
├── README.md            # 项目说明
└── .gitignore           # Git 忽略文件
```

### 2.3 requirements.txt

```
# requirements.txt —— 列出项目依赖的第三方库

# 安装所有依赖
# pip install -r requirements.txt
```

## 3. 重点说明

### 3.1 项目结构的设计原则

| 原则 | 说明 |
|------|------|
| **按功能分层** | `models` 放数据、`services` 放业务逻辑、`utils` 放工具 |
| **一个文件一个职责** | 不要把所有函数塞一个 `utils.py` |
| **入口文件简洁** | `main.py` 只是调用入口，逻辑放到各模块 |
| **配置集中管理** | 数据库连接、API 密钥等放 `config.py` |
| **测试与源码分离** | `tests/` 目录独立，不与业务代码混在一起 |

### 3.2 `src/` 还是直接用包名？

```python
# 方式一：src 布局（推荐，避免导入混淆）
my_project/
├── src/
│   └── my_project/
│       ├── __init__.py
│       └── ...

# 方式二：平铺布局（简单项目）
my_project/
├── __init__.py
└── ...
```

### 3.3 入口文件写法

```python
# main.py —— 程序入口模板

def main():
    """主逻辑"""
    print("程序启动")
    # 初始化配置、连接数据库等

if __name__ == '__main__':
    main()
```

## 4. Demo 样例

下面演示如何从一个单文件脚本重构为有组织的项目结构。

### 重构前：单文件 `app.py`

```python
# app.py —— 所有代码塞一个文件（难以维护）

students = []

def add_student(name, score):
    students.append({"name": name, "score": score})

def get_average():
    return sum(s['score'] for s in students) / len(students)

def save_to_file():
    import json
    with open('data.json', 'w') as f:
        json.dump(students, f)

# 主程序
add_student("张三", 85)
add_student("李四", 90)
print(f"平均分: {get_average()}")
save_to_file()
```

### 重构后：有组织的项目

```
student_manager/
├── src/
│   ├── __init__.py
│   ├── main.py          # 入口
│   ├── models.py        # 数据模型
│   ├── services.py      # 业务逻辑
│   └── storage.py       # 文件存储
├── tests/
│   └── test_services.py
├── requirements.txt
└── README.md
```

```python
# ======== src/models.py ========
"""数据模型"""
students = []      # 全局学生列表（简单示例）

# ======== src/services.py ========
"""业务逻辑"""
from .models import students

def add_student(name: str, score: int) -> None:
    students.append({"name": name, "score": score})

def get_average() -> float:
    if not students:
        return 0.0
    return sum(s['score'] for s in students) / len(students)

# ======== src/storage.py ========
"""文件存储"""
import json
from .models import students

def save_to_file(filepath: str = 'data.json') -> None:
    with open(filepath, 'w', encoding='utf-8') as f:
        json.dump(students, f, ensure_ascii=False, indent=2)

# ======== src/main.py ========
"""程序入口"""
from .services import add_student, get_average
from .storage import save_to_file

def main():
    add_student("张三", 85)
    add_student("李四", 90)
    print(f"平均分: {get_average():.1f}")
    save_to_file()
    print("数据已保存到 data.json")

if __name__ == '__main__':
    main()
```

## 6. 易错点

### 易错点 1：所有代码塞一个文件

错误：把几百行代码全放 `main.py`，后期改一个功能要翻半天。

正确：按功能拆分到不同模块：`models.py`、`services.py`、`utils.py`。

### 易错点 2：包缺少 `__init__.py`

错误：创建了目录但忘记加 `__init__.py`，导致导入失败。

正确：每个包目录下都要有 `__init__.py`（即使是空文件）。

### 易错点 3：循环导入

错误：`a.py` 导入 `b.py`，`b.py` 又导入 `a.py`，形成循环依赖。

正确：提取公共模块，或使用延迟导入。

### 易错点 4：忘记 `.gitignore`

错误：`__pycache__`、`.pyc`、`venv/` 等文件被提交到 Git。

正确：创建 `.gitignore`，排除编译产物和虚拟环境。

## 8. 答题总结

- 好的项目结构按功能分模块，一个文件一个职责。
- 标准项目通常包含：`src/`（源码）、`tests/`（测试）、`docs/`（文档）。
- 每个包目录都需要 `__init__.py`。
- `requirements.txt` 记录项目依赖，方便他人复现环境。
- `.gitignore` 排除编译产物和虚拟环境，保持仓库干净。
- `main.py` 是程序入口，只负责调用各模块，不写具体逻辑。
- 项目从小做起，功能变多时逐步拆分，不要一开始就过度设计。
