# 类型注解

## 1. 知识点定义

**类型注解（Type Hints）** 是 Python 3.5+ 引入的语法，允许在代码中标注变量、函数参数和返回值的预期类型。类型注解**不影响程序运行**，但能提高代码可读性，让 IDE 提供更好的自动补全和错误检查。

> 小白理解：
> 类型注解就像快递包裹上的"易碎品"标签——贴不贴标签包裹都能寄到，但贴了标签快递员会更小心，收件人也知道里面是什么。Python 不会因为类型注解标错了就不运行，但它能帮你自己和团队更清楚地理解代码。

它主要解决的问题是：
- 大型项目中代码可读性差，"这个参数到底是 list 还是 dict？"
- IDE 无法推断类型，自动补全和错误提示不准确
- 团队协作时接口约定不清晰

## 2. 语法格式

### 2.1 基本类型注解

```python
# 变量注解（Python 3.6+）
name: str = "张三"
age: int = 25
score: float = 95.5
is_student: bool = True

# 函数参数和返回值注解
def greet(name: str) -> str:
    return f"你好, {name}!"

def add(a: int, b: int) -> int:
    return a + b

# 无返回值
def log_message(msg: str) -> None:
    print(msg)
```

### 2.2 typing 模块常用类型

```python
from typing import List, Dict, Tuple, Set, Optional, Union, Any

# 容器类型
names: List[str] = ["张三", "李四"]
scores: Dict[str, int] = {"张三": 85, "李四": 90}
point: Tuple[int, int] = (10, 20)
tags: Set[str] = {"Python", "学习"}

# Optional：可以是某类型或 None
def find_user(name: str) -> Optional[dict]:
    """找到返回字典，找不到返回 None"""
    pass

# Union：可以是多种类型之一
def process(data: Union[int, str]) -> str:
    return str(data)

# Any：任意类型（相当于没有类型限制）
from typing import Any
def flexible_func(x: Any) -> Any:
    return x
```

### 2.3 Python 3.9+ 简化语法

```python
# Python 3.9+ 可以直接用内置类型，不需要从 typing 导入
names: list[str] = ["张三", "李四"]       # 替代 List[str]
scores: dict[str, int] = {"a": 1}         # 替代 Dict[str, int]
point: tuple[int, int] = (10, 20)         # 替代 Tuple[int, int]

# Optional 也可以用 | None 替代（Python 3.10+）
def find_user(name: str) -> dict | None:
    pass
```

## 3. 重点说明

### 3.1 类型注解不影响运行

```python
# 这个函数注解为 int，但传入 str 也不会报错
def double(x: int) -> int:
    return x * 2

print(double("hello"))  # "hellohello" —— Python 照常运行！
```

> 类型注解只是"提示"，Python 解释器**不做类型检查**。真正的类型检查需要用 mypy 等工具。

### 3.2 常用类型速查表

| typing 写法 | 3.9+ 简化写法 | 含义 |
|------------|-------------|------|
| `List[str]` | `list[str]` | 字符串列表 |
| `Dict[str, int]` | `dict[str, int]` | 键为 str、值为 int 的字典 |
| `Optional[int]` | `int \| None` | 整数或 None |
| `Union[int, str]` | `int \| str` | 整数或字符串 |
| `Tuple[int, str]` | `tuple[int, str]` | 固定长度的元组 |
| `Callable[[int], str]` | — | 接收 int 返回 str 的函数 |
| `Any` | — | 任意类型 |

### 3.3 类型注解的最佳使用场景

| 场景 | 是否加注解 |
|------|:---:|
| 公共 API / 库函数 | ✅ 强烈推荐 |
| 大型项目的核心函数 | ✅ 推荐 |
| 临时脚本 / 快速原型 | ⚠️ 可选 |
| 所有内部函数 | ⚠️ 视情况而定 |

## 4. Demo 样例

```python
from typing import List, Dict, Optional

# ========== 1. 基本函数注解 ==========
print("=" * 40)
print("1. 基本函数注解")

def calculate_average(scores: List[int]) -> float:
    """计算平均分，参数是整数列表，返回浮点数"""
    return sum(scores) / len(scores)

scores: List[int] = [85, 92, 78, 90, 88]
avg: float = calculate_average(scores)
print(f"平均分: {avg:.1f}")

# ========== 2. Optional 类型 ==========
print("\n" + "=" * 40)
print("2. Optional 类型（可以是 None）")

def find_student(name: str, database: Dict[str, int]) -> Optional[int]:
    """查找学生分数，找到返回分数，找不到返回 None"""
    return database.get(name)   # get() 本身返回 Optional

db: Dict[str, int] = {"张三": 85, "李四": 92}

result = find_student("张三", db)
if result is not None:               # Optional 类型需要判空
    print(f"张三的成绩: {result}")

result = find_student("王五", db)
print(f"王五的成绩: {result}")        # None

# ========== 3. 类型注解不强制检查 ==========
print("\n" + "=" * 40)
print("3. 类型注解不影响运行")

def greet(name: str) -> str:
    """注解说是 str，但传入 int 也不会报错"""
    return f"你好, {name}!"

# 下面这行正常执行，不会报错
print(greet(123))   # "你好, 123!" —— Python 不管类型注解

print("\n类型注解演示完成！")
```

**运行结果：**

```
========================================
1. 基本函数注解
平均分: 86.6

========================================
2. Optional 类型（可以是 None）
张三的成绩: 85
王五的成绩: None

========================================
3. 类型注解不影响运行
你好, 123!

类型注解演示完成！
```

## 5. 典型例题（附带答案）

### 例题 1

**题目：**

给以下函数添加类型注解：

```python
def get_top_students(students, n):
    """返回分数最高的 n 个学生姓名"""
    sorted_students = sorted(students.items(), key=lambda x: x[1], reverse=True)
    return [name for name, _ in sorted_students[:n]]
```

**答案：**

```python
from typing import List, Dict, Tuple

def get_top_students(students: Dict[str, int], n: int) -> List[str]:
    """返回分数最高的 n 个学生姓名"""
    sorted_students = sorted(students.items(), key=lambda x: x[1], reverse=True)
    return [name for name, _ in sorted_students[:n]]
```

### 例题 2

**题目：**

定义一个函数 `safe_divide(a, b)`，返回 `a / b`，但 `b` 可能是 0（此时返回 None）。写出带类型注解的版本。

**答案：**

```python
from typing import Optional

def safe_divide(a: float, b: float) -> Optional[float]:
    """安全除法，除数为 0 时返回 None"""
    if b == 0:
        return None
    return a / b
```

## 6. 易错点

### 易错点 1：以为类型注解会强制检查

错误：写了 `def add(a: int, b: int) -> int`，然后传了字符串，期待 Python 报错。但 Python 不会报错——类型注解只是提示。

### 易错点 2：忘记 Optional 需要判空

错误：函数返回 `Optional[int]`，但调用后直接用 `result + 1`，当结果为 None 时报错。

正确：对 Optional 类型的值，使用前先判断 `if result is not None`。

### 易错点 3：Python 版本和语法兼容

`list[int]` 是 Python 3.9+ 才支持的，`int | None` 是 Python 3.10+ 才支持的。老项目可能需要用 `from typing import List, Optional`。

```python
def merge_dicts(d1, d2):
    """合并两个字典，d2 的值覆盖 d1"""
    result = d1.copy()
    result.update(d2)
    return result
```

**答案：** `def merge_dicts(d1: dict, d2: dict) -> dict:`

**答案：** `def parse_age(age_str: str) -> Optional[int]:`

### 练习 3

**答案：** `list[int | str]` 和 `dict[str, tuple[int, float]]`

## 8. 答题总结

- 类型注解标注变量、参数、返回值的预期类型，是给人和 IDE 看的"说明书"。
- Python 解释器**不检查**类型注解，程序照常运行。
- `typing` 模块提供 `List`、`Dict`、`Optional`、`Union` 等泛型类型。
- Python 3.9+ 支持 `list[int]`、`dict[str, int]` 等简化写法。
- `Optional[X]` = `X | None`（Python 3.10+），表示可以是指定类型或 None。
- 类型注解推荐用于公共 API 和大型项目核心函数，小脚本可选。
- 真正的类型检查需要工具如 mypy：`pip install mypy && mypy script.py`。
