# 多线程与多进程

## 1. 知识点定义

**多线程（Threading）** 和 **多进程（Multiprocessing）** 是 Python 中实现并发和并行的两种方式。多线程适合 I/O 密集型任务，多进程适合 CPU 密集型任务。

> 小白理解：
> - 多线程 = 一个厨师同时看着多个锅（锅在煮的时候可以去切菜）
> - 多进程 = 多个厨师各自独立做菜（每个人有自己的灶台和工具）
> - Python 的 GIL 就像"厨房里只有一把刀"——多线程时大家排队用刀，多进程时每个厨师有自己的刀。

它主要解决的问题是：
- 提高程序执行效率，充分利用多核 CPU
- 避免程序在等待 I/O 时白白浪费时间
- 处理大量并发请求（如 Web 服务器）

## 2. 语法格式

### 2.1 threading 多线程

```python
import threading
import time

def worker(name, delay):
    for i in range(3):
        print(f"[{name}] 第 {i+1} 次")
        time.sleep(delay)

# 创建线程
t1 = threading.Thread(target=worker, args=("A", 1))
t2 = threading.Thread(target=worker, args=("B", 0.5))

t1.start()     # 启动线程
t2.start()
t1.join()      # 等待线程结束
t2.join()
```

### 2.2 Lock 线程锁

```python
import threading

counter = 0
lock = threading.Lock()

def increment():
    global counter
    for _ in range(100000):
        with lock:            # 加锁保护共享变量
            counter += 1
```

### 2.3 ThreadPoolExecutor 线程池

```python
from concurrent.futures import ThreadPoolExecutor
import time

def download(url):
    time.sleep(1)  # 模拟下载
    return f"{url} done"

urls = ["url1", "url2", "url3", "url4", "url5"]

with ThreadPoolExecutor(max_workers=3) as executor:
    results = executor.map(download, urls)
    for result in results:
        print(result)
```

### 2.4 multiprocessing 多进程

```python
import multiprocessing

def cpu_task(n):
    """CPU 密集型任务"""
    return sum(i ** 2 for i in range(n))

# 创建进程
p = multiprocessing.Process(target=cpu_task, args=(10000000,))
p.start()
p.join()

# 进程池
from multiprocessing import Pool

with Pool(processes=4) as pool:
    results = pool.map(cpu_task, [1000000, 2000000, 3000000])
```

## 3. 重点说明

### 3.1 GIL（全局解释器锁）

GIL 是 CPython 中的互斥锁，保证同一时刻只有一个线程执行 Python 字节码。

| 任务类型 | 多线程效果 | 推荐方案 |
|---------|:---:|------|
| I/O 密集（网络请求、文件读写） | ✅ 有效 | 多线程 / asyncio |
| CPU 密集（数学计算、图像处理） | ❌ 反而更慢 | 多进程 |

### 3.2 多线程 vs 多进程选择

| 对比维度 | 多线程 | 多进程 |
|---------|--------|--------|
| 适用任务 | I/O 密集型 | CPU 密集型 |
| GIL 影响 | 受限制 | 不受限制（独立解释器） |
| 内存开销 | 小（共享内存） | 大（独立内存空间） |
| 数据共享 | 直接共享（需加锁） | 需要 IPC（Queue/Pipe） |
| 创建开销 | 小 | 大 |

## 4. Demo 样例

```python
import threading
import time
from concurrent.futures import ThreadPoolExecutor

# ========== 1. 多线程基本使用 ==========
print("=" * 40)
print("1. 多线程基本使用")

def worker(name, delay):
    for i in range(3):
        print(f"[{name}] 第 {i+1} 次")
        time.sleep(delay)

t1 = threading.Thread(target=worker, args=("A", 0.3))
t2 = threading.Thread(target=worker, args=("B", 0.2))

start = time.time()
t1.start()
t2.start()
t1.join()
t2.join()
print(f"总耗时: {time.time() - start:.1f}s")

# ========== 2. 线程锁 ==========
print("\n" + "=" * 40)
print("2. 线程锁保护共享变量")

counter = 0
lock = threading.Lock()

def safe_increment():
    global counter
    for _ in range(100000):
        with lock:           # 加锁
            counter += 1

threads = [threading.Thread(target=safe_increment) for _ in range(5)]
for t in threads:
    t.start()
for t in threads:
    t.join()
print(f"加锁后计数: {counter}")   # 始终是 500000

# ========== 3. 线程池 ==========
print("\n" + "=" * 40)
print("3. 线程池并发下载")

def download(url):
    print(f"  开始下载: {url}")
    time.sleep(0.5)          # 模拟网络延迟
    print(f"  完成下载: {url}")
    return f"{url} 的内容"

urls = [f"file_{i}.txt" for i in range(6)]
start = time.time()

with ThreadPoolExecutor(max_workers=3) as executor:
    results = list(executor.map(download, urls))

print(f"全部下载完成，耗时: {time.time() - start:.1f}s")
print(f"结果: {len(results)} 个文件")

print("\n多线程演示完成！")
```

## 5. 典型例题（附带答案）

### 例题 1

**题目：**

用 `ThreadPoolExecutor` 模拟同时下载 10 个 URL（用 `time.sleep()` 模拟），统计总耗时。

**答案：**

```python
import time
from concurrent.futures import ThreadPoolExecutor

def download(url):
    time.sleep(1)  # 模拟下载
    return f"{url} done"

start = time.time()
with ThreadPoolExecutor(max_workers=3) as pool:
    results = list(pool.map(download, [f"url_{i}" for i in range(10)]))

elapsed = time.time() - start
print(f"总耗时: {elapsed:.1f}s")   # 10个URL / 3并发 ≈ 4秒
```

### 例题 2

**题目：**

判断以下场景该用多线程还是多进程：
- a) 压缩 100 个图片文件
- b) 并发请求 100 个 API
- c) 大文件逐行读写处理

**答案：**

```
a) 压缩 100 个图片 → 多进程（CPU 密集，压缩涉及大量计算）
b) 并发请求 100 个 API → 多线程（I/O 密集，大部分时间在等待网络）
c) 大文件逐行读写 → 单线程即可（磁盘 I/O 是串行瓶颈）
```

## 6. 易错点

### 易错点 1：多线程修改共享变量不加锁

错误：多线程同时 `counter += 1`（非原子操作），导致计数偏小。

正确：使用 `threading.Lock()` 保护临界区。

### 易错点 2：Windows 多进程没放 `if __name__ == "__main__"` 里

错误：Windows 下子进程会重新导入模块，导致无限递归。

正确：始终将 `Process` 启动代码放在 `if __name__ == "__main__":` 后面。

### 易错点 3：线程中异常主线程感知不到

错误：线程崩溃了但主线程正常结束，没有任何报错。

正确：使用 `ThreadPoolExecutor` + `future.result()` 捕获异常，或在线程 `run()` 中记录日志。

## 8. 答题总结

- Python 多线程受 GIL 限制，适合 I/O 密集任务；多进程不受 GIL 限制，适合 CPU 密集任务。
- `threading.Thread` 创建线程，`start()` 启动，`join()` 等待结束。
- `Lock` 保护共享变量，`ThreadPoolExecutor` 管理线程池。
- Windows 上多进程代码必须放在 `if __name__ == "__main__":` 中。
- 线程异常不会自动传播，推荐用 `ThreadPoolExecutor` 捕获。
