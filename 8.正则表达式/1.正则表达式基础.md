# 正则表达式基础语法

## 1. 知识点定义

**正则表达式（Regular Expression，简称 regex）** 是一种描述字符串匹配模式的工具。它不是 Python 独有的——几乎所有编程语言都支持正则表达式。

> 小白理解：
> 正则表达式就像"文字搜索的超级增强版"。平时你在 Word 里按 Ctrl+F 搜索固定文字，正则表达式可以搜索"符合某种规则"的文字——比如搜出所有手机号、搜出所有邮箱地址、搜出所有以"尊敬的"开头的段落。

它主要解决的问题是：
- **验证**：检查字符串是否符合特定格式（手机号、邮箱、身份证等）
- **提取**：从文本中抓取感兴趣的部分（URL、数值、日期）
- **替换**：将匹配到的内容替换成其他文本
- **分割**：按模式拆分字符串

Python 通过内置模块 `re` 提供正则表达式支持，使用 `import re` 即可。

## 2. 语法格式

### 2.1 元字符速查表

正则表达式中以下字符具有特殊含义，称为**元字符**：

| 元字符 | 含义 | 示例 |
|--------|------|------|
| `.` | 匹配任意**单个**字符（除换行符 `\n` 外） | `a.c` 匹配 `"abc"`、`"a1c"` |
| `^` | 匹配字符串的**开头** | `^Hello` 匹配以 `"Hello"` 开头的字符串 |
| `$` | 匹配字符串的**结尾** | `world$` 匹配以 `"world"` 结尾的字符串 |
| `*` | 前面的字符重复 **0 次或多次** | `ab*c` 匹配 `"ac"`、`"abc"`、`"abbc"` |
| `+` | 前面的字符重复 **1 次或多次** | `ab+c` 匹配 `"abc"`、`"abbc"`，**不**匹配 `"ac"` |
| `?` | 前面的字符重复 **0 次或 1 次** | `ab?c` 匹配 `"ac"`、`"abc"` |
| `{n}` | 前面的字符重复**恰好 n 次** | `a{3}` 匹配 `"aaa"` |
| `{n,}` | 前面的字符重复**至少 n 次** | `a{2,}` 匹配 `"aa"`、`"aaa"`... |
| `{n,m}` | 前面的字符重复 **n ~ m 次** | `a{2,4}` 匹配 `"aa"`、`"aaa"`、`"aaaa"` |
| `[]` | 字符集：匹配括号内**任意一个**字符 | `[aeiou]` 匹配任意一个元音字母 |
| `[^...]` | 否定字符集：匹配**不在**括号内的字符 | `[^0-9]` 匹配任意一个非数字字符 |
| `\` | 转义字符 | `\.` 匹配字面量句点，`\d` 匹配数字 |
| `\|` | 或：匹配左侧或右侧表达式 | `cat\|dog` 匹配 `"cat"` 或 `"dog"` |
| `()` | 分组：将表达式括起来视为一个单元 | `(ab)+` 匹配 `"ab"`、`"abab"` |

### 2.2 常用简写字符类

| 简写 | 等价于 | 含义 |
|------|--------|------|
| `\d` | `[0-9]` | 任意一个数字 |
| `\D` | `[^0-9]` | 任意一个非数字 |
| `\w` | `[a-zA-Z0-9_]` | 任意一个"单词字符"（字母、数字、下划线） |
| `\W` | `[^a-zA-Z0-9_]` | 任意一个非单词字符 |
| `\s` | `[ \t\n\r\f\v]` | 任意一个空白字符（空格、tab、换行等） |
| `\S` | `[^ \t\n\r\f\v]` | 任意一个非空白字符 |
| `\b` | — | 单词边界 |
| `\B` | — | 非单词边界 |

### 2.3 分组语法

```python
# 基本捕获分组
(pattern)         # 捕获匹配的内容，可通过 group() 提取

# 非捕获分组
(?:pattern)       # 只分组不捕获，不占用 group() 编号

# 命名分组
(?P<name>pattern) # 给分组起名字，通过 group('name') 访问

# 反向引用
\1                # 引用第一个分组捕获到的内容
```

### 2.4 锚点（匹配位置，不匹配字符）

| 锚点 | 含义 |
|------|------|
| `^` | 字符串 / 行的开头 |
| `$` | 字符串 / 行的结尾 |
| `\b` | 单词边界 |
| `\B` | 非单词边界 |

## 3. 重点说明

### 3.1 *, +, ? 三大基础量词对比

| 量词 | 含义 | `ab*c` | `ab+c` | `ab?c` |
|------|------|--------|--------|--------|
| `*` | 0次或多次 | 匹配 `ac`, `abc`, `abbc` | — | — |
| `+` | 1次或多次 | — | 匹配 `abc`, `abbc` | — |
| `?` | 0次或1次 | — | — | 匹配 `ac`, `abc` |

> 记忆口诀：`*` 像星星（可以没有也可以很多），`+` 像加号（至少要有一个），`?` 像问号（有还是没有？）。

### 3.2 . 和 \. 的区别

```python
# . 是元字符，匹配任意字符
re.findall(r"a.c", "abc a1c a c")    # ['abc', 'a1c', 'a c']

# \. 是转义后的字面量句点
re.findall(r"a\.c", "a.c a1c")       # ['a.c'] —— 只匹配字面量的 "a.c"
```

> 在 `[]` 字符集内部，`.` 自动失去特殊含义，表示字面量句点。

### 3.3 字符集内的注意事项

```python
# [] 内部大多数字符不需要转义
re.findall(r"[.]", "a.b.c")          # ['.', '.'] —— . 在 [] 内是字面量
re.findall(r"[*+]", "a*b+c")         # ['*', '+'] —— 元字符在 [] 内失去特殊含义

# - 放在开头或结尾避免歧义
re.findall(r"[-a-z]", "hello-world") # 匹配 - 或 a-z 范围内的字符

# ^ 在 [] 开头表示"取反"
re.findall(r"[^0-9]", "abc123")      # ['a', 'b', 'c']
```

### 3.4 捕获分组 vs 非捕获分组

| 对比项 | `(pattern)` | `(?:pattern)` |
|-------|------------|---------------|
| 分组给量词用 | ✅ | ✅ |
| 可通过 group() 提取 | ✅ | ❌ |
| 占用分组编号 | ✅ | ❌ |
| 使用场景 | 需要提取内容时 | 只需要分组、不需要提取时 |

## 4. Demo 样例

```python
import re

# ========== 1. 基础元字符演示 ==========
print("=" * 40)
print("1. 基础元字符")

# . 匹配任意单个字符（除换行符）
print(re.findall(r"a.c", "abc a1c a c a@c"))
# ['abc', 'a1c', 'a c', 'a@c']

# * 0次或多次
print(re.findall(r"ab*c", "ac abc abbc abbbc"))
# ['ac', 'abc', 'abbc', 'abbbc']

# + 1次或多次
print(re.findall(r"ab+c", "ac abc abbc abbbc"))
# ['abc', 'abbc', 'abbbc']  —— "ac" 不匹配

# ? 0次或1次
print(re.findall(r"ab?c", "ac abc abbc"))
# ['ac', 'abc']  —— "abbc" 不匹配

# {n,m} 精确控制次数
print(re.findall(r"a{2,4}", "a aa aaa aaaa aaaaa"))
# ['aa', 'aaa', 'aaaa', 'aaaa']

# ========== 2. 简写字符类 ==========
print("\n" + "=" * 40)
print("2. 简写字符类")

text = "我的手机号是 13812345678，价格是 99.9 元。"

# \d 匹配数字
print(f"所有数字: {re.findall(r'\d+', text)}")
# ['13812345678', '99', '9']

# \w 匹配单词字符
print(re.findall(r"\w+", "hello_world 你好123 abc"))
# ['hello_world', '你好123', 'abc']

# \s 匹配空白
print(re.findall(r"\S+", "hello world\tfoo\nbar"))
# ['hello', 'world', 'foo', 'bar']

# ========== 3. 字符集 [] ==========
print("\n" + "=" * 40)
print("3. 字符集 []")

# 匹配元音字母
print(re.findall(r"[aeiou]", "apple banana cherry"))
# ['a', 'a', 'a', 'a', 'e']

# 范围表示
print(re.findall(r"[a-z]", "Hello World 123"))
# ['e', 'l', 'l', 'o', 'o', 'r', 'l', 'd']

print(re.findall(r"[0-9]", "Room 305, Floor 12"))
# ['3', '0', '5', '1', '2']

# ========== 4. 分组和或 ==========
print("\n" + "=" * 40)
print("4. 分组和捕获")

# 捕获分组：提取姓名和年龄
m = re.search(r"姓名：(\w+)，年龄：(\d+)", "姓名：张三，年龄：28")
if m:
    print(f"完整匹配: {m.group(0)}")  # 姓名：张三，年龄：28
    print(f"姓名: {m.group(1)}")      # 张三
    print(f"年龄: {m.group(2)}")      # 28

# | 或运算
print(re.findall(r"cat|dog|bird", "I have a cat and a dog"))
# ['cat', 'dog']

# ========== 5. 锚点 ^ 和 $ ==========
print("\n" + "=" * 40)
print("5. 锚点 ^ 和 $")

# ^...$ 限定整串匹配（常用于验证）
print(re.findall(r"^\d+$", "12345"))  # ['12345'] —— 整串是数字
print(re.findall(r"^\d+$", "123abc")) # [] —— 包含字母，不匹配

# \b 单词边界
print(re.findall(r"\bcat\b", "cat category cat. scat"))
# ['cat', 'cat'] —— 只匹配独立单词 "cat"

print("\n正则表达式基础语法演示完成！")
```

## 5. 典型例题（附带答案）

### 例题 1

**题目：**

写出匹配以下要求的正则表达式：
1. 匹配以 1 开头的 11 位数字（手机号）
2. 匹配 6 位数字验证码
3. 匹配中文姓名（2~4 个汉字）

**答案：**

```python
import re

# 1. 手机号：1开头，第二位3-9，共11位
pattern_phone = r"^1[3-9]\d{9}$"
print(re.match(pattern_phone, "13812345678"))   # ✅ Match
print(re.match(pattern_phone, "12812345678"))   # ❌ None

# 2. 6位验证码：恰好6个数字
pattern_code = r"^\d{6}$"
print(re.match(pattern_code, "123456"))         # ✅ Match
print(re.match(pattern_code, "12345"))          # ❌ None

# 3. 中文姓名：2~4个汉字
pattern_name = r"^[一-龥]{2,4}$"
print(re.match(pattern_name, "张三"))           # ✅ Match
print(re.match(pattern_name, "张三丰"))         # ✅ Match
print(re.match(pattern_name, "张三李四王五"))   # ❌ 5个字
```

**解析：** `\d{9}` 表示恰好 9 个数字，`[一-龥]` 是汉字的 Unicode 范围，`{2,4}` 限制 2~4 个。

### 例题 2

**题目：**

从字符串 `"联系方式：张三 13811112222，李四 13922223333"` 中提取所有人的姓名和手机号。

**答案：**

```python
import re

text = "联系方式：张三 13811112222，李四 13922223333"

# 用捕获分组提取姓名和手机号
pattern = r"([一-龥]+)\s+(1[3-9]\d{9})"
matches = re.findall(pattern, text)

for name, phone in matches:
    print(f"姓名: {name}, 手机号: {phone}")

# 输出:
# 姓名: 张三, 手机号: 13811112222
# 姓名: 李四, 手机号: 13922223333
```

**解析：** 两个捕获分组分别匹配姓名和手机号。`findall()` 在有捕获分组时返回元组列表，每个元组对应一组捕获。

## 6. 易错点

### 易错点 1：忘记转义元字符

错误写法：
```python
# 想匹配字面量的 . 号，但 . 是元字符，匹配了任意字符
re.findall(r"3.14", "3.14 3a14 3b14")
# ['3.14', '3a14', '3b14']  —— 全部匹配了！
```

错误原因：`.` 是元字符，匹配除换行外的任意字符。

正确写法：
```python
# 转义 . 为字面量句点
re.findall(r"3\.14", "3.14 3a14 3b14")
# ['3.14']  —— 只匹配字面量 "3.14"
```

### 易错点 2：混淆 match 和 search 的行为

错误写法：
```python
# 想从字符串中搜索手机号，用了 match
re.match(r"\d+", "我的手机号是 13812345678")  # None!
```

错误原因：`re.match()` 只从字符串**开头**匹配，开头不是数字就直接返回 None。

正确写法：
```python
# 搜索提取用 search 或 findall
re.search(r"\d+", "我的手机号是 13812345678").group()  # '13812345678'
```

### 易错点 3：量词默认贪婪匹配

错误写法：
```python
# 想提取两个 div 标签各自的内容，却得到了一个整体
html = "<div>内容1</div><div>内容2</div>"
re.findall(r"<div>.*</div>", html)
# ['<div>内容1</div><div>内容2</div>']  —— 贪婪！两个 div 被当成一个了
```

错误原因：`*` 默认贪婪，尽可能多地匹配。

正确写法：
```python
# 用非贪婪 .*? 匹配
re.findall(r"<div>.*?</div>", html)
# ['<div>内容1</div>', '<div>内容2</div>']  —— 正确！
```

### 易错点 4：[] 内的 ^ 不是"开头"

错误写法：
```python
# 以为 [^abc] 是"以 abc 开头的"，实际是"不在 abc 中的字符"
```

错误原因：`^` 在 `[]` 内部表示**取反**，在 `[]` 外部才表示**字符串开头**。

正确理解：
```python
re.findall(r"^[abc]", "abc")     # ['a'] —— ^ 在 [] 外，匹配以 a 或 b 或 c 开头
re.findall(r"[^abc]", "abcdef")  # ['d', 'e', 'f'] —— ^ 在 [] 内，匹配不是 a/b/c 的字符
```

## 7. 课后练习（附带答案）

### 练习 1

**题目：**

写出正则表达式匹配以下内容：
- a) 以 `www.` 开头的网址域名
- b) 18 位身份证号（最后一位可以是数字或 X）
- c) 恰好 4 位的数字验证码

**答案：**

```python
import re

# a) 以 www. 开头的域名
pattern_a = r"^www\.[a-zA-Z0-9-]+\.[a-zA-Z]{2,}$"

# b) 18位身份证号
pattern_b = r"^\d{17}[\dXx]$"

# c) 4位数字验证码
pattern_c = r"^\d{4}$"

# 测试
print(re.match(pattern_a, "www.example.com"))     # ✅
print(re.match(pattern_b, "110101199001011234"))  # ✅
print(re.match(pattern_c, "5678"))                # ✅
```

### 练习 2

**题目：**

从文本 `"价格：苹果5元/斤，香蕉3.5元/斤，西瓜2元/斤"` 中提取所有水果名称和对应价格。

**答案：**

```python
import re

text = "价格：苹果5元/斤，香蕉3.5元/斤，西瓜2元/斤"

# 用捕获分组提取水果名和价格
pattern = r"([一-龥]+?)(\d+\.?\d*)元"
matches = re.findall(pattern, text)

for fruit, price in matches:
    print(f"{fruit}: {price}元")

# 输出:
# 苹果: 5元
# 香蕉: 3.5元
# 西瓜: 2元
```

**说明：** `\d+\.?\d*` 匹配整数或小数，`\.?` 表示小数点可以出现 0 次或 1 次。

### 练习 3

**题目：**

编写函数 `is_valid_password(s)`，验证密码必须同时包含字母和数字，且长度在 6-16 位之间。

**答案：**

```python
import re

def is_valid_password(s):
    """验证密码：6-16位，必须同时包含字母和数字"""
    # 方案：用两个前瞻断言分别检查是否包含字母和数字
    pattern = r"^(?=.*[a-zA-Z])(?=.*\d)[a-zA-Z\d]{6,16}$"
    return bool(re.match(pattern, s))

print(is_valid_password("abc123"))      # True
print(is_valid_password("abcdef"))      # False —— 没有数字
print(is_valid_password("123456"))      # False —— 没有字母
print(is_valid_password("ab12"))        # False —— 长度不够
print(is_valid_password("abc 123"))     # False —— 含空格
```

**说明：** `(?=.*[a-zA-Z])` 是正向前瞻断言，检查"后面某处有字母"，但不消耗字符。

## 8. 答题总结

- 正则表达式是一种描述字符串匹配模式的"微型语言"，用于验证、提取、替换和分割文本。
- 元字符 `. * + ? { } [ ] ( ) ^ $ \ |` 各有特殊含义，匹配字面量时需要 `\` 转义。
- `*` = 0次或多次，`+` = 1次或多次，`?` = 0次或1次，`{n,m}` = n~m次。
- `\d` = 数字，`\w` = 单词字符，`\s` = 空白字符，大写表示"取反"。
- `[abc]` 匹配字符集中的任意一个，`[^abc]` 匹配不在字符集中的任意一个。
- `()` 用于分组和捕获，`(?:)` 只分组不捕获。
- `^` 匹配开头，`$` 匹配结尾，`\b` 匹配单词边界。
- 量词默认**贪婪**匹配（尽可能多），加 `?` 变**非贪婪**（尽可能少）：`.*?`、`\d+?`。
