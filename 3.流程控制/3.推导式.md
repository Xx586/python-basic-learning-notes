# 推导式

## 1. 知识点定义

**推导式（Comprehension）** 是 Python 中一种**简洁构建新序列**的语法，可以在一行内完成"遍历 + 过滤 + 变换"的操作。

Python 支持四种类型：

| 类型 | 语法 | 结果 | 惰性 |
|------|------|------|:--:|
| 列表推导式 | `[expr for x in iterable if cond]` | list | 否 |
| 字典推导式 | `{k: v for x in iterable}` | dict | 否 |
| 集合推导式 | `{expr for x in iterable}` | set | 否 |
| 生成器表达式 | `(expr for x in iterable)` | generator | ✅ |

> 小白理解：
> 传统方式写循环要 3 行（创建空列表 → for 循环 → append），推导式一行搞定。就像用料理机一次性把食材打成汁，而不是一刀一刀慢慢切。

## 2. 语法格式

### 2.1 列表推导式

```python
# 基本语法：[表达式 for 变量 in 可迭代对象 if 条件]
squares = [x ** 2 for x in range(1, 6)]          # [1, 4, 9, 16, 25]
evens = [x for x in range(1, 11) if x % 2 == 0]  # [2, 4, 6, 8, 10]

# if-else 放在 for 前（变换），单独 if 放在 for 后（过滤）
nums = [1, 2, 3, 4, 5, 6]
result = [x * 10 if x % 2 == 0 else x for x in nums]  # 变换
result2 = [x * 10 for x in nums if x % 2 == 0]         # 过滤

# 多层循环
pairs = [(x, y) for x in range(3) for y in range(2)]
# → [(0,0), (0,1), (1,0), (1,1), (2,0), (2,1)]
```

### 2.2 字典推导式

```python
# 语法：{键表达式: 值表达式 for 变量 in 可迭代对象 if 条件}
word_len = {w: len(w) for w in ["apple", "banana"]}  # {'apple': 5, 'banana': 6}
squares = {x: x ** 2 for x in range(1, 4)}           # {1: 1, 2: 4, 3: 9}

# 筛选和变换
scores = {"张三": 85, "李四": 55, "王五": 92}
passed = {name: score for name, score in scores.items() if score >= 60}
# {'张三': 85, '王五': 92}
```

### 2.3 集合推导式

```python
# 语法：{表达式 for 变量 in 可迭代对象 if 条件}  — 自动去重
abs_set = {abs(x) for x in [1, -1, 2, -2, 3, -3]}  # {1, 2, 3}
```

### 2.4 生成器表达式

```python
# 语法：(表达式 for 变量 in 可迭代对象 if 条件)  — 惰性求值
gen = (x ** 2 for x in range(5))
print(next(gen))  # 0（按需计算）
print(list(gen))  # [1, 4, 9, 16]

# 作为函数唯一参数时可省略外层括号
total = sum(x ** 2 for x in range(1, 6))  # 55
```

## 3. 重点说明

### 3.1 推导式 vs 传统 for 循环

```python
# 传统方式：3 行
result = []
for x in range(10):
    if x > 3:
        result.append(x * 2)

# 推导式：1 行
result = [x * 2 for x in range(10) if x > 3]
```

### 3.2 生成器 vs 列表推导式

| 对比 | 列表推导式 `[...]` | 生成器表达式 `(...)` |
|------|-------------------|---------------------|
| 内存 | 一次生成全部，占内存多 | 逐个生成，占内存极少 |
| 访问 | 可多次遍历、索引、切片 | **只能遍历一次** |
| 适用 | 需要多次使用、数据量小 | 一次遍历、数据量大 |

### 3.3 可读性优先原则

```python
# ✅ 适合推导式：简单的一步变换
names = [user.name for user in users]

# ❌ 不适合推导式：逻辑太复杂
# 嵌套超过 2 层、有副作用操作（print）、需要异常处理 → 用 for 循环
```

## 4. Demo 样例

```python
# ===== 1. 列表推导式 =====
# 生成平方数
squares = [x ** 2 for x in range(1, 6)]
print(f"平方数: {squares}")  # [1, 4, 9, 16, 25]

# 过滤偶数
evens = [x for x in range(1, 11) if x % 2 == 0]
print(f"偶数: {evens}")  # [2, 4, 6, 8, 10]

# ===== 2. 字典推导式 =====
words = ["apple", "banana", "cherry"]
word_len = {w: len(w) for w in words}
print(f"单词长度: {word_len}")  # {'apple': 5, 'banana': 6, 'cherry': 6}

# ===== 3. 集合推导式 =====
nums = [1, 2, 2, 3, 3, 3]
unique_squares = {x ** 2 for x in nums}
print(f"去重平方: {unique_squares}")  # {1, 4, 9}

# ===== 4. 生成器表达式 =====
gen = (x ** 2 for x in range(5))
print(f"生成器第一个值: {next(gen)}")  # 0
print(f"剩余转为列表: {list(gen)}")   # [1, 4, 9, 16]
# 生成器已耗尽
print(f"再次遍历: {list(gen)}")  # []

# 作为函数参数直接使用
total = sum(x for x in range(1, 101) if x % 2 == 0)
print(f"1-100偶数之和: {total}")  # 2550
```

运行结果：

```
平方数: [1, 4, 9, 16, 25]
偶数: [2, 4, 6, 8, 10]
单词长度: {'apple': 5, 'banana': 6, 'cherry': 6}
去重平方: {1, 4, 9}
生成器第一个值: 0
剩余转为列表: [1, 4, 9, 16]
再次遍历: []
1-100偶数之和: 2550
```

## 5. 典型例题（附带答案）

### 例题 1

题目：

用列表推导式生成 1-20 中所有能被 3 整除的数的平方。

答案：

```python
result = [x ** 2 for x in range(1, 21) if x % 3 == 0]
print(result)  # [9, 36, 81, 144, 225, 324]
```

### 例题 2

题目：

有学生成绩列表，用字典推导式生成 `{姓名: 平均分}`，只保留平均分 ≥ 70 的学生。

```python
students = [
    {"name": "张三", "math": 85, "english": 78},
    {"name": "李四", "math": 55, "english": 90},
]
```

答案：

```python
avg_scores = {
    s["name"]: (s["math"] + s["english"]) / 2
    for s in students
    if (s["math"] + s["english"]) / 2 >= 70
}
print(avg_scores)  # {'张三': 81.5}
```

## 6. 易错点

### 易错点 1：生成器只能遍历一次

错误现象：`gen = (x**2 for x in range(5))`；`sum(gen)` 返回 30，再次 `sum(gen)` 返回 0。

错误原因：生成器遍历完后"耗尽"，无法再次使用。

正确做法：需要多次使用时转为列表 `[x**2 for x in range(5)]`。

### 易错点 2：if-else 位置混淆

- `if-else` 在 for **前面**：三元变换（对每个元素做条件变换）
- `if` 在 for **后面**：过滤（只保留满足条件的元素）

```python
[x * 10 if x % 2 == 0 else x for x in nums]  # 变换
[x * 10 for x in nums if x % 2 == 0]          # 过滤
```

### 易错点 3：字典推导式键重复导致覆盖

```python
pairs = [("a", 1), ("b", 2), ("a", 100)]
d = {k: v for k, v in pairs}
print(d)  # {'a': 100, 'b': 2}  —— 第一个 "a": 1 被覆盖
```

### 易错点 4：多层嵌套超过 2 层

超过 2 层嵌套的推导式极难阅读，应改用传统 for 循环。

### 易错点 5：推导式中做副作用操作

```python
[print(x) for x in range(5)]  # 滥用推导式！用 for 循环
```


### 练习 2

题目：

给定 `names = ["Alice", "Bob", "Charlie"]`，用字典推导式生成 `{名字: 长度}`，只保留长度 > 3 的名字。

答案：


```python

```

## 8. 答题总结

- 推导式一行完成"遍历+过滤+变换"，代码更简洁、更 Pythonic。
- 四种类型：`[...]`列表、`{k:v ...}`字典、`{...}`集合、`(...)`生成器表达式。
- `if-else`在 for 前（变换），`if`在 for 后（过滤）。
- 生成器表达式**惰性求值**，省内存，但**只能遍历一次**。
- 生成器作函数唯一参数时可省略外层括号：`sum(x**2 for x in data)`。
- 推导式不适合：逻辑太复杂、多层嵌套 > 2 层、有副作用操作、需要异常处理。
- **可读性优先**：如果一行推导式需要停下来思考才能理解，拆成 for 循环。
