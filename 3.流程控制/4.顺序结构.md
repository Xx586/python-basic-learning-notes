# 顺序结构

## 1. 知识点定义

**顺序结构**是程序中最基本、最自然的执行方式：代码**从上到下、逐行执行**，每一行执行完再执行下一行。这是程序默认的运行方式。

它主要解决的问题是：
- 定义程序的基本执行流程
- 把解决问题的步骤按先后顺序排列
- 作为理解其他流程控制（条件判断、循环）的基础

在实际写 Python 代码时，你写的每一段代码默认都是按顺序执行的。只有遇到 `if`、`for`、`while` 等语句时，才会改变执行顺序。

> 小白理解：
> 顺序结构就像菜谱上的步骤：第 1 步洗菜 → 第 2 步切菜 → 第 3 步炒菜 → 第 4 步装盘。你不会先装盘再洗菜，一定是按顺序来的。程序也是同理，从上到下一行一行执行。

---

## 2. 语法格式

```python
# 顺序结构没有特殊语法，就是按顺序写代码
语句1    # 先执行
语句2    # 再执行
语句3    # 最后执行
```

说明：Python 解释器默认按从上到下的顺序执行每行代码，不需要任何特殊的关键字。

```python
# 示例：一个简单的顺序结构程序
print("第一步：开始程序")     # ① 最先执行
name = input("请输入名字：")  # ② 等待用户输入
print("你好，", name)         # ③ 输出问候语
print("程序结束")             # ④ 最后执行
```

---

## 3. 重点说明

### 3.1 程序的三种基本结构

任何程序都由以下三种结构组合而成：

| 结构 | 说明 | 关键字 | 生活类比 |
|------|------|--------|----------|
| **顺序结构** | 从上到下逐行执行 | 无（默认） | 菜谱步骤 |
| **条件分支（选择结构）** | 根据条件选择执行 | `if`/`elif`/`else` | 如果下雨带伞，否则不带 |
| **循环结构** | 反复执行一段代码 | `for`/`while` | 吃饭：一口一口直到吃完 |

### 3.2 顺序结构是一切的基础

不管条件判断还是循环，**在满足条件时，执行的代码块内部仍然是顺序结构**：

```python
if score >= 60:          # 条件判断
    print("及格")         # ← 这两行在条件块内部
    print("恭喜！")       # ← 仍然是顺序执行
else:
    print("不及格")       # 条件块内部也是顺序结构
```

### 3.3 顺序结构的重要性

- 代码的执行顺序直接决定了程序的逻辑是否正确。
- 如果写代码时不注意顺序，结果可能完全错误。
- 顺序结构是调试代码时的基本思维方式——逐行阅读代码，模拟执行。

### 3.4 与条件结构、循环结构的区别

| 特性 | 顺序结构 | 条件结构 | 循环结构 |
|------|----------|----------|----------|
| 执行次数 | 每条语句执行 1 次 | 可能跳过某些语句 | 某些语句执行多次 |
| 可控性 | 无（全部执行） | 根据条件选择 | 根据条件重复 |
| 复杂度 | 最简单 | 中等 | 较复杂 |

## 4. Demo 样例

### Demo 1：简单的顺序执行

```python
# 一个完整的顺序结构程序：计算圆的面积和周长
print("===== 圆的面积和周长计算 =====")  # 第1步：输出标题

radius = float(input("请输入圆的半径："))  # 第2步：获取用户输入的半径

# 第3步：计算结果
pi = 3.14159                                # 定义圆周率
area = pi * radius * radius                 # 计算面积 = πr²
perimeter = 2 * pi * radius                 # 计算周长 = 2πr

# 第4步：输出结果
print("圆的半径是：", radius)               # 先输出半径
print("圆的面积是：", area)                  # 再输出面积
print("圆的周长是：", perimeter)             # 最后输出周长

print("===== 计算结束 =====")               # 第5步：结束提示
```

运行结果：

```text
===== 圆的面积和周长计算 =====
请输入圆的半径：5
圆的半径是： 5.0
圆的面积是： 78.53975
圆的周长是： 31.4159
===== 计算结束 =====
```

> 注意观察：代码严格按照从上到下的顺序执行。先获取输入 → 再计算 → 最后输出，三步不可颠倒。

### Demo 2：顺序结构中的变量依赖

```python
# 演示变量在顺序结构中的依赖关系
# 后面的代码可以使用前面定义的变量，反之则不行

# ✅ 正确：先定义再使用
x = 10                  # 第1步：定义变量 x
y = x + 5               # 第2步：y 依赖 x（此时 x 已存在）
z = y * 2               # 第3步：z 依赖 y（此时 y 已存在）
print("x =", x)
print("y =", y)
print("z =", z)

# ❌ 如果颠倒顺序就会出错
# print(result)          # 报错！result 还没定义
# result = x + y         # 这行定义在后面
```

运行结果：

```text
x = 10
y = 15
z = 30
```

> 核心理解：变量的使用必须在定义之后。就像你不能在"切菜"步骤之前就用"切好的菜"。

### Demo 3：交换两个变量的值（经典问题）

```python
# 经典问题：交换 a 和 b 的值
# 不能直接 a = b; b = a，因为顺序执行下会丢失数据

a = 10
b = 20
print("交换前：a =", a, ", b =", b)

# 方法1：使用第三个变量（经典方法）
temp = a    # 第1步：先把 a 的值暂时保存到 temp
a = b       # 第2步：把 b 的值赋给 a
b = temp    # 第3步：把 temp 中原来的 a 值赋给 b
print("交换后（方法1）：a =", a, ", b =", b)

# 方法2：Python 特有的简洁写法
a, b = b, a   # 一行搞定！Python 会先计算右边，再同时赋值
print("再次交换（方法2）：a =", a, ", b =", b)
```

运行结果：

```text
交换前：a = 10 , b = 20
交换后（方法1）：a = 20 , b = 10
再次交换（方法2）：a = 10 , b = 20
```

> 方法1 充分体现了顺序结构：每一步的顺序至关重要，temp 必须在 a 被覆盖前保存 a 的值。

## 5. 典型例题（附带答案）

### 例题 1

题目：

以下代码的输出是什么？请按顺序写出每一行执行后变量的变化。

```python
a = 5
b = a
a = 10
print(a)
print(b)
```

答案：

输出：

```text
10
5
```

解析（逐行分析执行过程）：

| 行号 | 代码 | 执行后的状态 |
|------|------|------------|
| 1 | `a = 5` | `a = 5` |
| 2 | `b = a` | `a = 5, b = 5`（b 得到 a 当前的值） |
| 3 | `a = 10` | `a = 10, b = 5`（修改 a，不影响 b） |
| 4 | `print(a)` | 输出 10 |
| 5 | `print(b)` | 输出 5 |

关键在于第 2 行：`b = a` 是把 a **当时的值**（5）赋给 b，而不是"让 b 永远跟着 a 变"。

### 例题 2

题目：

小明想交换 x 和 y 的值，写了以下代码。请问哪里有问题？如何修改？

```python
x = 100
y = 200
x = y     # 想交换
y = x     # 想交换
print("x =", x, "y =", y)
```

答案：

问题：第 3 行 `x = y` 后，x 原来的值 100 就丢失了。第 4 行 `y = x` 时 x 已经是 200 了，所以 y 还是 200。最终 x 和 y 都变成了 200。

正确写法：

```python
x = 100
y = 200
temp = x    # 先用 temp 保存 x 原来的值
x = y       # 把 y 的值给 x
y = temp    # 把之前保存的 x 值给 y
print("x =", x, "y =", y)  # x = 200 y = 100
```

## 6. 易错点

### 易错点 1：变量使用在定义之前

错误写法：

```python
print(name)         # NameError: name 'name' is not defined
name = "张三"       # 定义在使用的后面
```

错误原因：Python 按顺序执行，用到 `name` 时它还不存在。

正确写法：

```python
name = "张三"       # 先定义
print(name)         # 再使用
```

### 易错点 2：依赖关系顺序错误

错误写法：

```python
total = price * quantity   # NameError，price 和 quantity 还没定义
price = 25.5
quantity = 3
```

错误原因：计算 `total` 时用到了后面才定义的变量。

正确写法：

```python
price = 25.5
quantity = 3
total = price * quantity   # 所有变量都定义了，可以计算
```

### 易错点 3：交换变量时丢失数据

错误写法：

```python
a = 10
b = 20
a = b      # a=20，原来的10丢了
b = a      # b=20，因为a已经是20了
# 结果：a=20, b=20，交换失败
```

错误原因：直接赋值会覆盖原来的值，必须先保存。

正确写法：

```python
a, b = b, a   # Python 写法，简洁不易出错
```

### 练习 1

题目：

请写一个程序，按顺序完成以下三个步骤：
1. 定义变量 `length = 10`，`width = 5`
2. 计算矩形的面积（面积 = 长 × 宽）和周长（周长 = (长+宽)×2）
3. 输出面积和周长

答案：

```python
# 第1步：定义变量
length = 10
width = 5

# 第2步：计算面积和周长
area = length * width
perimeter = 2 * (length + width)

# 第3步：输出结果
print("矩形的面积是：", area)
print("矩形的周长是：", perimeter)
```

运行结果：

```text
矩形的面积是： 50
矩形的周长是： 30
```

### 练习 2

题目：

以下代码运行后会输出什么？请按顺序推导每一步。

```python
x = 3
y = x + 2
x = y - 1
y = x * 2
print("x =", x, "y =", y)
```

答案：

```text
x = 4 y = 8
```

推导过程：

| 行 | 代码 | 执行后 |
|----|------|--------|
| 1 | `x = 3` | `x=3` |
| 2 | `y = x + 2 = 3+2` | `x=3, y=5` |
| 3 | `x = y - 1 = 5-1` | `x=4, y=5` |
| 4 | `y = x * 2 = 4*2` | `x=4, y=8` |

### 练习 3

题目：

温度转换程序：请按顺序结构写一个程序，让用户输入摄氏温度，程序将其转换为华氏温度并输出。公式：华氏度 = 摄氏度 × 1.8 + 32。

答案：

```python
# 第1步：获取用户输入的摄氏温度
celsius = float(input("请输入摄氏温度："))

# 第2步：转换为华氏温度
fahrenheit = celsius * 1.8 + 32

# 第3步：输出结果
print(celsius, "摄氏度 =", fahrenheit, "华氏度")
```

说明：每一步按照先后顺序执行：输入 → 计算 → 输出。顺序不能乱。

## 8. 答题总结

- 顺序结构是程序最基本的执行方式：代码**从上到下、逐行**执行。
- 程序的三种基本结构：顺序结构、条件分支、循环结构。顺序结构是其他两种的基础。
- 变量必须在**使用前定义**，后面的代码可以依赖前面定义的变量。
- 交换两个变量的值时要注意顺序，直接赋值会丢失数据。Python 可以用 `a, b = b, a` 简化。
- 写代码时养成"逐行推演"的习惯——在脑中按顺序模拟每一行执行后的结果。
