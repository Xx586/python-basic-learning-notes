
# python-learning-material Skill 使用说明

## 这是什么？

`python-learning-material` 是一个 Claude Code 可复用 Skill，用于**系统化整理和生成面向零基础学生的 Python 基础语法学习资料**。

它把从多次 Claude Code 对话中沉淀出的完整工作流程固化为一套可复用的规则、模板和 prompt。

## 是否可以用于其他学习资料

**可以。** 这个 Skill 的核心是一套"学习资料整理流程"，不局限于 Python。

虽然 Skill 最初是从 Python 基础语法资料整理中沉淀出来的，但它的工作流程（资料盘点 → 生成框架 → 分章完善 → 自检 → 验收 → 修复）和 8 部分知识点结构是**学科无关**的，可以迁移到其他方向：

- **Java 基础语法**
- **Linux 基础**
- **MySQL / SQL**
- **前端基础（HTML/CSS/JavaScript）**
- **数据结构与算法**
- **其他有学习大纲、PPT、旧文档、截图资料的课程**

### 迁移时需要做什么

迁移到其他学科时，需要把 Python 相关规则替换成目标学科规则：

| 替换项 | Python | Java 示例 |
|--------|--------|----------|
| 学习路线 | Python 学习路线 | Java 学习大纲 / 课程大纲 |
| Demo 代码语言 | `.py` / Python 语法 | `.java` / Java 语法 |
| 旧资料 | Python 笔记、文档 | Java PPT、旧笔记、作业要求 |
| 验收知识点 | Python 44 个核心知识点 | Java 对应章节的核心知识点 |
| 易错点领域 | Python 缩进/中文符号/GIL | Java 分号/类型声明/编译错误 |

### 迁移示例：用于 Java 基础语法

如果要把这个 Skill 用于 Java 基础语法资料整理，应准备：

- Java 学习大纲（相当于 Python 学习路线）
- Java 课程 PPT 或讲义
- Java 旧版笔记或文档
- 章节目录（如：环境搭建 → 基本语法 → 面向对象 → 异常处理 → 集合框架 → IO 流 → 多线程）
- 知识点截图或思维导图
- 示例代码或作业要求

使用时仍然遵循原 Skill 的核心流程：

1. 资料盘点 → 扫描 Java 文件夹中的 PPT、大纲、旧笔记和代码
2. 生成知识点框架 → 根据 Java 大纲生成章节结构
3. 分章节完善 → 每个知识点一个独立 `.md` 文件
4. 生成 Demo 样例 → 使用 `.java` 代码，符合 Java 语法
5. 生成典型例题 → Java 语法相关的习题
6. 生成课后练习 → 由易到难的 Java 编程练习
7. 最终验收 → 对照 Java 大纲检查覆盖
8. 按问题修复 → A/B/C 分级修复

**原来的 8 部分知识点结构仍然可以保留**，因为它本身是学科无关的：

```markdown
# 知识点名称

## 1. 知识点定义
## 2. 语法格式
## 3. 重点说明
## 4. Demo 样例
## 5. 典型例题（附带答案）
## 6. 易错点
## 7. 课后练习（附带答案）
## 8. 答题总结
```

### Java 迁移使用示例

**示例 1：资料盘点**

```
请读取 python_learning_material_skill/SKILL.md。

本轮只做资料盘点，不要生成正文。

```

**示例 2：完善单个章节**

```

现在使用该 Skill 的流程完善 Java 基础语法第 2 章：数据类型与运算符。

要求：
1. 只处理第 2 章。
2. 每个知识点一个独立 .md 文件。
3. Demo 使用 Java 代码。
4. 每个知识点仍然使用固定 8 部分结构。
5. 完成后进行本章自检。
```

### 迁移注意事项

1. **不是"把 Python 资料翻译成 Java"**，而是"用同样的流程从零整理 Java 资料"。
2. 需要替换领域知识 — 验收标准中的 Python 特有问题（如 GIL、缩进错误）要换成 Java 特有问题（如分号遗漏、类型声明、编译错误）。
3. Demo 语言必须切换 — Python Demo 用 `.py` + Python 语法，Java Demo 用 `.java` + Java 语法。
4. 如果目标学科没有思维导图，可以用课程大纲或目录结构替代。
5. 部分 prompt（如 `05_final_acceptance.md`）中的 Python 特定检查项需要在执行时告知 AI 替换为目标学科对应的检查项。

## 为什么建议分批次完成

**不建议一上来就让 AI 直接生成完整的学习资料。** 本项目的实际经验表明，复杂的"学习资料生成 / 知识点讲解文档生成 / 课程资料整理"任务，最好采用分批次的方式逐步完成。

### 本项目不是一次性完成

本项目在生成 Python 基础语法资料时，并没有直接让 AI 一次性全部 10 章内容，而是按以下顺序逐步推进：

1. **先读取并盘点已有资料** — 扫描文件夹中的 .md / .py / .png / .txt，阅读学习路线、思维导图、截图和旧版知识点文档；
2. **再设计章节和知识点框架** — 根据学习路线为主、思维导图和截图为补充，生成 10 章知识点总框架；
3. **先生成所有章节的知识点框架** — 创建 `00_资料总框架.md`，标注每个知识点的来源和状态；
4. **再按章节分批完善** — 先完善前五章，再完善第 6-10 章，每章完成后执行 13 项自检；
5. **最后整体验收和修复** — 对 10 章执行最终验收，输出 A/B/C 三级问题清单，按优先级修复；
6. **沉淀为 Skill** — 将整个流程总结为可复用的规则包。

### 为什么不建议一次性生成

| 问题 | 说明 |
|------|------|
| **上下文长度有限** | 一次对话的上下文长度有限，10 章 × N 个知识点的完整内容很容易超出上下文窗口，导致后半部分被截断或遗忘 |
| **AI 可能读不完整现有资料** | 如果一上来就要求生成，AI 可能跳过对学习路线、思维导图、截图和旧资料的仔细阅读，导致知识点遗漏 |
| **内容质量前后不一** | 一次性生成时，前面的章节质量往往较高，后面章节内容容易变得空泛、重复或模板化 |
| **Demo/例题/练习容易退化** | 随着生成内容增多，Demo 样例、典型例题和课后练习容易变得重复、缺乏针对性 |
| **文件结构容易混乱** | 同时修改大量文件时，可能出现文件命名不一致、章节归类错误等问题 |
| **验收和修复成本更高** | 一次性生成后发现问题，需要回头逐个文件排查，修复成本远高于分批自检 |

### 推荐的分批流程

无论是 Python、Java、Linux、数据库、前端还是数据结构，都建议采用以下分批流程：

```
第 1 步：阅读现有资料（学习路线、大纲、PPT、旧笔记、截图）
    ↓
第 2 步：资料盘点（输出盘点报告，确认已有内容和缺失内容）
    ↓
第 3 步：设计章节和知识点框架（等待用户确认）
    ↓
第 4 步：生成知识点目录（标注来源和状态）
    ↓
第 5 步：按章节逐步完善（每次只处理一个章节，完成后自检）
    ↓
第 6 步：最终验收（整体检查章节完整性和内容质量）
    ↓
第 7 步：按问题优先级修复（A → B → C）
```

**核心原则**：先读资料 → 再设计框架 → 最后分批生成文档。每一步都在上一步的基础上推进，而不是跳过中间步骤直接跳到最终产出。

> **例如**，本项目在生成 Python 基础语法资料时，并没有直接让 AI 一次性生成全部 10 章内容，而是先让 AI 阅读现有资料，完成资料盘点；再根据学习路线和思维导图生成 10 章知识点框架；随后分批完善前五章和第 6-10 章；最后进行整体验收和问题修复。这样可以减少遗漏，提高每个知识点文档、Demo、例题和练习的质量。

## 适合什么任务？

- 有一套 Python 学习路线、思维导图、截图和零散的旧资料，需要整理成系统化学习材料。
- 已经有了章节框架，需要逐章补充知识点讲解。
- 需要验证一套 Python 学习资料的质量。
- 需要把"生成学习资料"这件事变成可重复执行的流程。

## 不适合什么任务？

- 写单个 Python 脚本（不需要完整资料体系）。
- 翻译或格式化现有文档（不需要生成新内容）。
- 制作高级/专家级 Python 教程（本 Skill 面向零基础）。

## 注意事项

1. **不要一次性生成全部内容** — 分批处理，每次一个章节，质量优先。
2. **先框架、再分章节、再验收** — 三阶段顺序不可跳过。
3. **不要删除原始资料** — 原始文件是宝贵资产。
4. **每章完成后自检** — 13 项检查标准，通过后再进入下一章。
5. **验收只查不改** — 验收报告输出后，由用户决定是否修复、修复哪些。
6. **任务过长时暂停** — 不为了赶进度压缩内容质量。
7. **不确定的内容标注"不确定"** — 不要编造。

## Skill 生成背景

`python_learning_material_skill` 是在本项目 Python 基础语法学习资料整理完成后，进一步沉淀出来的一套可复用 Skill。

本项目最初的目标是：基于已有的 Python 基础资料、学习路线、思维导图和知识点截图，生成一套适合零基础学生系统学习的 Python 基础语法资料。整个资料生成过程并不是一次性完成的，而是通过 Claude Code 多轮对话逐步推进完成的。

在资料生成过程中，Claude Code 结合 DeepSeek V4 Pro 模型，按照以下流程逐步完成资料整理：

1. 先扫描并阅读当前 `python1` 文件夹中的所有资料；
2. 识别已有的 1-10 个章节目录；
3. 阅读学习路线、思维导图、截图、旧版知识点文档和项目规则；
4. 生成 10 个章节的知识点总框架；
5. 按章节分批完善知识点讲解内容；
6. 为每个知识点补充 Demo 样例、典型例题、易错点、课后练习和答题总结；
7. 对前五章和后五章分别进行完善；
8. 最后对 10 个章节进行整体验收；
9. 根据验收结果修复结构缺失、内容不完整、Demo 不规范等问题；
10. 在资料生成完成后，再结合最终生成的资料、核心提示词文件和 Claude Code 对话记录，总结出本 Skill。

## Skill 的核心作用

这个 Skill 的作用是把一次成功的学习资料生成流程，沉淀成以后可以重复使用的规则包。

以后如果需要整理新的学习资料，可以使用这个 Skill 指导 AI 按照固定流程工作，而不是每次重新写一大段提示词。

它主要支持以下任务：

1. 自动盘点项目资料；
2. 阅读学习路线、旧资料、思维导图、截图和提示词；
3. 生成完整章节框架；
4. 按章节逐步完善知识点讲解；
5. 为每个知识点生成统一格式的 `.md` 文档；
6. 为每个知识点生成 Demo 样例；
7. 补充典型例题、课后练习和答案；
8. 总结易错点；
9. 对章节进行自检；
10. 对整个项目进行最终验收；
11. 根据验收报告修复问题。

## Skill 使用的知识点文档标准

本 Skill 要求每个知识点文档都使用统一的 8 部分结构：

```markdown
# 知识点名称

## 1. 知识点定义

## 2. 语法格式

## 3. 重点说明

## 4. Demo 样例

## 5. 典型例题（附带答案）

## 6. 易错点

## 7. 课后练习（附带答案）

## 8. 答题总结
```

这种结构的目的，是让每个知识点都既有理论解释，也有代码示例、练习题、易错点和复习总结，适合 Python 零基础学生系统学习。

## 为什么要生成这个 Skill

在本项目中，生成 Python 基础语法资料的过程比较复杂，涉及：

* 多个原始资料来源；
* 多个章节目录；
* 多个知识点文件；
* 统一的文档结构；
* Demo 代码要求；
* 例题和练习要求；
* 分章节生成；
* 最终验收和修复。

如果每次都重新写提示词，容易出现遗漏、格式不统一、质量不稳定等问题。

因此，在完成本项目后，将整个流程总结成 `python_learning_material_skill`，可以让后续类似任务更加稳定、高效、可复用。
