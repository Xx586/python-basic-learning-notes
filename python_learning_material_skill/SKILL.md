# python-learning-material

## Skill 简介

这是一个 Claude Code 可复用 Skill，用于**系统化整理和生成 Python 基础语法学习资料**。

本 Skill 将完整的工作流程（资料盘点 → 框架生成 → 分章完善 → 自检 → 最终验收 → 修复）沉淀为 16 个文件，包含规则集、模板、子阶段 prompt，可直接在 Claude Code 中按需加载执行。

---

## Skill 目标

1. **从零开始**：用户提供一个包含旧资料、学习路线、思维导图和截图的文件夹，Skill 自动完成从盘点 → 框架 → 逐章完善 → 验收 → 修复的全过程。
2. **增量完善**：用户已有部分成型的章节目录，Skill 在现有基础上继续完善指定章节，保持与已有资料的一致性。
3. **验收保障**：Skill 提供 A/B/C 三级问题分类标准，确保最终产出物质量可控。

## 输入资料要求

在使用本 Skill 之前，目标文件夹中应尽可能包含以下资料：

| 必需程度 | 文件/资料 | 作用 |
|:--:|------|------|
| **必需** | 学习路线文件（如 `Python基础学习路线(5).md`） | 知识点主线依据 |
| **必需** | 项目规则文件（如 `CLAUDE.md`） | 执行规则和框架定义 |
| **强烈建议** | 主 prompt 文件（如 `promopt.txt`） | 最详细的生成规范 |
| **强烈建议** | 思维导图（如 `思维导图.png`） | 知识点结构全景 |
| **建议** | 学习规范文件 | 文件组织规范 |
| **建议** | 知识点结构截图（多张 .png） | 知识点补充验证 |
| **可选** | 已有旧资料（.md / .py 文件） | 内容参考和保留 |
| **可选** | JSONL transcript | 历史对话记录 |

**兜底规则**：如果某些资料缺失，Skill 仍可基于已有的 prompt 文件、学习路线和项目规则文件运行。缺失内容需在报告中说明，但不阻止流程推进。

## 总体工作流程

```
准备工作（第 0 步）
    │  读取 CLAUDE.md / prompt / 学习路线 / 思维导图 / 截图 / 旧资料
    ▼
资料盘点（第 1 步）
    │  扫描文件夹 → 阅读所有资料 → 输出盘点报告
    ▼
生成框架（第 2 步）
    │  输出知识点清单 → 输出整理方案 → 等待用户确认 → 生成 00_资料总框架.md
    ▼
分章节完善（第 3 步）── 每章完成后自检（第 4 步）
    │  逐章：读取已有资料 → 补齐 8 部分 → 生成 Demo/例题/练习/易错点
    ▼
最终验收（第 5 步）
    │  10 项验收检查 → 输出 A/B/C 问题清单
    ▼
问题修复（第 6 步）
    │  按 A→B→C 优先级逐类修复 → 输出修复报告
    ▼
生成 README（第 7 步）
    │  生成或更新项目根目录 README.md（11 项内容）
    ▼
沉淀复用（第 8 步）
        提炼工作流 → 写入 memory → 可选：更新 Skill
```

## 关键规则摘要

### 知识点文档结构
每个知识点 `.md` 文件必须使用固定 8 部分结构：

```markdown
# 知识点名称
## 1. 知识点定义
## 2. 语法格式
## 3. 重点说明
## 4. Demo 样例
## 5. 典型例题（附带答案）
## 6. 易错点
## 7. 课后练习（附带答案）
## 8. 答题总结
```

### Demo 规则
- 每个知识点必须有 Demo。
- Demo 必须可运行、有注释、有运行结果。
- 注释写在代码对应行旁边或上方，解释"做什么"和"为什么这样写"。

### 例题与练习规则
- 典型例题 1-2 道，附带答案和解析。
- 课后练习 2-3 道，由易到难，附带可运行答案。

### 易错点规则
- 每个易错点包含：错误写法 + 错误原因 + 正确写法。
- 尽量结合 Python 常见报错信息。

### 分批处理规则
- 不一次性生成全部 10 章。
- 每次只处理一个章节。
- 每章完成后自检 13 项，通过后进入下一章。

### 验收规则
- 验收时只检查，不修改文件。
- 输出 A/B/C 三级问题清单。
- 按 A→B→C 优先级修复。

## 安全限制

1. **不删除原始资料** — 任何情况下都不删除用户提供的原始文件。
2. **不覆盖重要文件** — 修改前必须确认，不确定时先备份。
3. **不凭空重写** — 必须基于已读取的资料生成内容。
4. **不一次性生成全部** — 每次只处理一个章节。
5. **先报告再执行** — 需要移动、合并、重命名文件时先报告。
6. **验收不修改** — 验收阶段只输出报告，用户明确说"开始修复"后才修改。
7. **不确定就标注** — 不确定的内容标注"不确定"，不要编造。

## 拒绝场景

以下场景不适用本 Skill：
- 用户只想写一个单独的 Python 脚本（不需要完整资料体系）。
- 用户只需要翻译或格式化现有文档（不需要生成新内容）。
- 用户需要的是高级/专家级 Python 教程（本 Skill 面向零基础）。

在使用本 Skill 的 prompt 前，用户只需要说明：
1. 目标文件夹路径（默认当前工作目录）。
