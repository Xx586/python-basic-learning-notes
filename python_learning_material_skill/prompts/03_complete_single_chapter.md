> **用途**：完善用户指定的单个章节，补齐所有知识点 `.md` 文件的 8 个部分。
>
> **依据**：`references/workflow.md` 第 3-4 步，`references/knowledge_point_rules.md` 全部，`templates/knowledge_point_template.md`

---

本轮任务范围只包括第 X 章，请不要处理其他章节。

9. 当前第 X 章目录下的所有 `.md` 文件。

## 二、本轮任务目标

基于已经生成好的知识点框架，完善第 X 章每个知识点 `.md` 文件中的正文内容。

每个知识点文档必须完整包含以下 8 个部分：

```markdown
# 知识点名称

## 1. 知识点定义
## 2. 语法格式
## 3. 重点说明
## 4. Demo 样例
## 5. 典型例题（附带答案）
## 6. 易错点
## 7. 课后练习（附带答案）
## 8. 答题总结
```

- 如果某个文件缺少部分标题，请补齐。
- 如果标题顺序混乱，请整理为以上顺序。
- 如果已经存在"待完善"内容，请将其替换为完整讲解。
- 如果已有内容可用，请保留并优化。

---

## 三、每个部分的写作要求

### 1. 知识点定义
- 面向 Python 零基础学生，用通俗语言解释。
- 不要照搬官方文档，不要写得太抽象。
- 可以使用生活类比。
- 要说明这个知识点是什么、解决什么问题、为什么要学。

### 2. 语法格式
- 写出最常用、最基础、最实用的语法格式。
- 使用 Python 代码块展示。
- 每种语法后面必须有解释。
- 语法必须准确，不能是伪代码，不要一上来就堆高级写法。

### 3. 重点说明
- 结合旧资料、学习路线、思维导图和截图中的重点。
- 讲清楚"什么时候用"。
- 讲清楚"和相似知识点有什么区别"。
- 讲清楚"初学者最容易混淆的地方"。
- 可以用表格对比。

### 4. Demo 样例
- 必须提供一个可运行的 Python Demo。
- Demo 必须服务于当前知识点，不能是伪代码。
- 尽量只用标准库，不要依赖第三方库。
- Demo 场景要尽量贴近学习或生活。
- Demo 代码必须有清晰注释，注释写在代码对应行旁边或上方。
- 注释要解释"代码做什么"和"为什么这样写"。
- Demo 后面必须写出预期运行结果。
- 涉及文件读写要说明会生成或读取哪些文件。
- 涉及异常处理要体现异常捕获效果。
- 涉及正则表达式要展示匹配结果。
- 涉及迭代器/生成器要体现惰性计算。
- 涉及面向对象要体现类/对象/属性/方法。

### 5. 典型例题（附带答案）
- 提供 1-2 道典型例题。
- 题目必须围绕当前知识点，难度适合零基础。
- 必须附带完整答案和解析。
- 不能只给代码不解释。

### 6. 易错点
- 总结初学者最容易犯的错误（至少 2-3 个）。
- 每个易错点都要包含：错误写法 + 错误原因 + 正确写法。
- 尽量结合 Python 常见报错信息。

### 7. 课后练习（附带答案）
- 提供 2-3 道课后练习。
- 练习难度由易到难。
- 每道练习都必须附带答案，答案代码必须可运行。
- 不要超出当前知识点太多。

### 8. 答题总结
- 用 3-5 条 bullet 总结。
- 适合写进笔记、考试前快速复习。
- 总结定义、语法、使用场景、易错点。

## 四、旧资料处理规则

| 情况 | 处理方式 |
|------|---------|
| 原文正确但太简略 | 保留原文，在对应位置补充解释 |
| 原文有代码但缺少讲解 | 保留代码，并给代码加注释 |
| 原文适合有基础的人但不适合零基础 | 增加"小白理解版" |
| 原文内容重复 | 合并成一份更清晰的讲解 |
| 原文有错误 | 修正错误，不保留错误说法 |
| 原文与学习路线冲突 | 以学习路线为准 |
| 学习路线有但旧资料没有 | 新建对应知识点 `.md` 文件 |
| 截图或思维导图有但旧资料没有 | 作为补充知识点加入 |

5. [ ] Demo 是否有注释（在代码旁边或上方）
6. [ ] Demo 是否有运行结果
7. [ ] 是否有典型例题（1-2 道）
8. [ ] 典型例题是否附带答案和解析
9. [ ] 是否有课后练习（2-3 道）

12. [ ] 是否适合零基础学生学习
13. [ ] 是否有遗漏学习路线、思维导图、截图中的知识点

**发现问题请立即修复当前章节，再进入下一章节。**

3. 如果要修改已有文件，请在原文件基础上补充完善。

5. 如果需要新增知识点文件，必须在报告中说明。
6. 不要把所有内容写到一个大文件里。
7. 每个知识点必须保持独立 `.md` 文件。
8. 任务过长时暂停，等待用户说"继续"。
9. 如果旧内容正确但简略，请保留并补充，不要删除。
