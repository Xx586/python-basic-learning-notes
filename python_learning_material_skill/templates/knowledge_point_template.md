# 知识点名称

> 文件名规范：`序号.知识点中文名.md`
> 
> 特殊字符处理（在正文中说明即可）：
> - `*args` / `**kwargs` → 文件名中用 "可变参数" 替代
> - `__init__` → 文件名中用 "init构造方法" 替代
> - `__name__ == "__main__"` → 文件名中用 "name_main入口判断" 替代

## 4. Demo 样例

> **填写要求：**
> - 必须提供一个可以直接运行的 Python Demo。
> - Demo 必须服务于当前知识点。
> - Demo 不能是伪代码。
> - 尽量只用 Python 标准库，不要依赖第三方库（除非当前知识点本身涉及 pip、模块、包或第三方库）。
> - Demo 场景要尽量贴近学习或生活。
> - Demo 代码必须有清晰注释，注释写在代码对应行旁边或上方。
> - 不要单独新建"代码讲解文件"。
> - 注释要解释"这行代码做了什么"和"为什么这样写"。
> - 关键变量、判断、循环、函数调用、类定义、文件操作、异常处理必须解释。
>
> **代码注释正确示例：**
>
> ```python
> age = 18  # 用 age 保存年龄，后面可以根据年龄判断是否成年
> ```
>
> **代码注释错误示例（不要这样写）：**
>
> ```python
> a = 1  # a 等于 1
> ```
>
> **运行结果格式（Demo 后面必须写）：**
>
> ```text
> 运行结果：
> 及格
> ```
>
> **涉及文件读写的 Demo**：要说明会生成或读取哪些文件。
> **涉及异常处理的 Demo**：要能清楚展示异常被捕获后的效果。
> **涉及正则表达式的 Demo**：要展示匹配或替换结果。
> **涉及迭代器/生成器的 Demo**：要体现"逐个生成""节省内存"的思想。
> **涉及面向对象的 Demo**：要能清晰体现类、对象、属性或方法。
