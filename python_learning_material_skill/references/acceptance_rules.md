# 验收规则集

> **来源**：从 `promopt.txt` 验收相关的 prompt、Session 4 最终验收对话 JSONL transcript 中提炼。
>
> **用途**：执行最终验收时的"评分标准"。验收只在三个层面进行：结构层、内容层、覆盖层。

---

## 1. 结构层验收标准

### 1.1 章节目录

| 检查项 | 标准 | 不合格判定 |
|------|------|----------|
| 10 个章节目录 | 全部存在，命名一致 | 章节缺失、命名混乱导致无法对应 |
| 目录命名格式 | `N.章节中文名` | 格式不一致 |

### 1.2 知识点文件

| 检查项 | 标准 | 不合格判定 |
|------|------|----------|
| 文件数量 | 与总框架中的清单一致 | 文件缺失 |
| 文件非空 | 所有文件有实质内容 | 空文件（0 字节或仅有标题） |
| 无占位内容 | 无"待完善""TODO""后续补充" | 存在未完成的占位内容 |
| 文件命名 | `序号.知识点中文名.md` | 命名混乱无法识别 |

### 1.3 8 部分框架

| 检查项 | 标准 | 不合格判定 |
|------|------|----------|
| 部分完整性 | 每个文件包含全部 8 个部分 | 缺少 1 个或以上部分 |
| 标题顺序 | 按 1→2→3→...→8 顺序排列 | 标题顺序混乱 |
| 标题名称 | `## N. 部分名称` 格式 | 标题名称不规范 |
| 内容实质性 | 每个部分有实质内容 | 只有空话或仅重复标题 |

### 1.4 demo/README.md

| 检查项 | 标准 | 不合格判定 |
|------|------|----------|
| 每章一个 | 10 个章节各有一个 | 缺失 |

## 2. 内容层验收标准

### 2.1 Demo 样例检查（15 项）

| 序号 | 检查项 | 标准 |
|:--:|------|------|
| 1 | 相关性 | 是否围绕当前知识点 |
| 2 | 可运行性 | 是否是可运行的 Python 代码（非伪代码） |
| 3 | 非伪代码 | 是否有实际可执行的 Python 语句 |
| 4 | 注释完整性 | 是否有清晰注释 |
| 5 | 注释位置 | 注释是否写在代码对应行旁边或上方 |
| 6 | 注释质量 | 是否解释"这行代码做了什么"和"为什么这样写" |
| 7 | 运行结果 | Demo 后是否写了预期运行结果 |
| 8 | 零基础适配 | 是否适合零基础学生理解 |
| 9 | 语法正确性 | 是否有明显 Python 语法错误 |
| 10 | 第三方库依赖 | 是否依赖第三方库（如依赖，是否说明安装方式） |
| 11 | 文件操作透明度 | 文件操作类 Demo 是否说明会生成或读取哪些文件 |
| 12 | 异常捕获效果 | 异常处理类 Demo 是否能体现异常捕获效果 |
| 13 | 正则匹配效果 | 正则类 Demo 是否能展示匹配结果 |
| 14 | 惰性计算展示 | 迭代器和生成器类 Demo 是否体现逐步生成或惰性计算 |
| 15 | 面向对象展示 | 面向对象类 Demo 是否能清晰体现类、对象、属性或方法 |

### 2.2 典型例题检查

| 检查项 | 标准 | 不合格判定 |
|------|------|----------|
| 数量 | 1-2 道 | 0 道 |
| 相关性 | 围绕当前知识点 | 与知识点无关 |
| 难度 | 适合零基础 | 明显超纲 |
| 答案完整性 | 有完整答案 | 缺答案 |
| 解析完整性 | 有必要解析 | 只给代码不解释 |
| 格式 | 题目/答案/解析三段式 | 格式混乱 |

### 2.3 课后练习检查

| 检查项 | 标准 | 不合格判定 |
|------|------|----------|
| 数量 | 2-3 道 | 0 道或只有 1 道 |
| 难度递进 | 由易到难 | 全部同一难度 |
| 相关性 | 巩固当前知识点 | 偏离当前知识点 |
| 答案完整性 | 每道都有答案 | 缺答案 |
| 答案可运行性 | 答案代码可运行 | 答案代码有明显错误 |
| 超纲程度 | 不超出当前知识点太多 | 需要后续章节知识才能完成 |
| 说明完整性 | 有简要说明 | 纯代码无任何说明 |

### 2.4 易错点检查

| 检查项 | 标准 | 不合格判定 |
|------|------|----------|
| 数量 | 至少 2-3 个 | 0 个或只有 1 个 |
| 错误写法 | 每个易错点有错误代码 | 只有描述无代码 |
| 错误原因 | 每个易错点有原因说明 | 只有"这样会报错"无具体原因 |
| 正确写法 | 每个易错点有正确代码 | 只指出错误不给正确写法 |
| 报错关联 | 尽量结合 Python 报错信息 | 完全不提报错类型 |

### 2.5 内容质量检查（13 项）

| 序号 | 检查项 | 标准 |
|:--:|------|------|
| 1 | 零基础适配 | 是否适合 Python 零基础学生 |
| 2 | 通俗易懂 | 语言是否通俗，不官方文档化 |
| 3 | 有例子 | 是否结合例子讲解，不纯写概念 |
| 4 | 无重复 | 是否存在大量重复段落 |
| 5 | 无废话 | 是否为了凑字数写废话 |
| 6 | 可学习性 | 讲解是否足够让学生真正学习 |
| 7 | 不过深 | 是否超出基础阶段，过多讲高级特性 |
| 8 | 概念正确 | 是否存在明显 Python 概念错误 |
| 9 | 衔接合理 | 知识点的前后衔接是否合理 |
| 10 | 风格统一 | 不同章节的语言风格是否基本一致 |
| 11 | 命名清晰 | 文件命名是否能一眼看出是什么知识点 |
| 12 | 格式一致 | 代码块、表格、标题等格式是否一致 |
| 13 | 零基础友好 | 是否避免了"显然""众所周知""不言而喻"等不适合零基础的表达 |

## 3. 覆盖层验收标准

### 3.1 学习路线覆盖检查清单

对照 `Python基础学习路线(5).md` 中的 44 个核心知识点（按章节排列）：

| 阶段 | 核心知识点 | 应在章节 |
|------|---------|---------|
| 🟢 第一阶段 | Python安装与环境配置、IDE选择、pip包管理、虚拟环境、代码注释与编码规范 | 1 |
| 🔵 第二阶段 | 基本数据类型、字符串操作、容器类型、类型转换与类型判断、运算符 | 2 |
| 🟡 第三阶段 | 条件判断、循环、推导式 | 3 |
| 🟠 第四阶段 | 函数定义与调用、作用域、lambda匿名函数、递归函数、闭包、装饰器、类型注解、常用内置函数 | 4 |
| 🔴 第五阶段 | 类和对象基础、方法类型、继承与多态、封装、魔术方法、dataclass、抽象类 | 5 |
| 🟣 第六阶段 | 文件操作、路径操作、JSON读写、CSV读写、异常处理 | 6 |
| 🔵 第七阶段 | 模块导入、自定义模块与包、常用标准库 | 7 |
| 🟡 第八阶段 | 正则表达式（re模块） | 8 |
| 🔴 第九阶段 | 可迭代对象与迭代器、生成器 | 9 |
| 🟣 第十阶段 | 上下文管理器、深拷贝与浅拷贝、解包操作、多线程与多进程、协程基础 | 10 |

### 3.2 思维导图和截图覆盖

- 将 `思维导图.png` 和所有截图中的知识点结构与现有文件做对照。
- 标注"已覆盖""可能遗漏""不确定（图片内容无法完全识别）"。
- 图片中属于进阶方向的内容（如数据库编程、并发编程、网络编程）标为"进阶方向，不强制展开"。

### 3.3 跨章节一致性

- 同一知识点在多处出现时，以主要讲解章节为准。
- 其他章节中出现时，说明引用关系而不是重复展开。
- 例如："推导式"在第 3 章为主要讲解，第 9 章只需要简要回顾。

## 5. A/B/C 类问题分类标准

### A 类问题：必须修复（影响基本可用性）

| 问题类型 | 示例 |
|---------|------|
| 章节缺失 | 缺少某个章节目录 |
| 知识点文件缺失 | 学习路线中的知识点没有对应 .md 文件 |
| 知识点文件为空 | 文件存在但内容为空 |
| 缺少 8 部分结构 | 文件只有 3-4 个部分，缺少其余的 |
| 大量"待完善"占位 | 文件中存在未完成内容 |
| Demo 不存在 | 没有 Demo 样例 |
| Demo 不可运行 | Demo 是伪代码或有明显语法错误 |
| 典型例题缺答案 | 只有题目，没有答案和解析 |
| 课后练习缺答案 | 只有题目，没有答案 |
| 明显知识点遗漏 | 学习路线中的重要知识点完全没有涉及 |
| 明显概念错误 | 文件中存在错误的 Python 概念 |

### B 类问题：建议修复（影响学习体验）

| 问题类型 | 示例 |
|---------|------|
| 内容太短 | 某个部分只有一两句话，无法真正学习 |
| Demo 注释不够清晰 | 注释太简略，没有解释"为什么这样写" |
| 易错点不够具体 | 只说"容易出错"但不给错误代码 |
| 例题不够贴近 | 例题和当前知识点的关联度低 |
| 练习层次不明显 | 3 道练习难度完全一样 |
| 讲解不够友好 | 对零基础学生来说专业性太强 |
| 文件命名不够清晰 | 文件名和内容不完全对应 |
| 知识点归类不够合理 | 某个知识点放在 A 章比放在 B 章更合适 |

### C 类问题：可后续优化（锦上添花）

| 问题类型 | 示例 |
|---------|------|
| 可增加图示 | 用流程图说明条件判断逻辑 |
| 可增加综合练习 | 跨知识点的项目化练习 |
| 可统一语言风格 | 不同章节的语言表达风格不完全一致 |
| 可补充项目案例 | 面向对象章节增加完整迷你项目 |
| 可增加更多练习 | 某些知识点只有最低要求的 2 道练习 |

## 7. 修复优先级

1. **先修复 A 类问题**：结构缺失、文件缺失、Demo 不可运行、缺答案。这些是阻塞性问题，不修复则资料不可用。
2. **再修复 B 类问题**：内容太短、注释不清晰、易错点不够具体。这些影响学习体验但不阻断使用。
3. **最后修复 C 类问题**：命名优化、增加图示、更多练习。这些是锦上添花，可在后续迭代中逐步优化。

## 8. 安全限制

### 数据安全

| 序号 | 规则 |
|:--:|------|
| 1 | 不删除任何原始资料 |
| 2 | 不覆盖重要文件，修改前必须备份或确认 |
| 3 | 修改前先创建备份目录 |
| 4 | 不确定是否要删除的文件一律保留 |
| 5 | 重复文件先移动到备份或说明处理方式 |

### 内容安全

| 序号 | 规则 |
|:--:|------|
| 6 | 不凭空重写，必须基于已读取的资料生成 |
| 7 | 不忽略已有 prompt 文件 |
| 8 | 不忽略学习路线、思维导图和截图 |
| 9 | 不把所有内容写进一个大文件 |
| 10 | 不生成空泛、重复、模板化严重的内容 |

### 流程安全

| 序号 | 规则 |
|:--:|------|
| 11 | 先盘点 → 再框架 → 再完善，三阶段不可跳过 |
| 12 | 资料盘点报告和总框架必须经用户确认后再大规模修改 |
| 13 | 每次只处理一个章节，不一次性生成全部 |
| 14 | 每章完成后自检，不带着已知问题进入下一章 |
| 15 | 需要移动、合并、重命名文件时先报告 |
| 16 | 同名文件冲突时先说明再处理，不静默覆盖 |
| 17 | 验收时只输出报告，不直接修改文件 |
| 18 | 用户明确说"开始修复"后才修改文件 |
