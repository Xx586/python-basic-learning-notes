# 抽象类

## 1. 知识点定义

**抽象类**是"规定子类必须实现哪些方法"的模板。它自己不干活（不能实例化），只定义接口规范，子类必须实现所有抽象方法才能被实例化。

Python 通过 `ABC`（Abstract Base Class）和 `@abstractmethod` 实现抽象类。

> 小白理解：抽象类就像"岗位说明书"——规定了"这个岗位的人必须会做 A、B、C 三件事"，但不告诉你具体怎么做。不同的员工（子类）用不同的方式来完成这三件事。

## 2. 语法格式

```python
from abc import ABC, abstractmethod

class Animal(ABC):            # 继承 ABC 成为抽象类
    @abstractmethod
    def speak(self):          # 抽象方法：子类必须实现
        pass

    def sleep(self):          # 普通方法：子类可以继承
        print("Zzz...")

class Dog(Animal):
    def speak(self):          # 必须实现！
        return "汪汪"

# animal = Animal()           # ❌ TypeError: 不能实例化抽象类
dog = Dog()                   # ✅
print(dog.speak())            # 汪汪
```

## 3. 重点说明

### 3.1 抽象类的核心作用

| 作用 | 说明 |
|------|------|
| **定义接口** | 规定子类"必须有什么方法"，统一调用方式 |
| **防止实例化** | 抽象类不能直接创建对象 |
| **强制实现** | 子类如果没实现抽象方法，也无法实例化 |

### 3.2 抽象类 vs 普通父类

| 特性 | 普通父类 | 抽象类 |
|------|---------|--------|
| 可实例化 | ✅ | ❌ |
| 强制子类实现 | ❌（子类可跳过） | ✅（不实现就报错） |
| 适用场景 | 代码复用 | 接口规范 |

## 4. Demo 样例

```python
from abc import ABC, abstractmethod

class Shape(ABC):
    @abstractmethod
    def area(self):
        """计算面积"""
        pass

    @abstractmethod
    def perimeter(self):
        """计算周长"""
        pass

class Circle(Shape):
    def __init__(self, radius):
        self.radius = radius

    def area(self):            # 必须实现
        return 3.14 * self.radius ** 2

    def perimeter(self):       # 必须实现
        return 2 * 3.14 * self.radius

class Rectangle(Shape):
    def __init__(self, width, height):
        self.width, self.height = width, height

    def area(self): return self.width * self.height
    def perimeter(self): return 2 * (self.width + self.height)

shapes = [Circle(5), Rectangle(3, 4)]
for s in shapes:
    print(f"面积: {s.area():.1f}, 周长: {s.perimeter():.1f}")
# 面积: 78.5, 周长: 31.4
# 面积: 12.0, 周长: 14.0
```

## 5. 典型例题（附带答案）

题目：定义抽象类 `Vehicle`，要求子类实现 `move()` 和 `fuel_type()`。

答案：

```python
from abc import ABC, abstractmethod
class Vehicle(ABC):
    @abstractmethod
    def move(self): pass
    @abstractmethod
    def fuel_type(self): pass

class Car(Vehicle):
    def move(self): return "行驶"
    def fuel_type(self): return "汽油"
```

## 6. 易错点

### 易错点 1：忘记继承 ABC

错误：`class Animal:` + `@abstractmethod` → 不完全生效。正确：`class Animal(ABC):`

### 易错点 2：抽象类不能实例化

`Animal()` → TypeError。只有实现了全部抽象方法的子类才能实例化。

### 易错点 3：抽象方法不要写实现

`@abstractmethod` 修饰的方法在抽象类中通常只写 `pass`（接口定义）。

## 7. 课后练习（附带答案）

### 练习 1

题目：定义抽象类 `Database`，要求子类实现 `connect()` 和 `query(sql)`。

答案：

```python
class Database(ABC):
    @abstractmethod
    def connect(self): pass
    @abstractmethod
    def query(self, sql): pass
```

### 练习 2

题目：以下代码有什么问题？

```python
class Animal(ABC):
    @abstractmethod
    def speak(self): return "动物叫声"
class Dog(Animal): pass
d = Dog()
```

答案：Dog 未实现 `speak()`，不能实例化。加上 `def speak(self): return "汪汪"`。

## 8. 答题总结

- 抽象类（`ABC` + `@abstractmethod`）定义接口规范，强制子类实现特定方法。
- 抽象类不能实例化；子类未实现全部抽象方法也不能实例化。
- 抽象类 = 定规矩（接口），普通父类 = 提供默认实现（复用）。
- 常用于框架设计、插件系统、多态约束等场景。
