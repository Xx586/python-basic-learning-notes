# 反射

## 1. 知识点定义

**反射**（Reflection）是指程序在**运行时**能够获取对象的类型信息、属性、方法，并且可以动态地操作它们（获取、修改、调用），而不需要在写代码时就确定一切。

可以这样理解：正常情况下，我们在代码里写 `obj.name` 来访问属性，这在写代码时就固定了。而反射让我们能在程序运行时，根据一个字符串来决定访问哪个属性或调用哪个方法。

它主要解决的问题是：
- 根据用户输入或配置文件动态调用不同的功能
- 编写通用的框架代码（如插件系统、序列化工具）
- 在不知道对象具体类型的情况下操作对象

在实际写 Python 代码时，反射常用于框架开发、配置文件解析、插件系统、ORM（数据库映射）等场景。

> 小白理解：
> 假设你有一把万能遥控器（反射），你不知道要遥控的是电视还是空调，但只要你告诉遥控器"帮我找一下这个电器的'开关'按钮"，遥控器就能自动找到并按下。反射就是让程序在运行时自己去找对象上有没有某个属性或方法，而不需要提前在代码里写死。

---

## 2. 语法格式

Python 提供了四个核心内置函数来实现反射：

```python
# 1. hasattr(对象, "属性名或方法名")  — 判断对象有没有某个属性或方法
hasattr(obj, "name")     # 返回 True 或 False

# 2. getattr(对象, "属性名或方法名"[, 默认值])  — 获取对象的属性或方法
getattr(obj, "name")              # 等价于 obj.name
getattr(obj, "name", "未知")      # 如果属性不存在，返回默认值"未知"

# 3. setattr(对象, "属性名", 值)  — 给对象设置属性
setattr(obj, "name", "张三")      # 等价于 obj.name = "张三"

# 4. delattr(对象, "属性名")  — 删除对象的属性
delattr(obj, "name")              # 等价于 del obj.name
```

说明：
- 这四个函数通过**字符串**来操作对象的属性和方法。
- 普通写法是 `obj.xxx`，反射写法是用字符串 `"xxx"`。
- 字符串可以是变量，这意味着可以在运行时动态决定操作哪个属性。

## 3. 重点说明

### 3.1 普通调用 vs 反射调用

| 操作 | 普通写法 | 反射写法 | 说明 |
|------|----------|----------|------|
| 访问属性 | `obj.name` | `getattr(obj, "name")` | 属性名写死在代码里 vs 属性名是字符串变量 |
| 设置属性 | `obj.age = 18` | `setattr(obj, "age", 18)` | 同上 |
| 调用方法 | `obj.say_hello()` | `getattr(obj, "say_hello")()` | 先获取方法对象，再调用 |
| 判断存在 | `"name" in dir(obj)` | `hasattr(obj, "name")` | hasattr 更语义化 |

### 3.2 反射的核心价值：把"写死"变成"动态"

```python
# 没有反射：功能写死在代码里
def handle(action):
    if action == "run":
        run()
    elif action == "stop":
        stop()
    elif action == "pause":
        pause()
    # 每新增一个功能都要改这里的代码

# 有反射：根据字符串自动找到对应方法
def handle(action):
    method = getattr(self, action, None)  # 根据字符串找方法
    if method:
        method()                           # 找到了就调用
    else:
        print(f"不支持的操作：{action}")
```

> 反射的最大好处：增加新功能时不需要修改调用代码，只需要添加新方法即可。

### 3.3 什么时候用反射？

| 场景 | 说明 | 示例 |
|------|------|------|
| **命令分发** | 根据用户输入调用不同方法 | 菜单程序中根据选项调用对应函数 |
| **配置文件驱动** | 根据配置决定使用哪个类/哪个方法 | 从 JSON 配置中读取要调用的处理函数名 |
| **插件系统** | 动态加载和调用插件 | 框架自动发现并调用用户写的插件 |
| **序列化/反序列化** | 自动读取/设置对象属性 | ORM 把数据库行映射到对象属性 |

### 3.4 hasattr 和 getattr 的安全性

`getattr` 在属性不存在时默认会报 `AttributeError`。提供默认值可以避免报错：

```python
# 不安全的写法
value = getattr(obj, "name")    # 属性不存在时报 AttributeError

# 安全的写法
value = getattr(obj, "name", "默认值")  # 属性不存在时返回"默认值"
```

## 4. Demo 样例

### Demo 1：反射的四种基本操作

```python
class Student:
    """学生类，用于演示反射操作"""
    def __init__(self, name, age):
        self.name = name    # 实例属性：姓名
        self.age = age      # 实例属性：年龄
    
    def introduce(self):
        """自我介绍方法"""
        print(f"大家好，我叫{self.name}，今年{self.age}岁。")

# 创建一个学生对象
stu = Student("小明", 20)

# ===== 1. hasattr：判断有没有某个属性或方法 =====
print("有 name 属性吗？", hasattr(stu, "name"))        # True
print("有 score 属性吗？", hasattr(stu, "score"))       # False
print("有 introduce 方法吗？", hasattr(stu, "introduce")) # True

# ===== 2. getattr：获取属性或方法 =====
name_value = getattr(stu, "name")       # 等价于 stu.name
print("获取到的 name：", name_value)      # 小明

# 获取方法并调用
method = getattr(stu, "introduce")       # 获取方法对象
method()                                 # 调用方法，输出自我介绍

# 获取不存在的属性时给默认值
score = getattr(stu, "score", "暂无成绩") # score 不存在，返回默认值
print("score：", score)                   # 暂无成绩

# ===== 3. setattr：设置属性 =====
setattr(stu, "score", 95)                # 等价于 stu.score = 95
print("设置后的 score：", stu.score)      # 95
setattr(stu, "grade", "三年级")           # 甚至可以添加原本不存在的属性
print("新增的 grade：", stu.grade)        # 三年级

# ===== 4. delattr：删除属性 =====
delattr(stu, "grade")                    # 等价于 del stu.grade
print("删除后还有 grade 吗？", hasattr(stu, "grade"))  # False
```

运行结果：

```text
有 name 属性吗？ True
有 score 属性吗？ False
有 introduce 方法吗？ True
获取到的 name： 小明
大家好，我叫小明，今年20岁。
score： 暂无成绩
设置后的 score： 95
新增的 grade： 三年级
删除后还有 grade 吗？ False
```

### Demo 2：用反射实现简单的命令分发

```python
class Calculator:
    """计算器类，用反射实现命令分发"""
    
    def add(self, a, b):
        """加法"""
        return a + b
    
    def subtract(self, a, b):
        """减法"""
        return a - b
    
    def multiply(self, a, b):
        """乘法"""
        return a * b
    
    def divide(self, a, b):
        """除法"""
        if b == 0:
            return "错误：除数不能为0"
        return a / b
    
    def run(self, operation, a, b):
        """根据操作名称动态调用对应方法（反射的核心应用）"""
        # 先判断有没有这个方法
        if hasattr(self, operation):
            # 用 getattr 获取方法对象
            func = getattr(self, operation)
            # 调用方法并返回结果
            return func(a, b)
        else:
            return f"不支持的操作：{operation}"

# 使用计算器
calc = Calculator()

# 通过字符串指定要执行的操作
print(calc.run("add", 10, 5))          # 15  — 动态调用 add
print(calc.run("multiply", 3, 7))      # 21  — 动态调用 multiply
print(calc.run("divide", 100, 4))      # 25.0 — 动态调用 divide
print(calc.run("power", 2, 10))        # 不支持的操作：power

# 这样做的好处：以后增加新方法（比如 power），
# 不需要修改 run 方法的代码，直接添加新的方法即可
```

运行结果：

```text
15
21
25.0
不支持的操作：power
```

### Demo 3：根据用户输入动态访问属性

```python
class Person:
    """个人信息类"""
    def __init__(self):
        self.name = "张三"
        self.age = 25
        self.city = "北京"
        self.job = "程序员"

person = Person()

# 让用户输入想查看的信息
while True:

    if field == "q":
        print("再见！")
```

## 5. 典型例题（附带答案）

### 例题 1

题目：

有以下类和对象，请用反射的方式完成以下操作：

```python
class Book:
    def __init__(self):
        self.title = "Python入门"
        self.author = "张三"
        self.price = 59.9
    def get_info(self):
        return f"《{self.title}》作者：{self.author}，价格：{self.price}元"

book = Book()
```

要求：
1. 判断 book 有没有 `publisher` 属性
2. 用反射获取 `title` 属性的值
3. 用反射给 book 添加 `publisher` 属性，值为 "清华大学出版社"
4. 用反射调用 `get_info` 方法

答案：

```python
# 1. 判断有没有 publisher 属性
print(hasattr(book, "publisher"))  # False

# 2. 获取 title 属性
title = getattr(book, "title")
print(title)  # Python入门

# 3. 添加 publisher 属性
setattr(book, "publisher", "清华大学出版社")
print(book.publisher)  # 清华大学出版社

# 4. 调用 get_info 方法
method = getattr(book, "get_info")
result = method()
print(result)  # 《Python入门》作者：张三，价格：59.9元
```

解析：

- 四个反射函数的参数都是字符串。
- `getattr` 获取方法返回的是方法对象，需要再加一对括号 `()` 来调用。
- `setattr` 可以给对象添加原本不存在的属性。

### 例题 2

题目：

请用反射改写以下代码，消除重复的 if-elif 判断：

```python
class Animal:
    def dog(self):
        return "汪汪"
    def cat(self):
        return "喵喵"
    def duck(self):
        return "嘎嘎"

animal = Animal()
sound_type = input("请输入动物名（dog/cat/duck）：")

if sound_type == "dog":
    print(animal.dog())
elif sound_type == "cat":
    print(animal.cat())
elif sound_type == "duck":
    print(animal.duck())
else:
    print("不认识这个动物")
```

答案：

```python
class Animal:
    def dog(self):
        return "汪汪"
    def cat(self):
        return "喵喵"
    def duck(self):
        return "嘎嘎"

animal = Animal()
sound_type = input("请输入动物名（dog/cat/duck）：")

# 用反射代替 if-elif 链
if hasattr(animal, sound_type):
    method = getattr(animal, sound_type)
    print(method())
else:
    print("不认识这个动物")
```

解析：用 `hasattr` 判断方法是否存在，`getattr` 获取方法。以后添加新动物（如 `pig`）只需要增加一个 `pig` 方法，不需要修改调用代码。

## 6. 易错点

### 易错点 1：getattr 获取方法后忘记调用

错误写法：

```python
method = getattr(obj, "say_hello")   # 获取到了方法对象
print(method)                         # <bound method ...>，不是方法执行结果
```

错误原因：`getattr` 返回的是方法对象本身，还没有执行。

正确写法：

```python
method = getattr(obj, "say_hello")
method()                              # 加括号调用方法
# 或者一步到位：
getattr(obj, "say_hello")()           # 获取后直接调用
```

### 易错点 2：属性名写成了变量名

错误写法：

```python
attr_name = name          # 本意是字符串 "name"，但写成了变量 name
getattr(obj, attr_name)   # 如果 name 变量不存在，报 NameError
```

错误原因：反射的第二个参数应该是**字符串**。忘记加引号就变成了变量。

正确写法：

```python
getattr(obj, "name")      # 字符串 "name"
```

### 易错点 3：用 hasattr 检查方法时混淆了属性和方法

`hasattr` 可以用来检查属性和方法，但无法区分它们：

```python
class Demo:
    def __init__(self):
        self.value = 100   # 属性
    def run(self):         # 方法
        pass

d = Demo()
print(hasattr(d, "value"))  # True — 是属性
print(hasattr(d, "run"))    # True — 是方法
# hasattr 无法区分，如果需要区分，需要用 callable 判断
print(callable(getattr(d, "value")))  # False — value 不可调用
print(callable(getattr(d, "run")))    # True — run 可以调用
```

### 练习 1

题目：

有以下代码，请用反射的方式完成标注的操作：

```python
class Config:
    def __init__(self):
        self.host = "localhost"
        self.port = 8080

config = Config()
# 1. 用反射判断 config 有没有 debug 属性
# 2. 用反射获取 port 的值
# 3. 用反射给 config 添加 debug 属性，值为 True
```

答案：

```python
# 1. 判断有没有 debug 属性
print(hasattr(config, "debug"))  # False

# 2. 获取 port 的值
port = getattr(config, "port")
print(port)  # 8080

# 3. 添加 debug 属性
setattr(config, "debug", True)
print(config.debug)  # True
print(hasattr(config, "debug"))  # True
```

题目：

请用反射实现一个简单的"属性查看器"：让用户输入属性名，程序显示对象上该属性的值。如果属性不存在，显示"该属性不存在"。对象数据如下：

```python
    field = input("请输入要查看的属性名（输入 q 退出）：")
```

### 练习 3

题目：

请自己设计一个类，包含至少 3 个方法和 3 个属性。然后写一个程序，让用户输入方法名，程序用反射调用对应方法。如果方法不存在，给出友好提示。

答案：

```python
class MusicPlayer:
    """音乐播放器类"""
    def __init__(self):
        self.volume = 50         # 音量
        self.is_playing = False  # 是否正在播放
        self.current_song = ""   # 当前歌曲
    
    def play(self):
        self.is_playing = True
        return "开始播放音乐"
    
    def pause(self):
        self.is_playing = False
        return "暂停播放"
    
    def stop(self):
        self.is_playing = False
        self.current_song = ""
        return "停止播放"

player = MusicPlayer()

print("可用命令：play, pause, stop, quit")

while True:
    cmd = input("请输入命令：").strip().lower()
    
    if cmd == "quit":
        print("再见！")
        break
    
    if hasattr(player, cmd):
        method = getattr(player, cmd)
        result = method()
        print(result)
    else:
        print(f"未知命令：{cmd}，可用命令：play, pause, stop, quit")
```

说明：这个练习展示了反射在命令分发中的经典应用——用户输入什么字符串，就调用对应的方法。

## 8. 答题总结

- 反射让程序在运行时通过**字符串**动态操作对象的属性和方法，而不需要在代码中写死。
- Python 提供了四个反射函数：`hasattr`（判断有无）、`getattr`（获取值）、`setattr`（设置值）、`delattr`（删除属性）。
- 反射的核心价值是"动态"——增加新功能不需要修改调用代码。
- `getattr` 获取方法时返回的是方法对象，要加 `()` 才能调用。
- 反射常用于命令分发、插件系统、配置文件驱动等场景。
- 用 `callable(getattr(obj, name))` 可以判断一个名字对应的是属性还是方法。
