# 封装

## 1. 知识点定义

**封装**是把数据（属性）和操作数据的方法打包在一起，**隐藏内部实现细节**，只暴露必要的接口。Python 没有真正的"私有"，靠的是"君子协定"和名称改写。

> 小白理解：封装就像 ATM 机——你只能通过屏幕按钮（公开方法）操作，看不到内部的电路和钱箱（私有细节）。如果直接让你伸手进去拿钱（直接改属性），那太危险了。

## 2. 语法格式

```python
class BankAccount:
    def __init__(self, owner, balance=0):
        self.owner = owner           # 公开属性
        self._bank = "中国银行"       # 单下划线：约定私有（君子协定）
        self.__balance = balance     # 双下划线：名称改写（name mangling）

    def deposit(self, amount):
        """公开接口：存款"""
        if amount > 0:
            self.__balance += amount

    def get_balance(self):
        """公开接口：查询余额"""
        return self.__balance
```

## 3. 重点说明

### 3.1 Python 的三种访问级别

| 命名方式 | 示例 | 含义 |
|---------|------|------|
| `name` | `self.name` | 公开（public），外部直接访问 |
| `_name` | `self._name` | **约定**私有，"别碰我"（但不强制） |
| `__name` | `self.__name` | **名称改写**为 `_ClassName__name` |

### 3.2 名称改写的本质

```python
class Test:
    def __init__(self):
        self.__secret = 42

t = Test()
# print(t.__secret)          # ❌ AttributeError
print(t._Test__secret)       # ✅ 42 —— 还是能访问！只是名字变了
```

> Python 的理念是"We are all consenting adults"（大家都是成年人），不强制阻止访问，靠的是约定和信任。

### 3.3 封装的最佳实践

用 `@property` 提供受控访问：

```python
class Student:
    def __init__(self, name):
        self._name = name

    @property
    def name(self):
        return self._name     # 读取时做控制

    @name.setter
    def name(self, value):
        if not value.strip():
            raise ValueError("姓名不能为空")
        self._name = value.strip()
```

## 4. Demo 样例

```python
class BankAccount:
    def __init__(self, owner, balance=0):
        self.owner = owner           # 公开
        self.__balance = balance     # 私有（名称改写）

    def deposit(self, amount):
        if amount > 0:
            self.__balance += amount
            print(f"存入 {amount} 元，余额: {self.__balance}")
        else:
            print("存款金额必须为正")

    def withdraw(self, amount):
        if 0 < amount <= self.__balance:
            self.__balance -= amount
            print(f"取出 {amount} 元，余额: {self.__balance}")
        else:
            print("余额不足或金额无效")

    def get_balance(self):
        return self.__balance

acc = BankAccount("小明", 1000)
acc.deposit(500)     # 存入 500 元，余额: 1500
acc.withdraw(200)    # 取出 200 元，余额: 1300
# acc.__balance      # ❌ AttributeError
print(acc.get_balance())  # ✅ 1300
```

## 5. 典型例题（附带答案）

### 例题 1

题目：以下代码报错的原因是什么？

```python
class A:
    def __init__(self):
        self.__x = 10

a = A()
print(a.__x)  # AttributeError!
```

答案：`__x` 被名称改写为 `_A__x`，外部不能通过 `__x` 直接访问。需要通过 `a._A__x` 或提供公开方法。

### 例题 2

题目：单下划线 `_name` 和双下划线 `__name` 的区别。

答案：`_name` 是约定（不建议外部访问，但不强制）；`__name` 触发名称改写，改为 `_ClassName__name`，提供一定程度的保护但不是真正的私有。

## 6. 易错点

### 易错点 1：以为双下划线是"真正"的私有

Python 没有 Java/C++ 那样的 private。`__name` 只是被改名了，仍可通过 `_ClassName__name` 访问。

### 易错点 2：`__init__` 等魔术方法不是"私有"

`__init__`、`__str__` 等双下划线包围的方法是魔术方法，有特殊含义，不会被名称改写。

### 易错点 3：属性直接暴露

`account.balance = 999999` 可被任意修改。用 `@property` + `_` 前缀来保护。

## 7. 课后练习（附带答案）

### 练习 1

题目：为 `Student` 类添加 `@property` 保护 `_score`，确保分数 0-100。

答案：

```python
class Student:
    def __init__(self, score):
        self._score = None
        self.score = score  # 触发 setter
    @property
    def score(self): return self._score
    @score.setter
    def score(self, value):
        if 0 <= value <= 100: self._score = value
        else: raise ValueError("分数 0-100")
```

## 8. 答题总结

- 封装隐藏内部细节，通过公开方法访问数据，Python 靠"君子协定"。
- 单下划线 `_name`：约定私有（建议不碰）。双下划线 `__name`：名称改写为 `_ClassName__name`。
- `@property` 提供受控的属性访问（getter/setter），是 Python 的最佳实践。
- Python 没有真正的 private，设计理念是"大家都是成年人"——信任开发者。
- `__init__` 等魔术方法不是私有，不会被名称改写。
