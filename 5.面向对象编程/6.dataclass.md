# dataclass

## 1. 知识点定义

`dataclass` 是 Python 3.7+ 引入的装饰器，自动生成 `__init__`、`__repr__`、`__eq__` 等方法，大幅减少样板代码。它是"懒人版的类定义"，专为**存储数据**的类设计。

> 小白理解：传统类像从零开始搭房子——墙、门、窗都要自己建。dataclass 像买预制房——基本结构已经搭好，你只需要填家具（属性）就行。

## 2. 语法格式

```python
from dataclasses import dataclass

@dataclass
class Student:
    name: str              # 类型注解定义属性
    age: int = 18          # 默认值
    scores: list = None    # 可变默认值要用 None 处理

# 自动生成 __init__、__repr__、__eq__
s1 = Student("小明", 20)
s2 = Student("小明", 20)
print(s1)              # Student(name='小明', age=20, scores=None)
print(s1 == s2)        # True（自动按字段值比较）
```

### 常用参数

| 参数 | 默认 | 作用 |
|------|------|------|
| `frozen=True` | False | 设为不可变（类似元组） |
| `order=True` | False | 自动生成 `__lt__` 等排序方法 |
| `slots=True` | False | 使用 __slots__ 节省内存 |

## 3. 重点说明

### 3.1 dataclass vs 传统类

```python
# 传统写法：~10 行
class Point:
    def __init__(self, x, y):
        self.x, self.y = x, y
    def __repr__(self):
        return f"Point(x={self.x}, y={self.y})"
    def __eq__(self, other):
        return self.x == other.x and self.y == other.y

# dataclass 写法：3 行
@dataclass
class Point:
    x: int
    y: int
```

### 3.2 字段默认值注意事项

可变默认值（list/dict）**不能**直接写，会触发 TypeError：

```python
@dataclass
class Student:
    name: str
    scores: list = []  # ❌ TypeError！
    # ✅ 正确：scores: list = field(default_factory=list)
```

## 4. Demo 样例

```python
from dataclasses import dataclass, field

@dataclass
class Student:
    name: str
    age: int
    scores: list = field(default_factory=list)  # 可变默认值

    def average(self):
        if not self.scores:
            return 0
        return sum(self.scores) / len(self.scores)

# 使用
s1 = Student("小明", 18, [85, 92, 78])
s2 = Student("小明", 18, [85, 92, 78])
print(s1)                     # Student(name='小明', age=18, scores=[85, 92, 78])
print(s1 == s2)               # True（自动比较字段值）
print(f"平均分: {s1.average():.1f}")  # 85.0
```

## 5. 典型例题（附带答案）

题目：用 dataclass 定义 `Book`，包含 title、author、pages（默认 0）。

答案：

```python
@dataclass
class Book:
    title: str
    author: str
    pages: int = 0
```

## 6. 易错点

### 易错点 1：可变默认值

`list = []` 在 dataclass 中直接报错。用 `field(default_factory=list)`。

### 易错点 2：dataclass 不适合复杂业务逻辑

dataclass 适合纯数据类。如果有复杂的业务逻辑、方法、继承层次，用传统类更合适。

### 易错点 3：字段顺序

dataclass 按字段声明顺序生成 `__init__`，有默认值的字段必须在无默认值字段之后。

## 7. 课后练习（附带答案）

### 练习 1

题目：用 dataclass 定义 `Point`，包含 x、y 坐标，并用 `frozen=True` 让它不可变。

答案：

```python
@dataclass(frozen=True)
class Point:
    x: int
    y: int
```

### 练习 2

题目：用 dataclass 定义 `Config`，包含 host（默认 "localhost"）、port（默认 8080）。

答案：`@dataclass class Config: host: str = "localhost"; port: int = 8080`

## 8. 答题总结

- `@dataclass` 自动生成 `__init__`、`__repr__`、`__eq__`，减少样板代码。
- 数据类型用 dataclass 非常方便；复杂业务逻辑用传统类。
- 可变默认值用 `field(default_factory=list)`，不能直接写 `= []`。
- `frozen=True` 创建不可变数据类；`order=True` 支持排序比较。
- dataclass 不等于"万能类"，复杂继承或行为多的场景仍用传统类。
